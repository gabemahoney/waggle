## Context

After Epic 2 Task 1 lands, `spawn_worker` blocks until Claude Status registration. The README's current `spawn_worker` section (updated for Epic 1's 12-option surface) still implies fire-and-forget semantics. Operators reading the post-Epic-1 docs will not know that they can immediately use the returned `tmux_session_name` for `send_input` and friends, nor that the call can fail with the two new error names.

## What Needs to Change

In `README.md`, add a "Readiness blocking" subsection (or paragraph) inside the `spawn_worker` documentation block, after the parameter and settings-stack subsections from Epic 1. The subsection MUST state:

- `spawn_worker` blocks until the spawned worker appears in Claude Status's `instances` table under the returned `instance_id`. Callers can immediately use `send_input`, `get_output`, etc. against the returned `tmux_session_name` once the call returns.
- The default readiness timeout is **15 seconds** (normative — SR-5.2). The timeout is not configurable per-call or per-template in v1.
- On timeout, the call returns `ErrSpawnReadinessTimeout` and Claude Spawn automatically tears down the unregistered tmux session via `tmux kill-session`. Operators do not need to manually clean up the orphaned session.
- On worker early exit (the tmux session ends before registration), the call returns `ErrSpawnWorkerExitedEarly` with captured pane output in the error description for diagnostics.

Out of scope: do NOT add template-loading content, `list_templates`/`write_template` content, CLI subcommand content, or any per-call/template timeout override — those are sibling-Epic scope and were explicitly excluded from v1.

## Key Files

- `README.md` — new "Readiness blocking" subsection within the `spawn_worker` documentation block.

## Acceptance Criteria

- The README explicitly names the **15-second** default timeout.
- Both `ErrSpawnReadinessTimeout` and `ErrSpawnWorkerExitedEarly` are named with a one-line description each.
- The README explicitly states that the timeout is not caller-configurable in v1.
- The README explicitly describes the automatic `kill-session` cleanup on timeout, so an operator understands they will not accumulate orphaned tmux sessions.
- No template, `list_templates`, `write_template`, or CLI content is added in this subtask.
