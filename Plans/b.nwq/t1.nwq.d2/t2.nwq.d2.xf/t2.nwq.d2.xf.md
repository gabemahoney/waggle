# Task 2 — Update README and `docs/architecture/spawn_worker.md` for spawn-readiness blocking

## Context

After Task 1 lands, `spawn_worker` blocks until Claude Status registration. SR-12.1 requires this behavior in the README; SR-12.2 requires the existing arch doc to be updated for the readiness wait. Operators reading the post-Epic-1 docs will see fire-and-forget semantics that no longer match shipped behavior.

## What Needs to Change

- `README.md`: add a "Readiness blocking" subsection (or paragraph) to the `spawn_worker` documentation block. State the 15-second default; name both new error names (`ErrSpawnReadinessTimeout`, `ErrSpawnWorkerExitedEarly`); describe the automatic `tmux kill-session` cleanup on timeout so operators know orphaned sessions are not accumulating.
- `docs/architecture/spawn_worker.md`: expand the flow section with the readiness wait (poll → match by `instance_id` → success; or `has-session` probe → capture-pane → `ErrSpawnWorkerExitedEarly`; or timeout → `kill-session` → `ErrSpawnReadinessTimeout`). Update the Mermaid sequence diagram. Add the two new error names to the errors table.

Out of scope: template-related docs, the `docs/architecture/spawn_options_and_templates.md` arch doc, `list_templates`/`write_template`/CLI doc updates — all sibling Epics.

## Why

Brings shipped behavior and documentation back in sync after the readiness wait lands.

## Success Criteria

- README explicitly names the 15-second timeout default.
- README names both `ErrSpawnReadinessTimeout` and `ErrSpawnWorkerExitedEarly`.
- README describes the automatic cleanup on timeout.
- `docs/architecture/spawn_worker.md`'s flow section and Mermaid diagram include the readiness loop with timeout and early-exit branches.
- `docs/architecture/spawn_worker.md`'s errors table includes both new error names.

## Dependencies
