# Task 1 — Implement `claude-spawn write-template` CLI subcommand + tests

## Context

Epic 5 (`t1.nwq.vv`) provides `write_template_impl(name, options, force)` in `src/claude_spawn/templates.py`. SR-7.5 requires a CLI subcommand `claude-spawn write-template <name>` exposing the same authoring surface as the MCP tool, with both an interactive mode and a flag-driven mode. Both modes delegate to the same `write_template_impl` so the two surfaces produce identical files for identical inputs.

## What Needs to Change

- `src/claude_spawn/cli.py` — register a new `write-template` subparser via the existing subparsers registry. Two mutually-exclusive modes:
  - **Flag-driven (default):** scalars (`--model`, `--thinking`, `--cwd`, `--tmux-session-name`, `--instance-id`, `--claude-home`, `--claude-settings`); repeatable list flag `--claude-arg VALUE`; repeatable entry-form flags for maps and permission lists (`--extra-env-entry KEY=VALUE`, `--claude-status-labels-entry KEY=VALUE`, `--permissions-allow PATTERN`, `--permissions-deny PATTERN`, `--permissions-ask PATTERN`); `--force`.
  - **Interactive (`--interactive`):** prompt-driven walk through every option (except `template`) with current-value display, `skip` token support, and clean cancellation handling.
- Output convention: success prints a single JSON line on stdout matching `write_template_impl`'s return dict; impl errors print the error envelope on stdout (matching `ClaudeSpawnArgumentParser`'s convention). Exit 0 on success, exit 2 on error. Cancellation in interactive mode (Ctrl-C, EOF on stdin) prints `{"status":"error","message":"write-template cancelled"}` and writes no file.
- New `tests/test_cli_write_template.py` exercising both modes via `monkeypatch.setattr` on `sys.argv` (matching `tests/test_cli.py`'s pattern); interactive mode driven via `monkeypatch.setattr(builtins, "input", ...)`. Includes a cross-surface equivalence assertion (CLI flag-driven + CLI interactive + MCP `write_template` for the same input bundle produce identical files).

## Why

Closes the final operator-visible authoring surface. An operator can now write `claude-spawn write-template orch --cwd=/path --model=opus --permissions-allow=Bash --force` and spawn from the resulting template.

## Success Criteria

- `claude-spawn write-template foo --cwd=/tmp` exits 0 with a single JSON success line on stdout; the resulting file round-trips through the Epic 3 loader and a subsequent `spawn_worker(template="foo")` succeeds.
- Interactive mode driven by patched `builtins.input` produces an option dict matching the flag-driven equivalent.
- Interactive cancellation (EOFError or KeyboardInterrupt) exits non-zero with the documented cancellation JSON and writes no file.
- Mode conflict (`--interactive` + option flag) is an argparse-level error with exit 2 and no file written.
- Every impl-level error (`ErrTemplateNameUnsafe`, `ErrTemplateOptionsInvalid`, `ErrTemplateExists`) surfaces unchanged through the CLI as a JSON error line on stdout with exit 2.
- Same input bundle via flag-driven CLI, interactive CLI, and `write_template` MCP all produce byte-identical files.
- Full `pytest` is green.

## Dependencies

- Epic 5 (`t1.nwq.vv`) — provides `write_template_impl`.
