## Context

SR-11.6 mandates a new test file `tests/test_cli_write_template.py` exercising the CLI subcommand via direct `main()` invocation with `monkeypatch.setattr(sys, "argv", [...])` (matching the existing pattern in `tests/test_cli.py`). Interactive mode is driven via `monkeypatch.setattr(builtins, "input", ...)`. Tests use Epic 3's `fake_templates_dir` and `TEMPLATE_TOML_*` constants — no hardcoded TOML literals; no real-filesystem access.

## What Needs to Change

Create `tests/test_cli_write_template.py` with the following test classes / cases:

**`TestWriteTemplateFlagDriven`**
- Minimal happy path: `write-template foo --cwd=/tmp` → exit 0; stdout is one JSON line with `path` and `options`; the resulting file's content is byte-identical to the file produced by a direct `write_template_impl("foo", {"cwd":"/tmp"}, False)` call against the same `fake_templates_dir`.
- Full-options: every scalar flag plus repeatable flags (`--claude-arg` twice, `--extra-env-entry` twice, `--claude-status-labels-entry` twice, `--permissions-allow` twice, `--permissions-deny`, `--permissions-ask`) → exit 0; loaded file matches the programmatically-constructed expected option bundle.

**`TestWriteTemplateInteractive`**
- Happy path: patch `builtins.input` to return values for each prompt (mixing real values and `skip` tokens); run `write-template orch2 --interactive`; exit 0; file written; loaded options equal those produced by a flag-driven CLI call with the same effective inputs.
- Cancellation via `EOFError`: patch `builtins.input` to raise `EOFError` after some number of prompts → exit non-zero; stdout JSON matches `{"status":"error","message":"write-template cancelled"}`; no file created.
- Cancellation via `KeyboardInterrupt`: same as above but with `KeyboardInterrupt`.

**`TestWriteTemplateModeConflict`**
- `write-template foo --interactive --cwd=/tmp` → argparse-level error (SystemExit(2)); JSON error envelope on stdout; nothing written.

**`TestWriteTemplateErrorPassthrough`**
- Invalid `--thinking=huge` → exit 2; stdout JSON contains `err_name=ErrTemplateOptionsInvalid` (the impl rejects via the same validation surface).
- Unsafe `name`: `write-template foo/bar --cwd=/tmp` → exit 2; stdout JSON contains `err_name=ErrTemplateNameUnsafe`.
- Collision without `--force`: two consecutive flag-driven calls with the same name → second exits 2; stdout JSON contains `err_name=ErrTemplateExists`.
- Collision with `--force`: same setup, `--force` on the second call → exit 0; file replaced; loaded options reflect second call's inputs.

**`TestWriteTemplateCrossSurfaceEquivalence`**
- Pick one option bundle. Author via (a) flag-driven CLI; (b) interactive CLI driven by patched `input`; (c) `write_template` MCP tool. Assert all three files load to identical resolved option maps via the Epic 3 loader. This embodies the "both surfaces produce identical files for identical inputs" property.

Constraints:
- Every test uses `fake_templates_dir` (no real-filesystem access).
- No hardcoded TOML literals; use `TEMPLATE_TOML_*` or programmatic `tomli_w` construction.
- Tests follow the `monkeypatch.setattr(sys, "argv", [...])` pattern from `tests/test_cli.py`.
- `capsys` (or equivalent) captures stdout/stderr; tests assert stdout contains the JSON line and stderr is empty (or, for interactive mode, contains the prompts).
