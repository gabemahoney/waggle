## Context

SR-7.5 introduces a `claude-spawn write-template <name>` CLI subcommand exposing the same authoring surface as the MCP tool. Both surfaces delegate to the single `write_template_impl(name, options, force)` function (Epic 5) so they produce identical files for identical inputs. This subtask adds the subcommand registration and the default flag-driven mode; the `--interactive` mode is layered on top in a sibling subtask.

## What Needs to Change

In `src/claude_spawn/cli.py`:
- Register a new `write-template` subparser via the existing `subparsers` registry. The subparser uses the `ClaudeSpawnArgumentParser` subclass and its JSON-error-on-stderr convention.
- The subparser accepts:
  - Positional `name` (string).
  - Scalar flags: `--model`, `--thinking`, `--cwd`, `--tmux-session-name`, `--instance-id`, `--claude-home`, `--claude-settings`.
  - Repeatable list flag: `--claude-arg VALUE` (may appear multiple times; collects in order to a list).
  - Repeatable map-entry flags: `--extra-env-entry KEY=VALUE` and `--claude-status-labels-entry KEY=VALUE` (parse `KEY=VALUE` on each occurrence; collect into a dict).
  - Repeatable permission-list flags: `--permissions-allow PATTERN`, `--permissions-deny PATTERN`, `--permissions-ask PATTERN` (collect into a `permissions` sub-dict with `allow`/`deny`/`ask` arrays).
  - Boolean `--force`.
- Set up a mutually-exclusive argparse group for the option flags vs. `--interactive` (the sibling subtask adds `--interactive` to this group).
- Add a dispatch branch in `main()` for `args.subcommand == "write-template"` that:
  - Assembles an `options` dict from the parsed flags. Omit keys whose flags were not supplied. Map-entry flags split on `=` and accumulate; permission-list flags accumulate into `permissions["allow"|"deny"|"ask"]`.
  - Calls `write_template_impl(name, options, force)`.
  - On success: prints a single JSON line (the impl's return dict: `{path, options}`) to stdout and exits 0.
  - On any impl-level error (return dict with `err_name` key): prints the error envelope as a JSON line on stdout and exits 2.
  - Uses `ClaudeSpawnArgumentParser.error` only for argparse-level parse failures (not for impl errors).

## Key Files

- `src/claude_spawn/cli.py` — subparser registration + `main()` dispatch branch.
- `src/claude_spawn/templates.py` — read-only; provides `write_template_impl`.

## Acceptance Criteria

- `claude-spawn write-template foo --cwd=/tmp` exits 0 and prints a single JSON line with `path` and `options` keys.
- Every scalar flag round-trips: `--model`, `--thinking`, `--cwd`, `--tmux-session-name`, `--instance-id`, `--claude-home`, `--claude-settings` each appear under their SR-1.1 option name in the written file.
- `--claude-arg A --claude-arg B` produces `claude_args=["A","B"]` (order preserved).
- `--extra-env-entry FOO=bar --extra-env-entry BAZ=qux` produces `extra_env={"FOO":"bar","BAZ":"qux"}`. Same shape for `--claude-status-labels-entry`.
- `--permissions-allow X --permissions-allow Y` produces `permissions.allow=["X","Y"]`. Same for `deny` and `ask`.
- `--force` forwards to `write_template_impl`; a second call without `--force` returns `ErrTemplateExists` as a JSON line on stdout with exit 2.
- Impl-level errors (`ErrTemplateNameUnsafe`, `ErrTemplateOptionsInvalid`, `ErrTemplateExists`) appear on stdout as a single JSON line with exit 2; no file is written when the impl returns an error.
- `claude-spawn write-template --help` exits 0 and lists the subcommand's flags.
