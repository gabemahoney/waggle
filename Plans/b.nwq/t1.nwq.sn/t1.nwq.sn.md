# Epic 6 — `claude-spawn write-template` CLI subcommand

Add the operator-facing CLI surface for authoring templates, with both interactive and flag-driven modes, delegating to `write_template_impl` so both surfaces produce identical files.

## Scope

- **Subcommand registration (SR-7.5):** new `write-template <name>` subcommand in `src/claude_spawn/cli.py` via the existing argparse `subparsers` registry. Continues to use `ClaudeSpawnArgumentParser` (post-rename of `WaggleArgumentParser`) with its JSON-error-on-stderr convention.
- **Flag-driven mode (default):** every option in SR-1.1 except `template` is exposed as a CLI flag.
  - Scalars: `--model`, `--thinking`, `--cwd`, `--tmux-session-name`, `--instance-id`, `--claude-home`, `--claude-settings`.
  - List: `--claude-arg VALUE` (repeatable; singular flag name, ordered).
  - Maps and permission lists:
    - `--extra-env-entry KEY=VALUE` (repeatable).
    - `--claude-status-labels-entry KEY=VALUE` (repeatable).
    - `--permissions-allow PATTERN` (repeatable).
    - `--permissions-deny PATTERN` (repeatable).
    - `--permissions-ask PATTERN` (repeatable).
  - `--force` opt-in for overwrite.
- **Interactive mode (`--interactive`):** mutually exclusive with flag-driven mode. Walks through every option (except `template`) one prompt at a time, displaying option name, short description, and (if a template with that name already exists) the current resolved value. Operator types a value or the literal token `skip` to leave unset. After the last prompt, delegates to `write_template_impl` with the collected options.
- **Single delegation point:** both modes call `write_template_impl` (defined in Epic 5). The CLI does no schema validation of its own beyond the argparse-level parsing — schema validation errors come back as `ErrTemplateOptionsInvalid` etc. from the impl.
- **Output convention:** success prints a single JSON line on stdout matching the `write_template` impl's return dict (`{path, options}`). Errors print a single JSON error line on stdout (matching `ClaudeSpawnArgumentParser.error`); the subcommand exits 0 on success, 2 on error. Interactive cancellation (Ctrl-C, EOF on stdin) prints `{"status":"error","message":"write-template cancelled"}` and writes no file.
- **Tests (SR-11.6):** new `tests/test_cli_write_template.py` exercising the subcommand via direct `main()` calls with `monkeypatch.setattr` on `sys.argv` (matching the existing pattern in `tests/test_cli.py`). Coverage:
  - Flag-driven minimal: `claude-spawn write-template foo --cwd=/tmp` → JSON success line; resulting file matches a programmatic `write_template_impl` call with the same input.
  - Flag-driven full: every scalar flag, repeatable map/list flags exercised together.
  - Interactive: `monkeypatch.setattr(builtins, "input", ...)` to drive prompts (one prompt per option), collected values match expected file.
  - Interactive cancellation: input raises `EOFError`/`KeyboardInterrupt`; assert cancellation JSON, exit code non-zero, no file written.
  - Error pass-through: invalid `--thinking`, unsafe `name`, collision-without-`--force`, collision-with-`--force` — each produces the corresponding error line from `write_template_impl` and an exit-2.
  - Mode collision: passing both `--interactive` and an option flag is an argparse-level conflict; verify the JSON error envelope.
- **Documentation:** README documents both modes with worked examples — including a one-liner flag-driven invocation and a sample interactive transcript. Repeatable-flag forms for `--extra-env-entry`, `--claude-status-labels-entry`, `--permissions-{allow,deny,ask}`, and `--claude-arg` are shown. The architecture doc (`docs/architecture/spawn_options_and_templates.md`) notes that CLI and MCP authoring converge on a single `write_template_impl`.

## Out of scope for this Epic

- Any new authoring surface beyond CLI + MCP (e.g. a web UI, an editor extension).
- Per-call invocation of `spawn_worker` from the CLI (separate concern, not part of this feature).

## Acceptance Criteria

1. **Flag-driven happy path.** Run `claude-spawn write-template orch --cwd=/tmp --model=opus --thinking=xhigh --permissions-allow=Bash --permissions-deny=WebFetch --extra-env-entry FOO=bar --claude-status-labels-entry role=orchestrator --claude-arg --dangerously-skip-permissions` against `fake_templates_dir({})`. Assert: subprocess exits 0; stdout is a single JSON line with `{path, options}`; loading the file via Epic 3 yields the expected option bundle; spawning via `spawn_worker(template="orch")` produces the expected tmux invocation.
2. **Interactive happy path.** With `builtins.input` patched to return values in order (e.g. `/tmp`, `opus`, `xhigh`, `skip`, …), run `claude-spawn write-template orch2 --interactive`. Assert: file written; option map equivalent to a programmatic `write_template_impl` call with the same inputs; `skip` answers map to unset options.
3. **Interactive cancellation.** With `builtins.input` raising `EOFError` after the second prompt, run `claude-spawn write-template foo --interactive`. Assert: exit code ≠ 0; stdout JSON has `{"status":"error","message":"write-template cancelled"}`; no file written under `fake_templates_dir`.
4. **Mode conflict.** Run `claude-spawn write-template foo --interactive --cwd=/tmp`. Assert: argparse-level error (mutually-exclusive group), JSON error envelope on stdout, exit-2, no file written.
5. **Error pass-through.** Each of these produces the corresponding error JSON and exit-2, identical to the MCP error surface:
   - `claude-spawn write-template foo --thinking=huge` → `ErrThinkingInvalid` (or `ErrTemplateOptionsInvalid` per SRD mapping).
   - `claude-spawn write-template foo/bar --cwd=/tmp` → `ErrTemplateNameUnsafe`.
   - Two consecutive flag-driven calls with the same `name` (no `--force`) — second returns `ErrTemplateExists`.
   - Same setup with `--force` on the second call — exit 0 and the file is replaced.
6. **Surfaces converge.** Pick a single option bundle. Author it via (a) flag-driven CLI, (b) interactive CLI, (c) `write_template` MCP, (d) hand-edit (write the TOML via `fake_templates_dir`'s direct path). All four files load via Epic 3's loader to identical resolved option maps.
7. **Test suite green.** `pytest tests/test_cli_write_template.py` plus the full suite pass.
8. **Docs.** README has a `claude-spawn write-template` section with at least one flag-driven example and one interactive transcript example, plus a table of every CLI flag mapped to its SR-1.1 option name.
