## Context

SR-12.1 requires the README to document the `claude-spawn write-template` CLI subcommand with both interactive and flag-driven mode examples, the repeatable-flag forms for map-valued options, `--force` behavior, and a flag-to-option mapping table. Epic 3 added the README "Templates" section; Epics 4 and 5 extended it with `list_templates` and `write_template` MCP subsections. This subtask adds the CLI subsection.

## What Needs to Change

In `README.md`, add a CLI subsection within the existing "Templates" section. Cover:

- **One-liner flag-driven example** — a complete, copy-paste-ready invocation exercising several option types, e.g.:
  ```
  claude-spawn write-template orch \
    --cwd=/home/horde/projects/myrepo \
    --model=opus \
    --thinking=xhigh \
    --permissions-allow=Bash \
    --permissions-deny=WebFetch \
    --extra-env-entry FOO=bar \
    --claude-status-labels-entry role=orchestrator \
    --claude-arg --dangerously-skip-permissions
  ```

- **Interactive transcript example** — show at least three prompts and the operator's replies (including one `skip` to illustrate omitting an option), ending with the success JSON line on stdout.

- **Repeatable-flag forms** — short multi-entry example illustrating that `--extra-env-entry`, `--claude-status-labels-entry`, `--permissions-allow`, `--permissions-deny`, `--permissions-ask`, and `--claude-arg` are each specified once per entry (repeat the flag to accumulate).

- **`--force` flag** — one sentence: without `--force` a name collision returns `ErrTemplateExists`; with `--force` the file is atomically overwritten.

- **Flag-to-option table** — every CLI flag mapped to its SR-1.1 option name (and the special `--force` / `--interactive` rows clearly marked as not corresponding to an SR-1.1 option).

- **Cancellation note** — one sentence: Ctrl-C or EOF during interactive mode exits non-zero and writes no file.

## Key Files

- `README.md` — new CLI subsection within the existing "Templates" section.

## Acceptance Criteria

- The README has a `claude-spawn write-template` CLI subsection with: a complete flag-driven one-liner; an interactive transcript example; an explicit demonstration of repeatable-flag forms; the `--force` behavior described; a complete flag-to-option mapping table; the cancellation note.
- Every CLI flag exposed by the implementation appears in the flag-to-option table.
- The flag-to-option table marks `--force` and `--interactive` as not mapping to an SR-1.1 option (they are mode/behavior controls).
