# Task 1 — Lift Python floor to ^3.11 and add `tomli-w` runtime dep

## Context

SR-6.2 commits to TOML as the template file format and requires the Python floor to be ^3.11 so `tomllib` (stdlib in 3.11+) is available without a backport dependency. `tomli-w` is added as a runtime dependency for writing TOML (consumed by Epic 5's `write_template`, but the dep is added here as part of finalizing the template-file story so subsequent Epics build on the same dep surface).

## What Needs to Change

- Update `pyproject.toml` Python version constraint from `^3.10` (current) to `^3.11`.
- Add `tomli-w` as a runtime dependency.

## Why

The loader (Task 2) cannot import `tomllib` without the Python floor lift. Adding `tomli-w` here keeps the template dependency surface in one commit rather than introducing it later.

## Success Criteria

- `pyproject.toml` declares `python = "^3.11"`.
- `tomli-w` appears as a runtime dependency under `[tool.poetry.dependencies]`.
- `poetry install` (or equivalent) succeeds in the project venv.
- `python -c "import tomllib, tomli_w"` succeeds from within the venv.

## Dependencies

None.
