## Context

The template loader added in Task 2 imports `tomllib` from the Python standard library, which requires Python 3.11+. The project's current Python floor is `^3.10`. Additionally, `tomli-w` (the write-only counterpart to `tomllib`) is needed for Epic 5's `write_template`, but is added here as part of finalizing the template-file story so subsequent Epics build on the same dep surface.

## What Needs to Change

In `pyproject.toml`:
- Change the Python version constraint under `[tool.poetry.dependencies]` from `^3.10` to `^3.11`.
- Add `tomli-w` as a runtime dependency (not dev-only) under `[tool.poetry.dependencies]`. Pin to a recent stable version per project conventions.

No other files are modified in this subtask.

## Key Files

- `pyproject.toml` — version constraint update + new runtime dep.

## Acceptance Criteria

- `pyproject.toml` declares `python = "^3.11"`.
- `tomli-w` appears as a runtime dependency in `[tool.poetry.dependencies]`.
- `poetry check` (or `poetry install --dry-run`) succeeds with no errors.
- `python -c "import tomllib, tomli_w"` succeeds from within the project venv.
- `poetry.lock` is updated to include `tomli-w` and reflect the new Python floor.
