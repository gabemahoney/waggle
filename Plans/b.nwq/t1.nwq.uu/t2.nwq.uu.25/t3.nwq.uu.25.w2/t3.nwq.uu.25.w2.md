## Context

Task 3's engineer subtask wires `template=<name>` into `spawn_worker_impl` with full SR-2 resolution and merge semantics. The `tests/test_engine_spawn_worker_options.py` file (created in Epic 1) is the canonical home for these integration cases per SR-11.6. This subtask adds 12 template integration test cases covering every Epic 3 Acceptance Criterion.

## What Needs to Change

Extend `tests/test_engine_spawn_worker_options.py` with new test class(es) (e.g. `TestTemplateIntegration`) covering:

1. **Bare-spawn-no-read:** call `spawn_worker_impl(cwd="/tmp")` with NO `template` argument against `fake_templates_dir({"orch": TEMPLATE_TOML_FULL})`. Assert: the loader's `_read_template_file` mock was NOT called (verifiable via `mock.assert_not_called()`).
2. **Happy path:** drop a template via `fake_templates_dir({"orch": TEMPLATE_TOML_FULL})`. Call `spawn_worker_impl(template="orch")` (no per-call options). Assert: the captured tmux invocation reflects the options supplied by the template (e.g. `cwd`, `model`, `thinking`, `extra_env`).
3. **Per-call scalar overrides template scalar:** template sets `model="opus"`; per-call passes `model="sonnet"`. Assert: captured invocation uses `model=sonnet`.
4. **Map merge — `extra_env` union with per-call wins:** template `extra_env={"FOO":"1","BAR":"2"}`; per-call `extra_env={"BAR":"3","BAZ":"4"}`. Assert: resolved `extra_env={"FOO":"1","BAR":"3","BAZ":"4"}` reflected in the captured `-e KEY=VAL` env vars.
5. **Map merge — `claude_status_labels` union with per-call wins:** mirror case 4 for labels.
6. **`permissions` map merge — dual assertion (per-call replaces same-key, template preserves different-key):** template `permissions={"allow":["B"],"deny":["D"]}`; per-call `permissions={"allow":["A"]}`. Assert: resolved permissions are `{"allow":["A"],"deny":["D"]}` (allow REPLACED wholesale at the `allow` key; deny PRESERVED from template). Both halves of this assertion must be exercised in the same test case.
7. **List replacement:** template `claude_args=["--a","--b"]`; per-call `claude_args=["--c"]`. Assert: captured `send-keys` shows only `--c` (no `--a`/`--b`).
8. **No caching:** drop a template, call `spawn_worker_impl(template="orch")` once; mutate the on-disk template file via `fake_templates_dir`'s temp directory (rewrite the same name with different content); call `spawn_worker_impl(template="orch")` again. Assert: the second call observes the new content (verifying SR-6.6's no-caching requirement).
9. **Required-`cwd` satisfied by template:** template supplies `cwd`; per-call omits `cwd`. Assert: spawn succeeds using template's `cwd`.
10. **Required-`cwd` NOT satisfied:** template body has no `cwd`; per-call omits `cwd`. Assert: `err_name="ErrCwdMissing"` (covers Epic 3 AC-9 negative half).
11. **Missing template:** `spawn_worker_impl(template="nope")` against an empty `fake_templates_dir({})`. Assert: `err_name="ErrTemplateNotFound"`; description names both `"nope"` AND the templates directory.
12. **Malformed template surfaces `ErrTemplateMalformed`:** use `fake_templates_dir({"bad": TEMPLATE_TOML_MALFORMED_PARSE})` (or a body with `thinking="huge"` constructed programmatically). `spawn_worker_impl(template="bad")` MUST return `err_name="ErrTemplateMalformed"` — NOT remapped to `ErrThinkingInvalid` or any other Epic 1 per-call error. The loader's malformed-at-load-time surface wins.

Every test in this file MUST use:
- `tests.helpers.fake_templates_dir(...)` (no real `~/.claude-spawn/templates/`).
- `tests.helpers.fake_claude_status(...)`, `fake_worker_record(..., cwd=...)`, `fake_workers_response(...)` for any Claude Status interaction (the Epic 2 readiness wait runs on every successful spawn, so happy-path tests need a workers-response triple).
- `TEMPLATE_TOML_*` constants from `tests/sample_payloads.py` OR programmatic TOML construction via `tomli_w` — NEVER hardcoded TOML string literals.

## Acceptance Criteria

- All 12 cases above are present and individually identified by name.
- Case 1 (bare-spawn-no-read) uses `mock.assert_not_called()` on the loader's `_read_template_file` seam.
- Case 6 (permissions merge) asserts BOTH `allow` replaced AND `deny` preserved in the same test.
- Case 8 (no caching) verifies a file mutation is observed by the second call.
- Case 10 (cwd-not-satisfied) explicitly checks `err_name=ErrCwdMissing`.
- Case 12 (malformed surface) explicitly checks `err_name=ErrTemplateMalformed` and NOT a remapped per-call error name.
- No test hardcodes a TOML string literal; all TOML inputs come from constants or `tomli_w` construction.
- No test touches the real `~/.claude-spawn/templates/` directory.
- `pytest tests/test_engine_spawn_worker_options.py::TestTemplateIntegration` is green.
