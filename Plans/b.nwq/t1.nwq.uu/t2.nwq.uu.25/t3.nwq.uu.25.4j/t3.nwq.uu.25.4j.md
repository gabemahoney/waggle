## Context

Epic 1 added the `template` parameter to `spawn_worker_impl` but left it ignored at impl level. Task 2 created the loader module. This subtask wires the two together: `spawn_worker(template=<name>)` reads the named template via the Task 2 loader, runs the SR-2.1 resolution chain (per-call → template → default), applies the SR-2.2 map-merge semantics, and propagates the loader's `ErrTemplateNotFound` / `ErrTemplateMalformed` results back to the caller. SR-6.6 forbids any filesystem read when `template` is not supplied and forbids in-process caching.

## What Needs to Change

In `src/claude_spawn/spawn.py`:

- **Loader call:** when `template` is non-`None` after argument parsing, call the Task 2 loader's single-template lookup function. Propagate `ErrTemplateNotFound` or `ErrTemplateMalformed` as the return value of `spawn_worker_impl` (do NOT raise — surface as the operation-failed dict shape consistent with other errors).
- **No-template fast path:** a bare `spawn_worker(cwd=…)` with no `template` argument MUST NOT read the templates directory. Verifiable by patching `_read_template_file` in the loader module and asserting `not_called`.
- **No caching (SR-6.6):** "Templates are not cached between calls within a single MCP server process." Each `spawn_worker(template=<name>)` invocation re-reads the file. Do NOT add an in-process cache.
- **Option resolution chain (SR-2.1):** for every SR-1.1 option, the effective value is the first non-`None` source in the order:
  1. Per-call argument (any explicit non-sentinel value passed to `spawn_worker`).
  2. Template field (only consulted when the caller supplied `template=<name>`).
  3. The option's documented default per SR-1.3.
  The required-`cwd` rule from Epic 1 (`ErrCwdMissing`) is satisfied if the template supplies `cwd`. If neither per-call nor template supplies `cwd`, the spawn fails with `ErrCwdMissing` as before.
- **Map-valued merge (SR-2.2 — verbatim):** "Map fields (`extra_env`, `claude_status_labels`, `permissions`) merge by union; per-call entries override template entries on colliding keys." Shallow merge: a top-level key present in both sources resolves to the per-call value as a whole. For `permissions`, the merge keys are `allow`/`deny`/`ask` at the top level of the permissions sub-map. Examples:
  - Per-call `permissions={"allow":["A"]}` against template `permissions={"allow":["B"],"deny":["D"]}` resolves to `{"allow":["A"],"deny":["D"]}` (allow replaced wholesale; deny preserved from template).
  - Per-call `extra_env={"FOO":"x"}` against template `extra_env={"FOO":"y","BAR":"z"}` resolves to `{"FOO":"x","BAR":"z"}`.
- **Scalar and list resolution (SR-2.3):** `claude_args` and every scalar option — per-call value replaces template value wholesale when both are present.
- **`template`-in-template:** SR-2.4 says a template body cannot contain `template`. This is enforced at LOAD time in the loader (Task 2 → `ErrTemplateMalformed`). The `spawn_worker_impl` codepath does not need an additional guard.

No changes to the loader module (`templates.py`) in this subtask.

## Key Files

- `src/claude_spawn/spawn.py` — `spawn_worker_impl` loader call and merge logic.
- `src/claude_spawn/templates.py` — read-only; used via its public lookup function.

## Acceptance Criteria

- `spawn_worker_impl(cwd="/x")` with no `template` argument never calls the loader's `_read_template_file` seam.
- `spawn_worker_impl(template="missing", cwd="/x")` returns `{"ok": False, "err_name": "ErrTemplateNotFound", ...}` with description naming both the template name and the templates directory.
- `spawn_worker_impl(template="malformed")` returns `{"ok": False, "err_name": "ErrTemplateMalformed", ...}` (per the loader's surface — not remapped).
- Template supplies `cwd`; per-call omits `cwd` → spawn succeeds using template's `cwd`.
- Template does NOT supply `cwd` and per-call omits `cwd` → returns `ErrCwdMissing`.
- Per-call scalar overrides template scalar; template scalar is used when per-call is absent.
- Per-call list (`claude_args`) replaces template list wholesale.
- Map union for `extra_env` / `claude_status_labels`: per-call wins on colliding top-level keys; independent keys from either source preserved.
- `permissions` map-key merge: per-call's `allow`/`deny`/`ask` replaces template's matching key wholesale; independent keys from either side preserved.
- Two sequential `spawn_worker(template="x")` calls each invoke the loader's `_read_template_file` seam exactly once (no caching).
- `spawn.py` does NOT import `tomllib` or `tomli_w` directly; all template I/O stays in `templates.py`.
