# Task 3 — Wire `template=<name>` resolution into `spawn_worker_impl` + merge semantics + integration tests

## Context

Epic 1 introduced the `template` parameter on `spawn_worker_impl` but left it ignored. Task 2 created the templates loader. This Task connects them: `spawn_worker(template=<name>)` now reads the named template from disk, runs the SR-2 resolution chain (per-call → template → default), and applies the SR-2.2 map-merge semantics. SR-6.6 forbids any filesystem read when `template` is not supplied; SR-6.6 also forbids in-process caching, so each call re-reads the file.

## What Needs to Change

- `src/claude_spawn/spawn.py` — `spawn_worker_impl` calls the Task 2 loader when `template=<name>` is supplied; routes the loader's `ErrTemplateNotFound` / `ErrTemplateMalformed` results back to the caller. Implements the SR-2.1 resolution chain (per-call → template → default), SR-2.2 map-merge (`extra_env`, `claude_status_labels`, `permissions` union with per-call wins on top-level keys; `permissions` merge keys are `allow`/`deny`/`ask`), SR-2.3 scalar/list wholesale replacement (`claude_args` and every scalar option). No in-process caching. Required-`cwd` rule satisfied by template.
- `tests/test_engine_spawn_worker_options.py` — extended with template integration cases: bare-spawn-no-read, happy path, scalar override, map merge (extra_env / claude_status_labels), permissions merge (replacement of same-key + preservation of independent-key in the same test), list replacement, no caching (mutate file between two calls), required-`cwd` satisfied by template, required-`cwd` NOT satisfied (template with no cwd + no per-call cwd → `ErrCwdMissing`), missing template → `ErrTemplateNotFound`, malformed template → explicit assertion that `err_name=ErrTemplateMalformed` (not remapped to a per-call error name).

## Why

Realizes the operator-visible template capability: `spawn_worker(template="orchestrator")` becomes a one-liner that replaces a verbose options dict. Closes Epic 3's user-facing AC.

## Success Criteria

- Bare `spawn_worker(cwd="/tmp")` (no `template`) makes no call to the loader seam.
- `spawn_worker(template="x", ...)` reads `x.toml`, applies resolution + merge, and the resulting tmux invocation reflects the merged options.
- Two sequential `spawn_worker(template="x")` calls each re-read the file (no caching).
- `spawn_worker(template="missing")` returns `ErrTemplateNotFound`; `spawn_worker(template="malformed")` returns `ErrTemplateMalformed`.
- A template supplying `cwd` satisfies the Epic 1 required-`cwd` rule; a template with no `cwd` and a caller with no `cwd` returns `ErrCwdMissing`.
- Map merge: per-call wins on colliding top-level keys; independent keys from either source preserved. `permissions.allow=[A]` (per-call) vs `permissions.allow=[B], permissions.deny=[D]` (template) → resolved `{allow:[A], deny:[D]}`.
- Full `pytest` is green.
