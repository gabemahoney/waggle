# Task 2 — Create templates loader module + test helpers + sample payloads + load-tests

## Context

SR-6.5 introduces a new module `src/claude_spawn/templates.py` that owns all template I/O and schema validation. The loader exposes a single-template lookup function (consumed by `spawn_worker_impl` in Task 3) and an enumerate-all function (consumed by `list_templates_impl` in Epic 4). All filesystem reads go through a single patchable seam so tests never touch the real `~/.claude-spawn/templates/`. Per SR-11.4 the new test helper `fake_templates_dir(mapping)` lives in `tests/helpers.py`. Per SR-11.5 the five `TEMPLATE_TOML_*` constants live in `tests/sample_payloads.py`. The new test file `tests/test_engine_templates_load.py` covers every SR-6.4 load-time validation path. At this Task's boundary the loader is dead code — Task 3 wires it into `spawn_worker_impl`.

## What Needs to Change

- New `src/claude_spawn/templates.py` with: `_templates_dir()` and `_read_template_file(path)` patchable seams; resolve-one-by-name function; enumerate-all function with malformed-skip channel; SR-6.4 schema validation producing `ErrTemplateMalformed` / `ErrTemplateNotFound` results. Import-time performs no filesystem reads.
- `tests/helpers.py` — new `fake_templates_dir(mapping)` context manager (SR-11.4).
- `tests/sample_payloads.py` — five new constants (SR-11.5): `TEMPLATE_TOML_MINIMAL`, `TEMPLATE_TOML_FULL`, `TEMPLATE_TOML_MALFORMED_UNKNOWN_KEY`, `TEMPLATE_TOML_MALFORMED_PARSE`, `TEMPLATE_TOML_RECURSIVE` (body containing a top-level `template` key per SR-2.4).
- `tests/test_engine_templates_load.py` — new test file covering missing-dir → empty; minimal + full valid loads; every SR-6.4 malformed shape; `_read_template_file` patchable; import-time no filesystem reads.

## Why

Establishes the entire template loader subsystem in one commit while leaving the codebase green. Subsequent Epics (3 Task 3, Epic 4, Epic 5) build on the seams established here.

## Success Criteria

- `import claude_spawn.templates` succeeds with no filesystem reads at import time.
- The single-template lookup returns the parsed option map for valid files and `ErrTemplateNotFound` / `ErrTemplateMalformed` for the documented failure modes.
- The enumerate-all function yields one entry per file, with malformed files surfaced via a per-file `ErrTemplateMalformed` result and valid files surfaced with their parsed option maps.
- All five `TEMPLATE_TOML_*` constants exist in `tests/sample_payloads.py`.
- `fake_templates_dir({...})` creates a temp dir, writes the files, patches the templates-directory seam, and cleans up on exit.
- `pytest tests/test_engine_templates_load.py` is green.
- Full test suite remains green.

## Dependencies

- Task 1 (Python ^3.11 for `tomllib`; `tomli-w` available).
