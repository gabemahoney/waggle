## Context

SR-11.4 mandates a new test helper `fake_templates_dir` so no test in this Epic touches the real `~/.claude-spawn/templates/`. SR-11.5 mandates five new TOML body constants in `tests/sample_payloads.py` so no test in this Epic hardcodes a TOML string literal. Both belong in the same subtask because the constants are the natural inputs to the helper.

## What Needs to Change

In `tests/helpers.py`:
- Add a `fake_templates_dir(mapping)` context manager. Behavior:
  - Takes a mapping of `{template_name: toml_body_string}`.
  - Creates a temp directory (e.g. via `tempfile.TemporaryDirectory`).
  - Writes each entry as `<temp_dir>/<template_name>.toml` containing the body string.
  - Patches the templates-directory resolution seam in `src/claude_spawn/templates.py` (the `_templates_dir()` function added by the loader subtask) so it returns the temp directory path for the duration of the context.
  - Yields the temp directory path.
  - On exit, restores the patch and cleans up the temp directory.

In `tests/sample_payloads.py`, add the following constants (verbatim names from SR-11.5):
- `TEMPLATE_TOML_MINIMAL` — smallest valid template body with only `cwd` set.
- `TEMPLATE_TOML_FULL` — valid template body exercising every supported SR-1.1 option (excluding `template`): `cwd`, `model`, `thinking`, `tmux_session_name`, `instance_id`, `claude_home`, `claude_settings`, `extra_env` (table), `claude_status_labels` (table), `claude_args` (array), `permissions` (table with `allow`/`deny`/`ask` arrays).
- `TEMPLATE_TOML_MALFORMED_UNKNOWN_KEY` — valid TOML syntactically, but the top-level table contains a key NOT in the SR-1.1 schema (e.g. `not_a_real_option = "x"`). Used to exercise SR-6.4 bullet 2.
- `TEMPLATE_TOML_MALFORMED_PARSE` — TOML body that fails parsing entirely (syntactically invalid).
- `TEMPLATE_TOML_RECURSIVE` — valid TOML syntactically, but the top-level table contains a `template` key (e.g. `template = "other"`). Per SR-2.4 + SR-11.5 + SR-6.4 bullet 3, this is rejected at load time as malformed. (Note: "recursive" here means "templates cannot reference templates", NOT a circular cross-template reference.)

## Key Files

- `tests/helpers.py` — new `fake_templates_dir` context manager.
- `tests/sample_payloads.py` — five new constants.

## Acceptance Criteria

- `fake_templates_dir` is importable from `tests.helpers`.
- Entering the context manager with `{"orch": TEMPLATE_TOML_MINIMAL}` writes `<temp_dir>/orch.toml` containing the body and patches `_templates_dir()` to return `<temp_dir>`.
- Exiting the context manager restores the original `_templates_dir()` and removes the temp directory.
- All five `TEMPLATE_TOML_*` constants are defined and non-empty.
- `TEMPLATE_TOML_RECURSIVE` contains a top-level `template = "..."` key (matches SR-2.4 + SR-6.4 bullet 3 — NOT a cross-template circular reference).
- No TOML string literals appear in `tests/helpers.py` or `tests/sample_payloads.py` outside the five named constants.
