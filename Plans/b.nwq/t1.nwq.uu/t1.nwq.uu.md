# Epic 3 — TOML template loader and `template=<name>` resolution

Add the on-disk template layer at `~/.claude-spawn/templates/` and wire `spawn_worker(template=<name>)` to read it. Finalize the option-resolution chain (per-call → template → default) and the merge semantics.

## Scope

- **Python floor + dependency:** `pyproject.toml` Python floor lifted to `^3.11` so `tomllib` is available without backport. Add `tomli-w` as a runtime dependency for writing TOML (used by Epic 5, but the dep is added here as part of finalizing the template-file story).
- **Storage layout (SR-6.1):** templates live at `~/.claude-spawn/templates/<name>.toml`; name is the filename stem. Directory may be absent — read paths must treat absence as "zero templates", not an error.
- **TOML schema (SR-6.2, 6.3):** top-level table whose keys are a subset of the eleven option names from SR-1.1 excluding `template`. Map-valued options as TOML tables/inline tables, list-valued options as TOML arrays, scalars as strings.
- **Load-time validation (SR-6.4):** reject malformed files (parse failure, unknown key, `template` key, invalid `thinking` level, type mismatches). Reject produces a per-file `ErrTemplateMalformed` with `err_description` naming the path and parse problem.
- **Loader module (SR-6.5):** new `src/claude_spawn/templates.py` alongside `spawn.py` and `claude_status.py`. Exposes at least:
  - A "resolve one by name" function used by `spawn_worker_impl` — returns the parsed option map or raises a structured `ErrTemplateNotFound`/`ErrTemplateMalformed`.
  - An "enumerate all" function used (in Epic 4) by `list_templates_impl` — yields `(name, path, parsed_options | error)` tuples.
  - A single patchable filesystem-read seam (e.g. `_read_template_file(path)`) and a single patchable templates-directory seam (e.g. `_templates_dir()`) so tests never touch the real `~/.claude-spawn/templates/`.
  - Importing the module performs no filesystem reads (matches the convention in `src/claude_spawn/claude_status.py`).
- **Lookup semantics (SR-6.6):**
  - A bare `spawn_worker(cwd=…)` with no `template` argument MUST NOT read the templates directory.
  - No in-process caching: each `spawn_worker(template=<name>)` call re-reads the file.
  - Missing template returns `ErrTemplateNotFound` with description naming the template name and the directory searched.
- **Option resolution and merge (SR-2.1–2.4):**
  - Per-call → template → default order on every option.
  - Map-valued options (`extra_env`, `claude_status_labels`, `permissions`) union across per-call and template, with per-call winning on colliding top-level keys. Inside `permissions`, the merge keys are `allow`/`deny`/`ask` (no deep list-union).
  - List `claude_args` and every scalar option: per-call replaces template wholesale.
  - `template` field inside a template body is forbidden (SR-2.4) and surfaces as `ErrTemplateMalformed` at load time.
  - The required-`cwd` rule (Epic 1) is satisfied if the template supplies `cwd`.
- **Helpers (SR-11.4, 11.5):**
  - New `tests/helpers.py::fake_templates_dir` context manager: takes a `{name: toml_body}` mapping, creates a temp directory, writes the files, patches the templates-directory seam to point at it, cleans up on exit.
  - New constants in `tests/sample_payloads.py`: `TEMPLATE_TOML_MINIMAL`, `TEMPLATE_TOML_FULL`, `TEMPLATE_TOML_MALFORMED_UNKNOWN_KEY`, `TEMPLATE_TOML_MALFORMED_PARSE`, `TEMPLATE_TOML_RECURSIVE`. No test in this Epic (or later) hardcodes a TOML string literal — uses these constants or programmatic construction via `tomli_w`.
- **Tests:** new `tests/test_engine_templates_load.py` covering missing directory, valid template load (minimal and full), every SR-6.4 malformed shape, and the per-call/template/default merge rules including map-valued union with per-call wins. Plus targeted updates in `tests/test_engine_spawn_worker_options.py` (added by Epic 1) for `template=<name>` integration cases: spawn from template, spawn with per-call overrides, spawn against a malformed template, spawn against a missing template.
- **Documentation:** README documents the TOML template file format with at least one worked example, the merge rules (scalar/list per-call replaces wholesale; map union with per-call winning), and the explicit-only template lookup (no auto-loaded default). New `docs/architecture/spawn_options_and_templates.md` covers the option-resolution flow and the loader, in the same style as `docs/architecture/spawn_worker.md`.

## Out of scope for this Epic

- `list_templates` MCP tool (Epic 4).
- `write_template` MCP tool / CLI subcommand (Epics 5–6).

## Acceptance Criteria

1. **Bare spawn reads no template.** Call `spawn_worker(cwd="/tmp")` against `fake_templates_dir({"orch": TEMPLATE_TOML_FULL})`. Assert: the templates seam's `_read_template_file` was never called. (Verify via mock.assert_not_called.)
2. **Spawn from template — happy path.** Drop a template `orch.toml` (via `fake_templates_dir`) with `cwd="/tmp"`, `model="opus"`, `thinking="xhigh"`, `permissions.allow=["Bash"]`. Call `spawn_worker(template="orch")` (no `cwd` per-call). Assert: spawn succeeds; the captured tmux invocation reflects `cwd=/tmp`, `--model opus`, `--effort xhigh`, and a `--settings` overlay containing `permissions.allow=["Bash"]`.
3. **Merge semantics — per-call scalar wins.** Same template plus `spawn_worker(template="orch", model="sonnet")`. Assert: captured invocation has `--model sonnet`.
4. **Merge semantics — map union with per-call wins on top-level key.** Template `permissions.allow=["A"]`, `permissions.deny=["D"]`. Per-call `permissions={"allow":["B"]}`. Assert: resolved permissions are `{"allow":["B"], "deny":["D"]}` (allow replaced wholesale at top-level key, deny preserved from template).
5. **Merge semantics — map union with `extra_env` and `claude_status_labels`.** Template `extra_env={"FOO":"1","BAR":"2"}`. Per-call `extra_env={"BAR":"3","BAZ":"4"}`. Assert: tmux invocation has `FOO=1`, `BAR=3`, `BAZ=4`. Same shape verified for `claude_status_labels`.
6. **Merge semantics — list replaces wholesale.** Template `claude_args=["--a","--b"]`. Per-call `claude_args=["--c"]`. Assert: captured `send-keys` shows only `--c` (no `--a`/`--b`).
7. **Error paths.**
   - `spawn_worker(template="nope")` against an otherwise-empty `fake_templates_dir({})` → `ErrTemplateNotFound`; description names both `"nope"` and the directory.
   - Template body == `TEMPLATE_TOML_MALFORMED_PARSE` → `ErrTemplateMalformed`.
   - Template body == `TEMPLATE_TOML_MALFORMED_UNKNOWN_KEY` → `ErrTemplateMalformed` listing the unknown key.
   - Template body == `TEMPLATE_TOML_RECURSIVE` (top-level `template` key) → `ErrTemplateMalformed`.
   - Template body sets `thinking="huge"` → `ErrTemplateMalformed` (or `ErrThinkingInvalid` — pick the SRD's mapping; the loader rejects at load time so it's `ErrTemplateMalformed`) listing the four valid values.
8. **No caching.** Within a single test, mutate the on-disk template file (via `fake_templates_dir`) between two `spawn_worker(template=<name>)` calls; the second call observes the new content.
9. **Required-`cwd` satisfied by template.** `spawn_worker(template=<name>)` where the template supplies `cwd` and the caller does not — succeeds. Same call against a template with no `cwd` and no per-call `cwd` → `ErrCwdMissing`.
10. **Python floor + dependency check.** `pyproject.toml` shows `python = "^3.11"` and `tomli-w` as a runtime dep; `poetry install` (or equivalent) succeeds; `python -c "import tomllib, tomli_w"` works inside the venv.
11. **Tests + docs green.** `pytest tests/test_engine_templates_load.py` plus the integration tests in `tests/test_engine_spawn_worker_options.py` pass. README has a worked template example and the merge rules section. `docs/architecture/spawn_options_and_templates.md` exists and covers the loader + resolution chain.
