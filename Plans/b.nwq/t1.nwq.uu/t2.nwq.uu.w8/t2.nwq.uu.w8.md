# Task 4 — Documentation: README templates section + new arch doc

## Context

Per SR-12.1 the README must document the TOML template file format with a worked example, the SR-2 merge rules, and the explicit-only lookup. Per SR-12.2 a new arch doc `docs/architecture/spawn_options_and_templates.md` covers the option-resolution flow and the loader. The existing `docs/architecture/spawn_worker.md` (already updated by Epic 1 + Epic 2) receives only a small "See also" cross-reference in this Epic.

## What Needs to Change

- `README.md` — new "Templates" section (after the `spawn_worker` doc block) containing the storage layout, the TOML schema, a worked `orchestrator.toml` example, the three merge rules (scalar/list wholesale; map union; `permissions` sub-key wholesale-replacement-with-independent-key-preservation), the explicit-only lookup statement, and one-line mentions of `ErrTemplateNotFound` and `ErrTemplateMalformed`.
- `docs/architecture/spawn_options_and_templates.md` — NEW arch doc in the same style as `docs/architecture/spawn_worker.md`, covering: Overview; Storage layout; Option-resolution order (per-call → template → default); Merge semantics (with concrete examples mirroring the README); Loader module (`templates.py`) with the two public functions and the patchable seams; Load-time validation (the SR-6.4 conditions → `ErrTemplateMalformed`); Lookup semantics (no caching, explicit-only); Errors table (`ErrTemplateNotFound`, `ErrTemplateMalformed`); Mermaid sequence diagram showing the loader call path.
- `docs/architecture/spawn_worker.md` — add a one-line "See also" cross-reference to the new arch doc. No other changes (Epic 1+2 already updated this file).

Out of scope: `list_templates` content (Epic 4 extends the new arch doc), `write_template` (Epic 5), CLI subcommand (Epic 6), readiness wait (Epic 2 already documented).

## Why

Closes the SR-12 doc bar for Epic 3 — an operator can read the README, author a working template by hand, and spawn from it without reading source code.

## Success Criteria

- README has a "Templates" section with a copy-paste-ready `orchestrator.toml` example, the three merge rules, the explicit-only lookup statement, and mentions of the two new error names.
- `docs/architecture/spawn_options_and_templates.md` exists with the documented sections and a syntactically-valid Mermaid sequence diagram.
- `docs/architecture/spawn_worker.md` contains a "See also" link to the new arch doc.
- An operator can author and spawn from a template without consulting source code.
