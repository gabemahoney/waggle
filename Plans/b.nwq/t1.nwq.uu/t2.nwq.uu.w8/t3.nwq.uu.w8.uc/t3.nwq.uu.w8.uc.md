## Context

SR-12.2 mandates a new architecture doc `docs/architecture/spawn_options_and_templates.md` covering the option-resolution flow, the templates loader (`templates.py`), the storage layout, and the load-time validation rules. The doc style matches the existing `docs/architecture/spawn_worker.md` and `docs/architecture/send_input.md`.

## What Needs to Change

Create `docs/architecture/spawn_options_and_templates.md` with these sections (in this order):

1. **Overview** — two or three sentences: what `templates.py` does; where templates live; that the loader is invoked only when `template=<name>` is supplied.

2. **Storage layout** — the canonical path pattern `~/.claude-spawn/templates/<name>.toml`; name == filename stem; absent directory == zero templates (not an error); no in-process caching.

3. **Option-resolution order** — a numbered three-step list (per-call → template → default) with a small table mapping each SR-1.1 option to its SR-1.3 fallback category. Note that `cwd` can be satisfied by the template.

4. **Merge semantics** — mirror the README's three-rule breakdown (scalar/list wholesale; map union; `permissions` sub-key) with concrete examples. Add an implementation note: the map union is shallow at the top level of each map; `permissions` is merged key-by-key at the `allow`/`deny`/`ask` level using the same shallow-union rule.

5. **Loader module (`src/claude_spawn/templates.py`)** — list the two public functions (resolve-one-by-name, enumerate-all), the two patchable seams (`_templates_dir`, `_read_template_file`), and the import-inertia property.

6. **Load-time validation** — the SR-6.4 rejection conditions as a bullet list, each mapped to `ErrTemplateMalformed`; `ErrTemplateNotFound` as a separate bullet (file simply absent).

7. **Lookup semantics** — two bullets: (a) bare `spawn_worker(cwd=…)` does not invoke the loader; (b) no in-process caching — hand-edits take effect on the next spawn.

8. **Errors** — table with columns `err_name | Surface | Condition`. Cover `ErrTemplateNotFound` and `ErrTemplateMalformed`. Do NOT include errors from `list_templates` or `write_template` (sibling Epic scope; Epic 4 will extend this doc with those).

9. **Sequence diagram** — Mermaid diagram showing the `spawn_worker_impl` call path when `template=<name>` is supplied: `spawn_worker_impl` → `templates.resolve_by_name` → filesystem read → merge → proceed to validation + tmux + readiness wait. Include the `ErrTemplateNotFound` / `ErrTemplateMalformed` alt branches.

Out of scope: `list_templates`, `write_template`, the CLI subcommand, the readiness wait (Epic 2 owns its arch doc updates).
