## Context

SR-12.1 requires a new "Templates" section in the README documenting the TOML file format, a worked example, the merge rules, and the explicit-only lookup. Epic 1 already shipped the README's 12-option `spawn_worker` table and the Claude Code settings-stack subsection; Epic 2 already shipped the readiness-blocking subsection. This subtask adds the templates content and a short "Option resolution order" paragraph.

## What Needs to Change

In `README.md`, add a "Templates" section (placement: after the `spawn_worker` tool section, or wherever makes structural sense relative to the existing layout). Contents:

1. **Option resolution order** — a short paragraph stating: per-call argument → template field (when `template=<name>` is supplied) → SR-1.3 hardcoded/inherited default. State explicitly that a bare `spawn_worker(cwd=…)` with no `template=` argument reads no template file (the explicit-only lookup).

2. **Storage layout** — one sentence: templates live at `~/.claude-spawn/templates/<name>.toml`; the template name is the filename stem.

3. **TOML schema** — describe the top-level table: keys are the eleven SR-1.1 option names minus `template`; map options as TOML tables; list options as TOML arrays; scalar options as TOML strings.

4. **Worked example** — a complete, copy-paste-ready `~/.claude-spawn/templates/orchestrator.toml` example exercising all three TOML value types (scalar string, array, table). For example:

   ```toml
   cwd = "/home/horde/projects/waggle-project/waggle-main"
   model = "sonnet"
   thinking = "high"
   claude_args = ["--verbose"]

   [extra_env]
   LOG_LEVEL = "debug"

   [permissions]
   allow = ["Bash(git *)"]
   deny = ["Bash(rm -rf *)"]
   ```

   The example must be verbatim-usable (no placeholders requiring substitution).

5. **Merge rules** — three named rules with concrete examples:
   - **Scalar and list options** — per-call replaces template wholesale (e.g., per-call `model="haiku"` overrides template `model="opus"`; per-call `claude_args=["--c"]` discards template `claude_args=["--a","--b"]` entirely).
   - **Map options (`extra_env`, `claude_status_labels`)** — top-level keys union; per-call wins on colliding keys (e.g., template `{"FOO":"1","BAR":"2"}` + per-call `{"BAR":"3","BAZ":"4"}` → `{"FOO":"1","BAR":"3","BAZ":"4"}`).
   - **`permissions` map** — per-call's `allow` / `deny` / `ask` key replaces the template's matching key wholesale (NOT list-unioned); independent keys from either side are preserved (e.g., template `{"allow":["A"],"deny":["D"]}` + per-call `{"allow":["B"]}` → `{"allow":["B"],"deny":["D"]}`).

6. **Errors** — one sentence each on `ErrTemplateNotFound` (missing template file) and `ErrTemplateMalformed` (TOML parse failure or SR-6.4 schema violation), pointing to the README's broader error catalogue if one exists.

Out of scope: do NOT add `list_templates` / `write_template` / CLI subcommand content. Do NOT redo the 12-option table or the Claude Code settings stack subsection (Epic 1 owns those).
