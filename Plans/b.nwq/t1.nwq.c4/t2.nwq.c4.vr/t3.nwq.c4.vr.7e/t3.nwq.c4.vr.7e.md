## Context

Epic 4's engineer subtask registers a new `list_templates` MCP tool. This subtask creates the dedicated test file covering the SR-7.1 contract: empty dir → operation-success with empty lists; valid templates surfaced under `templates[]` with the documented fields; malformed templates surfaced under `skipped[]` with `ErrTemplateMalformed`; unexpected exceptions mapped to `ErrUnexpected` via the existing FastMCP wrapper.

## What Needs to Change

Create `tests/test_mcp_stdio_list_templates.py`. Use Epic 3's `fake_templates_dir` context manager and the `TEMPLATE_TOML_*` constants — no hardcoded TOML string literals. Cover:

- **Empty directory:** `fake_templates_dir({})` (empty mapping) AND a configuration where the templates dir does not exist at all. Both calls return `{"templates": [], "skipped": []}` with NO `ok: False`. Confirms SR-7.1's "missing directory = operation-success" semantic.
- **Valid templates:** `fake_templates_dir({"orch": TEMPLATE_TOML_FULL, "mini": TEMPLATE_TOML_MINIMAL})`. Assert `templates[]` has exactly two entries (order-insensitive); each entry has `name` ∈ {`orch`, `mini`}, `path` ending in `/templates/<name>.toml`, and `options` reflecting the parsed file content. Assert `skipped[]` is empty.
- **Malformed channel:** `fake_templates_dir({"bad": TEMPLATE_TOML_MALFORMED_PARSE, "good": TEMPLATE_TOML_MINIMAL})`. Assert `templates[]` has one entry for `good`; `skipped[]` has one entry for `bad` with `err_name="ErrTemplateMalformed"` and a non-empty `err_description`.
- **Error wrapping:** patch the impl function (whichever module the engineer placed it in) to raise an unexpected exception (e.g. `RuntimeError("boom")`). Call `list_templates()`. Assert `result["ok"] is False` and `result["err_name"] == "ErrUnexpected"`.
