## Context

Epic 3 delivers `src/claude_spawn/templates.py` with an enumerate-all function yielding `(name, path, parsed_options_or_error)` tuples; malformed files come back in the error half as `ErrTemplateMalformed`. The two patchable seams are `_templates_dir()` and `_read_template_file(path)`. SR-7.1 defines the new `list_templates` MCP tool's contract: take no parameters; return `{"templates": [...], "skipped": [...]}`; missing directory is operation-success with empty lists.

## What Needs to Change

In `src/claude_spawn/mcp_stdio.py`:
- Register a new FastMCP tool `list_templates` taking no parameters and returning a dict. Follow the existing tool registration pattern (the `list_spawned_workers` tool is the closest analog). Route through the established try/except wrapper that maps unexpected exceptions to `err_name=ErrUnexpected` per the existing `_err` helper pattern.

Add the impl function as a thin wrapper around the Epic 3 loader's enumerate-all function. The wrapper's home (alongside `list_spawned_workers_impl` in `src/claude_spawn/spawn.py`, or alongside the loader in `src/claude_spawn/templates.py`) is an implementer choice — both are reasonable. The wrapper:
- Iterates the enumerate-all output.
- Routes valid entries into `templates[]` with at minimum `name` (filename stem), `path` (absolute on-disk path), `options` (parsed option map keyed by SR-1.1 names excluding `template`).
- Routes error entries into `skipped[]` with at minimum `path`, `err_name`, `err_description`.
- Returns `{"templates": [...], "skipped": [...]}`. Missing templates directory yields empty lists; this is operation-success, not an error (the Epic 3 loader already treats directory absence as zero-template success).

The existing `list_spawned_workers` tool MUST remain unchanged: not renamed, merged, or shadowed (SR-7.2). The two tools query different sources (Claude Status vs. the templates directory) and return different shapes.

No new filesystem code is introduced in this subtask. All disk interaction stays inside the Epic 3 loader module.

## Key Files

- `src/claude_spawn/mcp_stdio.py` — new tool registration.
- `src/claude_spawn/spawn.py` OR `src/claude_spawn/templates.py` — new `list_templates_impl` wrapper (implementer choice).
- `src/claude_spawn/templates.py` — read-only consumption of the enumerate-all entry point.

## Acceptance Criteria

- `list_templates()` (called via the FastMCP test surface) with an empty templates dir returns `{"templates": [], "skipped": []}` and no `ok: False`.
- With two valid templates on disk, the returned `templates[]` has exactly two entries, each with `name`/`path`/`options` keys; `skipped[]` is empty.
- With one valid + one malformed template on disk, `templates[]` has one entry for the valid file and `skipped[]` has one entry for the malformed file with `err_name="ErrTemplateMalformed"` and a non-empty `err_description`.
- An unexpected exception in the impl is mapped to `err_name=ErrUnexpected` by the FastMCP wrapper.
- `list_spawned_workers` is unchanged in name, signature, and behavior.
- `src/claude_spawn/mcp_stdio.py` has 7 registered FastMCP tools after this subtask.
