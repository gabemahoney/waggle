# Task 1 — Implement `list_templates` MCP tool

## Context

SR-7.1 introduces a new MCP tool `list_templates` that surfaces the Epic 3 loader's enumerate-all output to operators. The tool takes no parameters and returns a dict with `templates[]` (valid files with `name`/`path`/`options`) and `skipped[]` (malformed files with `path`/`err_name`/`err_description`). Missing templates directory is operation-success with empty lists — never an error. The existing `list_spawned_workers` MCP tool (SR-7.2) is unchanged.

## What Needs to Change

- Register a new `list_templates` MCP tool in `src/claude_spawn/mcp_stdio.py`. Routes through the existing FastMCP try/except wrapper so unexpected exceptions surface as `ErrUnexpected`.
- Add the impl function as a thin wrapper around the Epic 3 loader's enumerate-all function. The wrapper's home (in `spawn.py` next to `list_spawned_workers_impl`, or in `templates.py` next to the loader) is an implementer choice — both fit the codebase.
- Update `tests/test_mcp_stdio_server.py` tool-count assertion from 6 → 7 and extend error-wrapping coverage to `list_templates`.
- Add new `tests/test_mcp_stdio_list_templates.py` covering empty-dir, valid templates, malformed-template `skipped[]` channel, and FastMCP error-wrapping.

## Why

Lets operators (and LLM orchestrators) discover what templates exist and inspect their resolved option maps without reading source code or shelling into `~/.claude-spawn/templates/`. Surfaces malformed-file diagnostics on a dedicated channel so a single bad file doesn't hide the rest.

## Success Criteria

- `list_templates()` with no templates directory returns `{"templates": [], "skipped": []}` with no `ok: False`.
- With valid templates, `templates[]` has one entry per file with `name`, `path`, `options`; `skipped[]` is empty.
- With malformed files, valid entries appear under `templates[]` and malformed under `skipped[]` with `err_name="ErrTemplateMalformed"`.
- An unexpected exception in the impl surfaces as `ErrUnexpected` via the FastMCP wrapper.
- `list_spawned_workers` is unchanged (not renamed, merged, or shadowed).
- Tool-count test in `tests/test_mcp_stdio_server.py` asserts 7.
- Full `pytest` is green.

## Dependencies

- Epic 3 (`t1.nwq.uu`) — provides the loader module and the `fake_templates_dir` / `TEMPLATE_TOML_*` test surface.
