# Epic 4 — `list_templates` MCP tool

Register a new MCP tool that enumerates the operator's saved templates and surfaces malformed files in a separate `skipped[]` channel.

## Scope

- **New MCP tool (SR-7.1):** `list_templates` registered in `src/claude_spawn/mcp_stdio.py`. Takes no parameters. Routes through the existing FastMCP try/except wrapper that maps unexpected exceptions to `ErrUnexpected`.
- **Return shape:** dict with two keys:
  - `templates[]` — one entry per valid file. Each entry has at least `name` (filename stem), `path` (absolute on-disk path), `options` (resolved option map keyed by SR-1.1 names minus `template`).
  - `skipped[]` — one entry per malformed file. Each entry has at least `path`, `err_name` (e.g. `ErrTemplateMalformed`), `err_description` (parser detail).
- **Missing directory:** operation-success with `templates=[]` and `skipped=[]`. NOT an error.
- **Implementation:** thin wrapper around the Epic 3 loader's enumerate-all function. No new filesystem code.
- **`list_spawned_workers` unaffected (SR-7.2):** the existing tool is not renamed, merged, or shadowed.
- **Tests:**
  - New `tests/test_mcp_stdio_list_templates.py` covering: empty dir, one valid template, one valid + one malformed (assert `templates[]` has the valid one with full `options` map, `skipped[]` has the malformed one with `err_name=ErrTemplateMalformed`).
  - Update `tests/test_mcp_stdio_server.py`: tool-count assertion rises from 6 → 7, error-wrapping coverage extends to the new tool.
- **Documentation:** README documents the tool's parameters (none), the result shape including `skipped[]`, and a worked example output. The architecture doc from Epic 3 (`docs/architecture/spawn_options_and_templates.md`) gets a short section on `list_templates` as the discovery surface.

## Out of scope for this Epic

- `write_template` MCP tool (Epic 5).
- CLI authoring (Epic 6).

## Acceptance Criteria

1. **Empty directory.** Against `fake_templates_dir({})` (or with the directory absent), `list_templates()` returns `{"templates": [], "skipped": []}` — no error.
2. **Valid template enumeration.** Drop `orch.toml` (`TEMPLATE_TOML_FULL`) and `mini.toml` (`TEMPLATE_TOML_MINIMAL`) via `fake_templates_dir`. Call `list_templates()`. Assert: `templates[]` has two entries (order-insensitive), each with `name` ∈ {`orch`,`mini`}, `path` ending in `…/templates/<name>.toml`, and `options` reflecting the parsed file's content. `skipped[]` is empty.
3. **Malformed channel.** Drop `bad.toml` (`TEMPLATE_TOML_MALFORMED_PARSE`) and `good.toml` (`TEMPLATE_TOML_MINIMAL`). Assert: `templates[]` has the one valid entry; `skipped[]` has one entry for `bad.toml` with `err_name=ErrTemplateMalformed` and a non-empty `err_description`.
4. **Tool count + error wrapping.** `pytest tests/test_mcp_stdio_server.py` shows the tool-count assertion updated to 7 and the error-wrapping assertion extended to cover `list_templates`. A test that forces `list_templates_impl` to raise an unexpected exception surfaces as `ErrUnexpected` via the FastMCP wrapper.
5. **No accidental real-disk access.** No test in this Epic touches `~/.claude-spawn/templates/`; the `fake_templates_dir` helper is the only filesystem surface used.
6. **Test suite green.** `pytest tests/test_mcp_stdio_list_templates.py tests/test_mcp_stdio_server.py` plus the full suite pass.
7. **Docs.** README has a `list_templates` section showing a worked output including a `skipped[]` example; the architecture doc references the tool.
