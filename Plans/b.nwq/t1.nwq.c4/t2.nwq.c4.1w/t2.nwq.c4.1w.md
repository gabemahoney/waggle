# Task 2 — Documentation: README and arch doc for `list_templates`

## Context

Per SR-12.1, the README documents the `list_templates` tool's output shape including the `skipped[]` channel. Per the Epic body, the arch doc `docs/architecture/spawn_options_and_templates.md` (created in Epic 3) is extended with a short section describing `list_templates` as the discovery surface for the loader's enumerate-all function.

- `README.md` — new `list_templates` subsection within the existing "Templates" section. Covers: no parameters; return shape; `templates[]` entry fields (`name`, `path`, `options`); `skipped[]` entry fields (`path`, `err_name`, `err_description`); missing-directory-is-success behavior; a worked example output showing one valid template and one malformed entry.

Out of scope: `write_template` (Epic 5), CLI subcommand (Epic 6).

## Why

Closes the SR-12 doc bar for Epic 4 — operators can discover what templates exist and recover from malformed-file diagnostics without consulting source code.

- The arch doc has a "Discovery surface: list_templates" section noting both the loader-wrap relationship and the distinction from `list_spawned_workers`.
- No `write_template`, CLI, or readiness content is added.
