## What Needs to Change

In `docs/architecture/spawn_options_and_templates.md`, append a new section (e.g. "Discovery surface: list_templates") after the loader-module section. Cover:

- `list_templates` is the MCP tool that wraps the loader's enumerate-all entry point.
- The data flow: `list_templates_impl` → loader's enumerate-all function → one `(name, path, options-or-error)` result per `.toml` file in the templates directory.
- Valid files surface under `templates[]`; malformed files surface under `skipped[]` with `ErrTemplateMalformed`.
- Missing templates directory yields empty lists (the loader's behavior, surfaced unchanged through `list_templates`).
- A one-sentence note distinguishing `list_templates` (templates directory) from `list_spawned_workers` (Claude Status `instances` table) — different data sources, different return shapes.

Out of scope: `write_template` (Epic 5 extends this doc later), CLI (Epic 6).
