## Context

Epic 4 ships the `list_templates` MCP tool. SR-12.1 requires the README to document the tool's output shape including the `skipped[]` channel, with a worked example. Epic 3 already added a "Templates" section to the README covering the TOML format, worked example, and merge rules — this subtask extends that section.

## What Needs to Change

In `README.md`, add a `list_templates` subsection within the existing "Templates" section (placement: after the TOML format + merge rules content from Epic 3, before any error-catalogue or later content). Cover:

- The tool takes no parameters.
- Return shape: `{"templates": [...], "skipped": [...]}` — operation-success even when the templates directory is empty or absent.
- Each `templates[]` entry has at minimum: `name` (filename stem), `path` (absolute on-disk path), `options` (resolved option map).
- Each `skipped[]` entry has at minimum: `path`, `err_name` (e.g. `ErrTemplateMalformed`), `err_description`.
- A worked example showing the return value when the templates directory has one valid template and one malformed file — the valid one in `templates[]` with all three fields populated, the malformed one in `skipped[]` with a representative `err_description`.
- A note that `list_spawned_workers` (the other "list" tool) queries Claude Status, not the templates directory — they are independent surfaces.

Out of scope: do NOT add `write_template` (Epic 5) or `claude-spawn write-template` CLI (Epic 6) content.

## Key Files

- `README.md` — `list_templates` subsection under the existing "Templates" section.

## Acceptance Criteria

- The README has a `list_templates` subsection within the "Templates" section.
- The worked example shows both a `templates[]` entry (with `name`/`path`/`options`) and a `skipped[]` entry (with `path`/`err_name`/`err_description`).
- The "missing directory returns empty lists, not an error" semantic is stated explicitly.
- The distinction from `list_spawned_workers` is noted.
- No `write_template` or CLI content is added.
