# Task 4 — Update README and `docs/architecture/spawn_worker.md` for Epic 1

## Context

SR-12.1 and SR-12.2 require documentation updates. Epic 1's slice of those requirements is the option list, the per-option fallback table, the Claude Code 4-layer settings stack, the `cwd` behavioral change (no URL cloning, no relative paths), the `--dangerously-skip-permissions` precedence note, and the existing arch doc's rewrite to reflect the new options and overlay synthesis. The template-related docs, the readiness-blocking note, and the new `docs/architecture/spawn_options_and_templates.md` arch doc are deferred to their respective Epics (3 and 2).

## What Needs to Change

- `README.md`:
  - Replace the legacy `spawn_worker` parameter table with the new 12-option table plus the SR-1.3 fallback-category table.
  - Replace the return-shape snippet with `{instance_id, tmux_session_name}` (drop `ok`).
  - Add a `cwd` behavioral-change callout: URL inputs are rejected; relative paths are rejected; `~`-prefixed paths are accepted and expanded.
  - Add a "Claude Code settings stack" subsection covering the 4-layer stack and how `claude_home`, `claude_settings`, and `permissions` map onto it, including the SR-9.4 `--dangerously-skip-permissions` precedence note.
- `docs/architecture/spawn_worker.md`:
  - Update the parameters table to the new 12-option signature.
  - Update the env-var list in the flow section to the SR-4.2 set (renamed `_CWD` label, conditional `_MODEL`).
  - Update the command-line composition step to show conditional `--model`/`--effort`/`--settings` + verbatim `claude_args`.
  - Add a high-level note on the three overlay-synthesis paths (permissions-only, claude_settings-only, composite) with a deferral pointer for the templates story (Epic 3 will create the dedicated arch doc).
  - Update the errors table to include the eight new Epic 1 error names.
  - Update the mermaid sequence diagram to reflect the new env vars, start-directory, and conditional command.

## Why

Closes the operator-facing documentation gap so a developer reading the README can use the new options without reading source code. Brings the existing arch doc in sync with shipped behavior so it remains the canonical internal reference.

## Success Criteria

- The README `spawn_worker` section describes all 12 options and includes the fallback-category table.
- The README has the `cwd` behavioral-change callout and the settings-stack subsection.
- `grep -i "\brepo\b" README.md` returns no matches in the `spawn_worker` section (other than possible historical context outside the spawn section).
- `docs/architecture/spawn_worker.md` no longer references `repo` cloning and reflects the SR-4.2 env-var set, the SR-4.4 command composition, and the SR-9.1 in-scope error names.
- Operators can read the README and successfully spawn a worker using the new options without consulting source code.
