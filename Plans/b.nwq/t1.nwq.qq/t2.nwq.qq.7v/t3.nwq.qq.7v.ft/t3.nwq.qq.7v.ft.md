## Context

The old `repo` parameter accepted certain inputs that the new `cwd` rejects. Without a clear callout, operators migrating from the old shape will pass HTTPS GitHub URLs or relative paths and get opaque `ErrCwdNotAPath` errors. Depends on Task 3 landing.

## What Needs to Change

In `README.md`, immediately after the new parameter tables (subtask 4-A), add a short callout (note box or prose paragraph) titled something like **`cwd` accepts local paths only** describing:

- `cwd` must be an absolute local filesystem path (or a `~`-prefixed path that expands via `os.path.expanduser`) pointing at a directory that already exists.
- HTTPS URLs and SSH URLs are rejected with `ErrCwdNotAPath`. Claude Spawn never clones from a remote URL on the operator's behalf — pre-clone, then pass the absolute path.
- Relative paths (e.g. `./myrepo`) are rejected with `ErrCwdNotAPath` so the resolved directory is never ambiguous about which process's working directory it is relative to.
- A non-existent path is rejected with `ErrCwdNotFound`. Claude Spawn does not create the directory.

## Key Files

- `README.md` — new callout in or immediately after the `spawn_worker` parameter section.

## Acceptance Criteria

- The callout explicitly states that HTTPS/SSH URL inputs return `ErrCwdNotAPath`.
- The callout explicitly states that relative paths are rejected.
- `~`-prefixed paths are documented as valid.
- The callout does not describe the legacy `repo` parameter as still supported anywhere.
