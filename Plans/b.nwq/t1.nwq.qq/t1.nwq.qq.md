# Epic 1 — New `spawn_worker` option surface and Claude Code launch composition

Replace the legacy `(model, repo, session_name)` parameters with the v1 12-option surface (minus `template`, which lands in Epic 3) and update the tmux launch path accordingly.

## Scope

- **Signature (SR-1.1, 1.2, 1.4):** `spawn_worker(template?, model?, thinking?, cwd, tmux_session_name?, instance_id?, claude_home?, claude_settings?, extra_env?, claude_status_labels?, claude_args?, permissions?)`. `cwd` required; all others optional. Return shape `{instance_id, tmux_session_name}` — rename `session_name` → `tmux_session_name` with no alias. Old positional `repo` and old `session_name` keyword are removed (no compatibility shim). `template` parameter is accepted and validated as a string but loads nothing in this Epic — template-driven defaults arrive in Epic 3. `permissions` keys (`allow`/`deny`/`ask`) are honored via overlay synthesis (SR-4.5).
- **Default categories (SR-1.3):** hardcoded defaults for `tmux_session_name`, `instance_id`, `extra_env`, `claude_status_labels`, `claude_args`, `permissions`; inherits-Claude-Code defaults for `model`, `thinking`, `claude_home`, `claude_settings`.
- **Folder-name sanitize helper (SR-3):** new helper in `src/claude_spawn/spawn.py` implementing the SR-3.2 algorithm and the SR-3.3 example mapping. Used only by the `tmux_session_name` default; `instance_id` default is a bare UUIDv4. The 8-char slice uses the resolved `instance_id` raw (no UUID parsing, no padding).
- **Launch composition (SR-4.1–4.4):** `tmux new-session -d -s <name>` with `-e KEY=VALUE` for the built-in env set (`CLAUDE_STATUS_INSTANCE_ID`, `CLAUDE_STATUS_RELAY_MODE=on`, `CLAUDE_STATUS_AUQ_MODE=record`, `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_OWNED=1`, `..._SESSION_NAME`, `..._CWD`, and conditional `..._MODEL`) plus merged `extra_env` and `claude_status_labels`, plus `HOME=<claude_home>` when supplied. Working directory set via tmux's start-directory flag (no `cd` send-keys preamble). Command line composes `claude` + `--model <m>` + `--effort <level>` + `--settings <path>` + `claude_args[…]` + `Enter`. The `_REQUIRED_ENV_VARS` tuple is updated to the SR-4.2 unconditional set; conditional `MODEL` label is asserted separately.
- **Settings overlay synthesis (SR-4.5):** when `permissions` is non-empty and no `claude_settings` supplied, synthesize a per-spawn JSON overlay containing `{"permissions": {...}}` and pass via `--settings`. When both are supplied, read the caller's file, layer the resolved `permissions` keys (`allow`/`deny`/`ask`) on top of the file's matching keys, write composite to a per-spawn temp file, pass that. SR-9.3 precedence: first-class `permissions` wins on `allow`/`deny`/`ask`; other top-level keys pass through unchanged.
- **`claude_args` --settings conflict (SR-4.6):** before constructing the command line, scan `claude_args` for `--settings` (both `--settings VAL` and `--settings=VAL` forms). If present and `claude_settings` is also resolved non-empty, fail with `ErrClaudeArgsSettingsConflict`. No silent strip/rewrite. Guard is `--settings`-only; `--model`/`--effort` duplicates remain unguarded (Claude Code's last-wins).
- **Validation errors (SR-9.1, 9.2):** `ErrCwdMissing`, `ErrCwdNotFound`, `ErrCwdNotAPath` (rejects HTTPS/SSH URLs and relative paths; `~`-prefixed accepted via `os.path.expanduser`), `ErrClaudeSettingsNotFound`, `ErrInstanceIdCollision` (checked via existing `claude_status.workers` call), `ErrClaudeArgsSettingsConflict`, `ErrReservedEnvKey` (the SR-4.3 reserved set), `ErrThinkingInvalid` (only `low`/`medium`/`high`/`xhigh`). `ErrSpawnReadinessTimeout`/`ErrSpawnWorkerExitedEarly` are deferred to Epic 2.
- **Label rename (SR-10.2):** rename `claude_spawn_repo` → `claude_spawn_cwd` at the env-var emit site in `spawn_worker_impl`, at the read site in `list_spawned_workers_impl`, and in `tests/helpers.py::fake_worker_record` defaults (SR-11.3 — also add the new `cwd=` keyword on the helper).
- **Tests:** update `tests/test_engine_spawn_worker.py` for the new signature (`cwd` parameter, `tmux_session_name` return key, removed positional `repo`). Add `tests/test_engine_spawn_worker_options.py` covering: every new option flowing into the tmux invocation, SR-3 sanitize helper unit cases, SR-4.5 overlay synthesis (`permissions`-only, `claude_settings`-only, composite), SR-4.6 conflict guard, every SR-9.1 error path in this Epic. Patches `claude_spawn.spawn._tmux` and `claude_spawn.claude_status._run` via `tests.helpers.fake_claude_status` for the `instance_id`-collision precheck.
- **Documentation:** README updated with the full 12-option list, the per-option fallback table, the Claude Code 4-layer settings stack and `claude_home`/`claude_settings`/`permissions` mapping onto it, and the behavioral change from the old `repo` parameter (URL cloning no longer happens; relative paths rejected). `docs/architecture/spawn_worker.md` updated with the new options and replaces `repo`-centric phrasing. Template and readiness sections deferred to their respective Epics.

## Out of scope for this Epic

- `template=<name>` loading from disk (Epic 3).
- Spawn-readiness blocking (Epic 2). `spawn_worker` still returns as soon as the tmux session is created, same as today.
- `list_templates`, `write_template`, `claude-spawn write-template` (Epics 4–6).

## Acceptance Criteria

The User (or the agent on their behalf) demonstrates the following:

1. **New signature accepted, old one rejected.**
   - `spawn_worker(cwd="/tmp")` succeeds (against stubbed tmux). Return shape is exactly `{"instance_id": ..., "tmux_session_name": ...}` with no `session_name` key.
   - `spawn_worker("opus", "/tmp", "name")` (old positional shape with `repo` second) fails at the MCP layer with a parameter-mismatch error from FastMCP.
2. **Sanitize helper produces SR-3.3 outputs.** A unit test in `tests/test_engine_spawn_worker_options.py` iterates the SR-3.3 table and asserts each row.
3. **Launch composition observed via stubbed tmux.** Run `spawn_worker(cwd="/tmp", model="opus", thinking="xhigh", extra_env={"FOO":"bar"}, claude_status_labels={"role":"orch"}, permissions={"allow":["Bash"],"deny":["WebFetch"]})`. Inspect the captured `_tmux` calls:
   - `new-session` invocation has `-e CLAUDE_STATUS_INSTANCE_ID=...`, `-e CLAUDE_STATUS_RELAY_MODE=on`, `-e CLAUDE_STATUS_AUQ_MODE=record`, `-e CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_OWNED=1`, `-e CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_SESSION_NAME=...`, `-e CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_CWD=/tmp`, `-e CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_MODEL=opus`, `-e CLAUDE_STATUS_LABEL_ROLE=orch`, `-e FOO=bar`, plus start-directory `/tmp`.
   - `send-keys` invocation contains `claude --model opus --effort xhigh --settings <path>` followed by `Enter`. The overlay file at `<path>` contains `{"permissions":{"allow":["Bash"],"deny":["WebFetch"]}}`.
4. **Composite overlay precedence.** With `claude_settings="<file with permissions.allow=['X'] and theme='dark'>"` and per-call `permissions={"allow":["Y"]}`, the synthesized composite overlay has `permissions.allow=["Y"]` and `theme="dark"`.
5. **Error paths.** Each of these returns the documented `err_name` and writes/spawns nothing:
   - `spawn_worker()` (no `cwd`) → `ErrCwdMissing`.
   - `spawn_worker(cwd="/nonexistent")` → `ErrCwdNotFound`.
   - `spawn_worker(cwd="https://github.com/x/y")` → `ErrCwdNotAPath`.
   - `spawn_worker(cwd="./rel")` → `ErrCwdNotAPath`.
   - `spawn_worker(cwd="/tmp", claude_settings="/nope.json")` → `ErrClaudeSettingsNotFound`.
   - `spawn_worker(cwd="/tmp", instance_id="<active-worker-id>")` against fake Claude Status returning that worker → `ErrInstanceIdCollision`.
   - `spawn_worker(cwd="/tmp", claude_args=["--settings","/x"], claude_settings="/y")` → `ErrClaudeArgsSettingsConflict`.
   - `spawn_worker(cwd="/tmp", extra_env={"CLAUDE_STATUS_INSTANCE_ID":"x"})` → `ErrReservedEnvKey`.
   - `spawn_worker(cwd="/tmp", thinking="huge")` → `ErrThinkingInvalid` listing the four valid values.
6. **Label rename complete.** `grep -r "claude_spawn_repo\|waggle_repo" src/ tests/` returns no live code matches (post-rename label is `claude_spawn_cwd`). `list_spawned_workers` projects the worker's `cwd` from the new label key.
7. **Test suite green.** `pytest` against the full suite (the existing `tests/test_engine_spawn_worker.py` plus the new `tests/test_engine_spawn_worker_options.py`) passes with no skips for the new tests.
8. **Docs reflect the new surface.** README's spawn section describes all 12 options and the fallback table; `docs/architecture/spawn_worker.md` no longer references `repo` cloning.
