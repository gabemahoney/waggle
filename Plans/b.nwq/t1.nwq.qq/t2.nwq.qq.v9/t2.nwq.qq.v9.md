# Task 1 — Add folder-name sanitize helper

## Context

The new `tmux_session_name` default in Task 3 is composed as `<sanitize(cwd)>-<instance_id[:8]>`. The sanitize helper is a self-contained algorithm that produces tmux-safe identifiers from a filesystem path. It is also the simplest piece of Epic 1 to land independently.

## What Needs to Change

- New private helper in `src/claude_spawn/spawn.py` implementing the SR-3.2 algorithm.
- Output must satisfy the SR-3.3 example mapping verbatim and be a non-empty string of `[A-Za-z0-9_-]+`.
- The helper is dead code at the end of this Task; Task 3 wires it into the `tmux_session_name` default and adds the unit tests (in the new `tests/test_engine_spawn_worker_options.py` file created in Task 3, per SR-11.6).

## Why

Lands the smallest standalone piece of Epic 1 first. Decouples the algorithm from the larger signature change in Task 3 so it can be reviewed and merged independently.

## Success Criteria

- The helper exists and is importable from `claude_spawn.spawn`.
- Manual ad-hoc check (or REPL spot-check) confirms each SR-3.3 row maps to the documented output.
- Existing test suite remains green (the helper is unused until Task 3).

## Dependencies

None.
