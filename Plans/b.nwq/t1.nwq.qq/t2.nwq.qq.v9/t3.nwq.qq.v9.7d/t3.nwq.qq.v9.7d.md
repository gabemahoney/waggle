## Context

Task 3 will compute the default `tmux_session_name` as `<sanitize(cwd)>-<instance_id[:8]>`. The sanitize helper is a self-contained algorithm with no runtime dependencies; landing it now lets it be reviewed in isolation. Nothing calls the helper until Task 3 wires it in.

## What Needs to Change

Add a module-private function to `src/claude_spawn/spawn.py` implementing the SR-3.2 algorithm:
1. Normalize the input path so trailing slashes and redundant separators don't affect the result.
2. Extract the final path segment.
3. Substitute `root` when the segment is empty or `"."`.
4. Replace tmux-illegal characters (at minimum `.` and `:`) with `-`.
5. Substitute `root` again if the result is empty or all-punctuation after step 4.

Output is always a non-empty string of `[A-Za-z0-9_-]+`. Unit tests for the helper are added in Task 3's new `tests/test_engine_spawn_worker_options.py` (SR-11.6); this subtask just ships the impl.
