# Task 3 — Replace `spawn_worker` signature: 12-option surface, launch composition, validation, and overlay synthesis

## Context

The legacy `(model, repo, session_name)` signature is undersized. SR-1.1 defines a new 12-option surface; SR-4 defines how those options compose into the tmux launch; SR-9 (in-scope subset) defines the validation errors; SR-4.5 defines the settings-overlay synthesis that realizes the first-class `permissions` option. All of these changes are interlocked — the signature change alone leaves the tmux path inconsistent, and the launch composition cannot land without the validation block protecting it. So this Task is single commit-sized but internally subdivided into four engineer subtasks (T3-A through T3-D) and five test subtasks (3.1–3.5) plus a run-tests gate.

## What Needs to Change

- `src/claude_spawn/mcp_stdio.py` — `spawn_worker` MCP tool parameter list replaced with all twelve SR-1.1 options; return docstring updated to `{instance_id, tmux_session_name}`.
- `src/claude_spawn/spawn.py` — `spawn_worker_impl` signature replaced; hardcoded defaults per SR-1.3 (UUIDv4 `instance_id`, `<sanitize(cwd)>-<instance_id[:8]>` `tmux_session_name` using the helper from Task 1, empty collections for the rest); validation block for the eight in-scope SR-9.1 errors; tmux launch composition with start-directory, conditional `_MODEL` label, merged `extra_env`/`claude_status_labels` (uppercased label keys), `HOME=<claude_home>` when supplied; command line composition with conditional `--model`/`--effort`/`--settings` + verbatim `claude_args`; settings overlay synthesis (permissions-only, claude_settings-only, composite); return key renamed `session_name` → `tmux_session_name`; `_REQUIRED_ENV_VARS` tuple updated to the SR-4.2 unconditional set.
- `tests/test_engine_spawn_worker.py` — call sites updated to the new signature; assertions updated for `tmux_session_name` return key; negative-case test for old positional shape rejection added.
- `tests/test_engine_spawn_worker_options.py` — NEW file covering the sanitize-helper SR-3.3 table, default `tmux_session_name` composition, every new option's effect on the tmux invocation, the three overlay-synthesis cases plus the no-overlay case, and every in-scope SR-9.1 error path.
- The `template` parameter is accepted as an optional string but not loaded from disk — that wiring is Epic 3's job.

## Why

Realizes the core Epic 1 capability: operators can configure 12 first-class options per spawn, with validation and the settings overlay producing the right behavior at the worker. Replaces the legacy three-knob signature with no compatibility shim.

## Success Criteria

- `spawn_worker_impl(cwd="/tmp")` (against stubbed tmux) returns `{instance_id, tmux_session_name}` with no `session_name` key and no `ok` key.
- Passing the old positional `(model, repo, session_name)` shape fails at the MCP layer with a FastMCP parameter-mismatch error.
- Each of the eight in-scope SR-9.1 errors (`ErrCwdMissing`, `ErrCwdNotFound`, `ErrCwdNotAPath`, `ErrClaudeSettingsNotFound`, `ErrInstanceIdCollision`, `ErrClaudeArgsSettingsConflict`, `ErrReservedEnvKey`, `ErrThinkingInvalid`) is reachable, returns the documented `err_name`, and causes zero tmux invocations.
- The captured `_tmux` calls for a representative full-option invocation match SR-4.1–4.4 (env vars present, start-directory set, command line conditional flags applied, claude_args appended verbatim).
- The settings-overlay synthesis produces the documented file content for each of the three cases (permissions-only, claude_settings-only, composite) and emits no `--settings` flag when neither is supplied.
- Composite overlay precedence: first-class `permissions.allow`/`deny`/`ask` overrides the caller's file, other top-level keys pass through unchanged.
- `pytest` is green on the full suite, including the new `tests/test_engine_spawn_worker_options.py`.

## Dependencies

- Task 2 (label rename: `CLAUDE_SPAWN_CWD` is the env var key emitted by the new launch composition).
