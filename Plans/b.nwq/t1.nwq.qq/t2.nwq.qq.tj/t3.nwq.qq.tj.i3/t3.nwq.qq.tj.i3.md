## Context

Lays the foundation for Task 3: replaces the legacy `(model, repo, session_name)` parameter list with the new SR-1.1 twelve-option surface and wires the SR-1.3 hardcoded defaults (using the sanitize helper from Task 1). The `template` parameter is accepted as an optional string but ignored at impl level — wiring it to load from disk is Epic 3's job. The existing tmux launch path is preserved as-is in this subtask; new launch composition lands in a sibling subtask.

## What Needs to Change

In `src/claude_spawn/mcp_stdio.py`:
- Replace the `spawn_worker` FastMCP tool's parameter list with all twelve SR-1.1 options, typed appropriately. `cwd` is required; all other options are optional with their SR-1.3 defaults. Update the tool's docstring to describe the return as `{instance_id, tmux_session_name}`.

In `src/claude_spawn/spawn.py`:
- Replace `spawn_worker_impl`'s signature with the same twelve parameters.
- Compute the SR-1.3 hardcoded defaults: bare UUIDv4 for `instance_id`; `<sanitize(cwd)>-<instance_id[:8]>` for `tmux_session_name` (using Task 1's helper); empty dicts/lists for `extra_env`, `claude_status_labels`, `claude_args`, `permissions`.
- For SR-1.3 inherits-Claude-Code-default options (`model`, `thinking`, `claude_home`, `claude_settings`), leave them unset when omitted — do not inject a value.
- Rename the return dict key `session_name` → `tmux_session_name`. No alias.
- Update `_REQUIRED_ENV_VARS` to the SR-4.2 unconditional set (the `_MODEL` label is conditional and is removed from the tuple; the renamed `_CWD` label from Task 2 is in the tuple).

In `tests/test_engine_spawn_worker.py`:
- Update every existing call site to the new keyword-only signature (`cwd=`, `tmux_session_name=`).
- Update assertions on `result["session_name"]` → `result["tmux_session_name"]`.

The validation block, the new tmux launch composition, and the settings overlay synthesis are intentionally NOT included in this subtask — they land in sibling subtasks T3-B / T3-C / T3-D within the same Task 3 commit.

## Key Files

- `src/claude_spawn/mcp_stdio.py` — tool parameter list and return-shape docstring.
- `src/claude_spawn/spawn.py` — `spawn_worker_impl` signature, hardcoded default computation, return key, `_REQUIRED_ENV_VARS`.
- `tests/test_engine_spawn_worker.py` — call-site and assertion updates.

## Acceptance Criteria

- `spawn_worker_impl(cwd="/tmp")` (with patched `_tmux`) returns a dict with keys exactly `{instance_id, tmux_session_name}`.
- The default `tmux_session_name` matches `<sanitize(cwd)>-<instance_id[:8]>`.
- Passing the old positional `repo` argument raises `TypeError` (or surfaces as a FastMCP parameter-mismatch at the MCP layer).
- `_REQUIRED_ENV_VARS` membership matches the SR-4.2 unconditional set.
- The existing `tests/test_engine_spawn_worker.py` suite is green after the call-site/assertion updates (with the legacy tmux path still in place — sibling subtasks T3-B/T3-C/T3-D will replace it).
