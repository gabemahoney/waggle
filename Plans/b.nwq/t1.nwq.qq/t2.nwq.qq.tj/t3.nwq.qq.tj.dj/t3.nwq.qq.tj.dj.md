## Context

All validation must happen before any tmux call, with each failure returning the SR-7.1 operation-failed dict (`{ok: False, operation: "spawn_worker", err_name: ..., err_description: ...}`) and an `err_description` that names the offending input so the LLM can act without consulting source code. The in-scope SR-9.1 error set for Epic 1 is eight names. `ErrSpawnReadinessTimeout` and `ErrSpawnWorkerExitedEarly` are Epic 2's responsibility and are NOT included here.

## What Needs to Change

In `src/claude_spawn/spawn.py`, add a validation block in `spawn_worker_impl` that runs before any env-arg construction or tmux call. Each of the following triggers must surface as the named error:

- **`ErrCwdMissing`** — `cwd` is `None` after default resolution (no per-call and no template — and in this Epic, templates are not loaded, so any unspecified `cwd` is missing).
- **`ErrCwdNotAPath`** — resolved `cwd` is an HTTPS URL, SSH URL, or any string that is not absolute or `~`-prefixed. `~`-prefixed paths must be expanded via `os.path.expanduser` and then validated for absoluteness.
- **`ErrCwdNotFound`** — expanded `cwd` does not exist on the filesystem. Do not auto-create the directory.
- **`ErrThinkingInvalid`** — `thinking` is supplied but not one of `{low, medium, high, xhigh}`. The `err_description` must list the four valid values.
- **`ErrClaudeSettingsNotFound`** — `claude_settings` is supplied but the path does not exist or is not readable. Do not create or substitute an empty file.
- **`ErrClaudeArgsSettingsConflict`** — `claude_args` contains a `--settings` flag in EITHER form (`--settings VALUE` as two list entries, or `--settings=VALUE` as a single entry) AND `claude_settings` is also resolved to a non-empty value. Do not silently strip or rewrite. Guard `--settings` only; do NOT guard `--model` or `--effort` duplicates (Claude Code defines last-wins for those).
- **`ErrReservedEnvKey`** — any key in merged `extra_env` collides with the SR-4.2 built-in env-var set: `CLAUDE_STATUS_INSTANCE_ID`, `CLAUDE_STATUS_RELAY_MODE`, `CLAUDE_STATUS_AUQ_MODE`, any `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_*` key, or `HOME` (only when `claude_home` is also supplied). The `err_description` must name the offending key.
- **`ErrInstanceIdCollision`** — when caller supplies an explicit `instance_id`, query `claude_status.workers(label="claude_spawn_owned=1")` and check for exact-equality match against the supplied id. The collision check uses the existing `_run` subprocess seam.

## Key Files

- `src/claude_spawn/spawn.py` — validation block at the top of `spawn_worker_impl`.

## Acceptance Criteria

- Each of the eight error names is reachable. Each returns the operation-failed dict with the documented `err_name` and a description naming the offending value.
- No tmux call is made when any validation fails.
- The dual-form `--settings` scan correctly detects both `claude_args=["--settings","/x"]` and `claude_args=["--settings=/x"]`.
- The reserved-env-key check covers the SR-4.2 set exactly — including conditional `HOME` only when `claude_home` is supplied.
- Valid inputs pass the validation block without raising or returning an error and reach the launch composition (T3-C).
