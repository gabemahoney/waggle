## Context

With the new signature in place (T3-A) and inputs validated (T3-B), the tmux invocation must be rebuilt to compose the SR-4.1–4.4 launch surface from the new options: built-in env set, merged caller env, `HOME=<claude_home>`, start-directory, and a conditional command line with `--model`/`--effort`/`--settings`/`claude_args`. The settings overlay file path consumed by `--settings` comes from sibling subtask T3-D.

## What Needs to Change

In `src/claude_spawn/spawn.py`, replace the env-arg construction and `send-keys` command string in `spawn_worker_impl`:

**Tmux `new-session` invocation:**
- Set the session's working directory via tmux's start-directory flag (e.g. `-c <resolved cwd>`). Do NOT prepend a `cd` command in the send-keys payload — the load-bearing realization of "Claude Status's `cwd` reflects the supplied `cwd`" is the start-directory flag.
- Emit the SR-4.2 unconditional env-var set as `-e KEY=VALUE` pairs:
  - `CLAUDE_STATUS_INSTANCE_ID=<resolved>`
  - `CLAUDE_STATUS_RELAY_MODE=on`
  - `CLAUDE_STATUS_AUQ_MODE=record`
  - `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_OWNED=1`
  - `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_SESSION_NAME=<resolved tmux_session_name>`
  - `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_CWD=<resolved cwd>`
- Emit the conditional `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_MODEL=<resolved model, lowercased>` ONLY when `model` is resolved to a non-empty value; omit otherwise.
- Append every entry of merged `extra_env` as `-e KEY=VALUE`. Values pass through unchanged (no sanitization beyond the reserved-key guard in T3-B).
- Append every entry of merged `claude_status_labels` as `-e CLAUDE_STATUS_LABEL_<UPPERCASED_KEY>=<VALUE>` (operators pass the bare label name; the impl uppercases for env-var conformance).
- Append `-e HOME=<claude_home>` when `claude_home` is supplied.

**Tmux `send-keys` command-line composition:**
The launch command is built from these pieces, in order:
1. The literal `claude` invocation.
2. `--model <resolved model>` IFF `model` is non-empty.
3. `--effort <resolved thinking>` IFF `thinking` is non-empty.
4. `--settings <path>` IFF the overlay-synthesis step (T3-D) produced a path.
5. Every entry of resolved `claude_args` appended verbatim.

Final keysym to `send-keys` is the literal `Enter`, matching the existing pattern.

## Key Files

- `src/claude_spawn/spawn.py` — env-arg construction and send-keys composition inside `spawn_worker_impl`.

## Acceptance Criteria

- `tmux new-session` argv contains the start-directory flag pointing at the resolved `cwd`.
- The six SR-4.2 unconditional env vars are present on every spawn.
- The `_MODEL` label is present when and only when `model` is supplied.
- `extra_env` entries appear as `-e KEY=VAL`.
- `claude_status_labels` entries appear as `-e CLAUDE_STATUS_LABEL_<UPPER>=<VAL>`.
- `HOME=<claude_home>` appears when `claude_home` is supplied.
- The `send-keys` command omits `--model` when not supplied, includes `--effort <level>` only when `thinking` is supplied, includes `--settings <path>` only when an overlay path is resolved, and appends `claude_args` verbatim.
- Existing happy-path tests in `tests/test_engine_spawn_worker.py` (updated in T3-A) continue to pass after this subtask lands.
