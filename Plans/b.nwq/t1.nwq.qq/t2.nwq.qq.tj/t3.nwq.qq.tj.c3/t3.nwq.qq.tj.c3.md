## Context

T3-D implements the SR-4.5 settings overlay synthesis. This subtask extends `tests/test_engine_spawn_worker_options.py` with coverage for the four cases (neither / permissions-only / claude_settings-only / composite) including the SR-9.3 composite-precedence assertion (first-class `permissions` wins on `allow`/`deny`/`ask`; unrelated top-level keys pass through unchanged).

## What Needs to Change

Extend `tests/test_engine_spawn_worker_options.py` with a `TestSettingsOverlay` class covering:

- **Neither supplied:** call with `permissions={}` and no `claude_settings`. Assert `--settings` does NOT appear in the send-keys argv.
- **`permissions`-only:** call with `permissions={"allow":["Bash"],"deny":["WebFetch"]}`. Assert `--settings <path>` appears in send-keys. Read the file at `<path>` and assert content is exactly `{"permissions": {"allow": ["Bash"], "deny": ["WebFetch"]}}` (JSON parse to compare).
- **`claude_settings`-only:** write a temp JSON file with arbitrary content (e.g. `{"theme":"dark"}`) and supply its path as `claude_settings=`. Assert `--settings <that-same-path>` appears verbatim in send-keys; no temp file is synthesized (the path in the command equals the caller's path).
- **Composite:** write a temp JSON file with `{"permissions":{"allow":["X"]},"theme":"dark"}` and supply its path AND `permissions={"allow":["Y"]}`. Assert `--settings <composite-path>` appears (NOT the caller's original path). Read the composite file and assert: `permissions.allow == ["Y"]` (per-call wins on the `allow` key); `theme == "dark"` (unrelated key passes through unchanged).
- **Composite preserves untouched permission keys:** caller's file has `permissions.deny=["D"]`, per-call has `permissions.allow=["A"]` only. Assert composite has `{"permissions":{"allow":["A"],"deny":["D"]}}` (the per-call `allow` replaces, the file's `deny` is preserved since the per-call did not supply `deny`).

Tests inspect the resolved `--settings` path captured from the send-keys argv; they do NOT hardcode temp file paths.
