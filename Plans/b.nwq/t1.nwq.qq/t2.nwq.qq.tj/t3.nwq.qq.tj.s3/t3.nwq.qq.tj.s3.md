## Context

Task 3 removes the old `(model, repo, session_name)` signature outright and renames the return key. Every existing test in `tests/test_engine_spawn_worker.py` that calls `spawn_worker_impl` or asserts on `result["session_name"]` will break. This subtask updates those tests to the new shape and adds a negative-case test confirming the old signature is rejected (Epic 1 AC1 second clause).

## What Needs to Change

In `tests/test_engine_spawn_worker.py`:
- Update every call to `spawn_worker_impl` to use the new keyword-only signature (`cwd=…`, `tmux_session_name=…`, etc.). Remove all positional `repo`/`session_name` arguments.
- Replace every assertion on `result["session_name"]` with `result["tmux_session_name"]`.
- Update the default-session-name composition test(s) to expect `<sanitize(cwd)>-<instance_id[:8]>` (SR-3.4), not the legacy `spawn-…` prefix.
- Update any shared helper inside the test module (e.g. `_run_and_get_new_session_argv`) to the new keyword API.
- Add a NEW negative-case test verifying that calling the FastMCP tool with the old positional shape (e.g. `spawn_worker("opus", "/tmp", "name")` against the MCP test surface) is rejected with a parameter-mismatch error from FastMCP. This is Epic 1 AC1 second clause and must be explicit.
- Where a happy-path test asserts on launch composition that includes `--model`, ensure it passes `model=...` so the conditional `--model` flag is emitted (the impl now omits `--model` when `model` is unsupplied).

## Key Files

- `tests/test_engine_spawn_worker.py` — wholesale call-site/assertion updates plus one new negative test.

- The negative-case test for the old positional shape exists and asserts a FastMCP parameter-mismatch / TypeError-equivalent outcome.
