## Context

T3-B implements the validation block with eight in-scope SR-9.1 errors. This subtask extends `tests/test_engine_spawn_worker_options.py` with a dedicated test per error name, asserting both the `err_name` value and that no tmux call was issued.

## What Needs to Change

Extend `tests/test_engine_spawn_worker_options.py` with a `TestValidationErrors` class covering EXACTLY the eight in-scope SR-9.1 errors. Each test asserts the expected `err_name` AND that the patched `_tmux` was never invoked.

- **`ErrCwdMissing`** — call with no `cwd`.
- **`ErrCwdNotFound`** — call with `cwd="/nonexistent/path/<unique>"`.
- **`ErrCwdNotAPath` (HTTPS URL)** — call with `cwd="https://github.com/x/y"`.
- **`ErrCwdNotAPath` (SSH URL)** — call with `cwd="git@github.com:x/y"`.
- **`ErrCwdNotAPath` (relative path)** — call with `cwd="./relative"`.
- **`ErrClaudeSettingsNotFound`** — call with a valid `cwd` and `claude_settings="/nonexistent.json"`.
- **`ErrInstanceIdCollision`** — call with a valid `cwd` and a supplied `instance_id`. Arrange `fake_claude_status` (via `tests.helpers.fake_claude_status` / `fake_workers_response` / `fake_worker_record`) so the workers list contains the same `instance_id`.
- **`ErrClaudeArgsSettingsConflict` (two-entry form)** — call with `claude_args=["--settings","/x"]` AND `claude_settings=<real-path>`.
- **`ErrClaudeArgsSettingsConflict` (equals form)** — call with `claude_args=["--settings=/x"]` AND `claude_settings=<real-path>`.
- **`ErrReservedEnvKey` (control env)** — call with `extra_env={"CLAUDE_STATUS_INSTANCE_ID":"x"}`.
- **`ErrReservedEnvKey` (label env)** — call with `extra_env={"CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_OWNED":"x"}`.
- **`ErrReservedEnvKey` (`HOME` when `claude_home` set)** — call with `claude_home="/x"` AND `extra_env={"HOME":"y"}`.
- **`ErrThinkingInvalid`** — call with `thinking="huge"`. Additionally assert the description names the four valid values (`low`, `medium`, `high`, `xhigh`).

Tests use `tests.helpers.fake_claude_status` for any Claude Status interaction; they do NOT patch `_run` directly. Tests do NOT hardcode label keys; use `fake_worker_record(..., cwd=...)` if needed.

## Key Files

- `tests/test_engine_spawn_worker_options.py` — extended with `TestValidationErrors`.

## Acceptance Criteria

- Every in-scope SR-9.1 error name has at least one test.
- Both `--settings` forms (space-separated and equals) are covered for `ErrClaudeArgsSettingsConflict`.
- Three sub-cases of `ErrReservedEnvKey` are covered (control env, label env, conditional `HOME`).
- Every test asserts the patched `_tmux` was NEVER invoked (zero calls).
- `pytest tests/test_engine_spawn_worker_options.py::TestValidationErrors` is green after T3-B has landed.
