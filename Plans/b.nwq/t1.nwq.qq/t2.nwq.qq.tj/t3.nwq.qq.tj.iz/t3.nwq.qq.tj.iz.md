## Context

SR-4.5 defines how the first-class `permissions` option and the caller-supplied `claude_settings` overlay file combine into the `--settings <path>` argument that T3-C passes to Claude Code. There are three non-trivial cases plus a no-op case. The resolved path (or `None`) feeds into T3-C's `--settings` argument decision.

## What Needs to Change

In `src/claude_spawn/spawn.py`, after the validation block (T3-B) and before the tmux invocation construction (T3-C), add an overlay-resolution step that produces the effective `--settings` path:

- **Neither supplied** (`permissions` empty AND `claude_settings` unset): resolved path is `None`; T3-C omits the `--settings` flag.
- **`permissions` only** (`permissions` non-empty, no `claude_settings`): synthesize a JSON file containing exactly `{"permissions": <permissions dict>}` and write it to a per-spawn temp location. Resolved path is the temp file.
- **`claude_settings` only** (`claude_settings` set, `permissions` empty): resolved path is the caller's `claude_settings` path. No temp file is created; the caller's file is passed through verbatim.
- **Composite** (`claude_settings` set AND `permissions` non-empty): read the caller's `claude_settings` JSON, replace the keys `allow`/`deny`/`ask` under the file's top-level `permissions` block with the resolved `permissions` map's matching keys (first-class `permissions` wins on those three keys per SR-9.3), preserve all other top-level keys (e.g. `theme`, `defaultMode`, `mcpServers`) unchanged, and write the composite to a per-spawn temp file. Resolved path is the composite path (NOT the caller's original path).

**Temp file lifecycle:** use a per-spawn temp file path (e.g. `tempfile.NamedTemporaryFile(suffix=".json", delete=False)` or an equivalent that Claude Code can read after the function returns). The temp file's lifetime is bounded to the worker's tmux session; explicit cleanup on session termination is OUT of scope for this Epic (best-effort only; documented as such).

## Key Files

- `src/claude_spawn/spawn.py` — overlay-resolution block between the validation block and the tmux invocation.

## Acceptance Criteria

- The four cases (neither / permissions-only / claude_settings-only / composite) each produce the documented resolved path.
- The synthesized `permissions`-only file contains valid JSON `{"permissions": {...}}`.
- The composite file preserves unrelated top-level keys from the caller's file unchanged.
- The composite file's `permissions.allow`/`deny`/`ask` reflect the resolved per-call values (not the file's original values on those three keys).
- The caller's original `claude_settings` file is never modified in place — composite goes to a temp file.
- The resolved path is exposed to T3-C such that `--settings <path>` is emitted iff the path is non-`None`.
