# Task 2 — Rename `claude_spawn_repo` label to `claude_spawn_cwd` end-to-end

## Context

SR-10.2 renames the `claude_spawn_repo` label to `claude_spawn_cwd` to reflect the semantic shift: the value is the worker's working directory, not a cloned repo. The rename must land in lockstep across the env-var emit site, the label read sites in `list_spawned_workers_impl` (and anywhere else `claude_spawn_repo` is consumed), the `fake_worker_record` helper defaults, and any test assertions on the old name. SR-11.3 also adds a `cwd=` keyword argument to `fake_worker_record` so tests asserting on the new label can override the default without constructing a full `labels=` dict.

## What Needs to Change

- Rename `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_REPO` → `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_CWD` in `src/claude_spawn/spawn.py` (emit site and `_REQUIRED_ENV_VARS` tuple membership). Update read sites that look up `claude_spawn_repo` from a worker's `labels` dict.
- Rename `claude_spawn_repo` → `claude_spawn_cwd` in `tests/helpers.py::fake_worker_record` defaults and in `tests/sample_payloads.py` `WORKER_*` constants.
- Add a `cwd=` keyword argument to `fake_worker_record` (additive — existing tests with no `labels=` continue to work).
- Update any test assertions that reference the old key name.

## Why

Lands the label rename cleanly in one commit so subsequent changes (especially Task 3's tmux composition rewrite) build on the new name consistently. Adds the `cwd=` helper keyword that downstream test subtasks depend on.

## Success Criteria

- `grep -r "claude_spawn_repo\|CLAUDE_SPAWN_REPO\|waggle_repo\|WAGGLE_REPO" src/ tests/` returns no live-code matches after this Task lands.
- `fake_worker_record(instance_id="x", status="working", cwd="/tmp")` returns a record with `labels["claude_spawn_cwd"] == "/tmp"`.
- `list_spawned_workers_impl` projects the worker's `cwd` from the new label key.
- Full `pytest` suite is green.

## Dependencies

None.
