## Context

After the impl-side label rename in the sibling subtask, the test helpers and canonical payloads must also rename `claude_spawn_repo` → `claude_spawn_cwd` to keep the suite green. SR-11.3 additionally requires adding a `cwd=` keyword argument to `fake_worker_record` so future tests asserting on the new label can override the default without constructing a full `labels=` dict (additive — existing `labels=`-less callers keep working).

## What Needs to Change

In `tests/helpers.py`:
- Rename the `claude_spawn_repo` key in `_DEFAULT_LABELS` (or wherever the helper currently embeds it) to `claude_spawn_cwd`. Keep the value a plausible local path.
- Add a `cwd=` keyword argument to `fake_worker_record`. When supplied, it sets `labels["claude_spawn_cwd"]` in the returned record. The keyword is additive; tests that omit `cwd=` continue to use the default.

In `tests/sample_payloads.py`:
- Rename `claude_spawn_repo` → `claude_spawn_cwd` in every `WORKER_*` canonical constant's `labels` dict.

## Key Files

- `tests/helpers.py` — `_DEFAULT_LABELS` rename, new `cwd=` keyword.
- `tests/sample_payloads.py` — label key rename in all `WORKER_*` constants.

## Acceptance Criteria

- `fake_worker_record(instance_id="x", status="working")` returns a record with `labels["claude_spawn_cwd"]` and no `claude_spawn_repo` key.
- `fake_worker_record(instance_id="x", status="working", cwd="/tmp/foo")` returns `labels["claude_spawn_cwd"] == "/tmp/foo"`.
- `grep "claude_spawn_repo" tests/helpers.py tests/sample_payloads.py` returns no matches.
- After the sibling impl-rename subtask also lands, full `pytest` is green.
