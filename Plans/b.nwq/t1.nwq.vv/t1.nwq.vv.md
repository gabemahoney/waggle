# Epic 5 — `write_template` MCP tool

Register a new MCP tool that authors a template file from a dict of options, with name-safety, schema validation, and `force` overwrite semantics.

## Scope

- **New MCP tool (SR-7.3):** `write_template(name, options, force=False)` in `src/claude_spawn/mcp_stdio.py`. Wraps through the existing FastMCP try/except → `ErrUnexpected` pattern.
- **Behavior:**
  - Validates `name` per SR-7.4 (no path separators; not `.`, `..`, or any name with `..` substring; non-empty; does not start with `.`).
  - Validates `options` with the same schema rules as load-time validation (SR-6.4): unknown keys, `template` key, invalid `thinking` level, type mismatches → `ErrTemplateOptionsInvalid` with description enumerating the rejected keys/values. No file written on validation failure.
  - Creates `~/.claude-spawn/templates/` if it does not exist.
  - On name collision with an existing file: `force=False` → `ErrTemplateExists` (no modification); `force=True` → atomic overwrite (write to a sibling temp file in the same directory, then `os.rename` to the canonical path).
  - Defensive path check: after constructing the on-disk path, resolve its parent and confirm it equals the templates directory. Any escape attempt → `ErrTemplateNameUnsafe`.
- **Implementation:** a single `write_template_impl(name, options, force)` function in `src/claude_spawn/templates.py` (so Epic 6's CLI can delegate to the same impl). Serializes via `tomli_w`. Returns `{"path": <abs path>, "options": <normalized options>}` on success.
- **Errors surfaced:**
  - `ErrTemplateExists` — collision without `force`.
  - `ErrTemplateNameUnsafe` — name fails SR-7.4.
  - `ErrTemplateOptionsInvalid` — schema validation failure.
- **Tests:**
  - New `tests/test_engine_templates_write.py` covering: minimal success (round-trips through Epic 3 loader), full-options success, name collision without `force`, name collision with `force=True` (replaces atomically), each unsafe-name shape (separator, leading dot, `..` substring, empty), unknown option key, recursive `template` key, invalid `thinking` value, atomic-write resilience (force a crash mid-write — verify the canonical file is unchanged).
  - New `tests/test_mcp_stdio_write_template.py` exercising the MCP-level tool: minimal call, options-invalid call, collision-with-force call, error-wrapping (forced exception → `ErrUnexpected`).
  - Update `tests/test_mcp_stdio_server.py`: tool-count assertion rises 7 → 8, error-wrapping coverage extends to `write_template`.
- **Documentation:** README documents `write_template`'s three parameters, the success return shape, the `force` flag, each error condition (with example error responses), and notes that the resulting file is identical regardless of authoring surface (MCP, CLI, or hand-edit). Architecture doc updated to show `write_template` writing through the same on-disk path that the loader reads.

## Out of scope for this Epic

- `claude-spawn write-template` CLI subcommand (Epic 6).

## Acceptance Criteria

1. **Minimal success + round-trip.** Call `write_template(name="orch", options={"cwd":"/tmp"})` against `fake_templates_dir({})`. Assert: returns `{"path": "<dir>/orch.toml", "options": {"cwd":"/tmp"}}`; the file exists; calling `spawn_worker(template="orch")` succeeds and uses `cwd=/tmp`.
2. **Full options round-trip.** Call `write_template(name="full", options=<full options bundle exercising every supported key>)` ; load via Epic 3's loader; assert the resolved option map equals the input.
3. **Collision without `force`.** Two consecutive calls with the same `name`. The second returns `ErrTemplateExists`; description names the existing path. Assert the file's contents are unchanged from the first write.
4. **Collision with `force=True`.** Same setup; second call with `force=True` succeeds and replaces the file's content. Assert atomic via either: (a) instrument the temp-file → rename path, or (b) inject a faulty serializer that raises mid-write and assert the canonical file is still the original content.
5. **Unsafe-name errors.** Each of these returns `ErrTemplateNameUnsafe` and writes no file:
   - `name="foo/bar"` (path separator)
   - `name=".hidden"` (leading dot)
   - `name=".."`
   - `name="foo/../escape"` (path traversal substring)
   - `name=""` (empty)
6. **Options validation errors.** Each returns `ErrTemplateOptionsInvalid` with the offending keys/values enumerated, and writes no file:
   - `options={"bogus": 1}` (unknown key)
   - `options={"template": "x"}` (recursive)
   - `options={"thinking": "huge"}` (invalid enum)
   - `options={"cwd": 123}` (type mismatch — int vs string)
7. **Directory auto-creation.** Against a `fake_templates_dir` configured so the directory does NOT exist initially, a `write_template` call creates the directory and the file together.
8. **MCP error wrapping.** Force `write_template_impl` to raise; the FastMCP layer returns `err_name=ErrUnexpected` and does not propagate the raw exception.
9. **Tool count.** `pytest tests/test_mcp_stdio_server.py` shows the tool-count assertion at 8.
10. **Test suite green.** `pytest tests/test_engine_templates_write.py tests/test_mcp_stdio_write_template.py tests/test_mcp_stdio_server.py` plus the full suite pass.
11. **Docs.** README has a `write_template` section with examples for each error path; architecture doc shows the tool writing to the same path the loader reads.
