## Context

SR-7.3 registers a new `write_template(name, options, force=False)` MCP tool that delegates to the `write_template_impl` function from the sibling subtask. The tool follows the existing FastMCP pattern in `src/claude_spawn/mcp_stdio.py`: async wrapper, try/except around the impl call, error wrapping via the established `_err` helper that maps unexpected exceptions to `ErrUnexpected`. Tool count rises from 7 (after Epic 4) to 8.

## What Needs to Change

In `src/claude_spawn/mcp_stdio.py`:
- Register a new `@mcp.tool()` async function `write_template(name, options, force=False)`. Parameters match SR-7.3 exactly: `name` is a string, `options` is a dict matching `spawn_worker`'s named parameters minus `template`, `force` is a boolean defaulting to false.
- Delegate to `write_template_impl` from `claude_spawn.templates` via the same `asyncio.to_thread` (or equivalent) pattern used by the other tools.
- Wrap in the established try/except so unexpected exceptions return the standard operation-failed dict with `err_name=ErrUnexpected`.
- Docstring covers: three parameters; success return shape `{path, options}`; each error name (`ErrTemplateNameUnsafe`, `ErrTemplateOptionsInvalid`, `ErrTemplateExists`, `ErrUnexpected`).

No changes to other source files.

## Key Files

- `src/claude_spawn/mcp_stdio.py` — new tool registration.
- `src/claude_spawn/templates.py` — read-only consumption of `write_template_impl`.

## Acceptance Criteria

- The MCP server registers 8 tools total after this subtask.
- Calling `write_template` via the FastMCP test surface with valid inputs returns the impl's success dict (`{path, options}`).
- Each of the impl's error conditions surfaces through the MCP tool unchanged (`ErrTemplateNameUnsafe`, `ErrTemplateOptionsInvalid`, `ErrTemplateExists`).
- An unexpected exception in the impl maps to `err_name=ErrUnexpected` via the FastMCP wrapper.
- The tool signature visible to FastMCP matches SR-7.3 exactly.
