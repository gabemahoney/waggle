# Task 1 — Implement `write_template` MCP tool + tests

## Context

SR-7.3 introduces a new MCP tool `write_template(name, options, force=False)` that authors a template file from a dict of options. SR-7.4 enforces name safety. SR-6.4's load-time validation rules are reused at write time. The underlying impl `write_template_impl` lives in `src/claude_spawn/templates.py` alongside the Epic 3 loader so Epic 6's CLI can delegate to the same function — both surfaces produce identical files for identical inputs. Atomic write via temp + rename within the same directory ensures a crash mid-write does not leave a half-written canonical file.

## What Needs to Change

- `src/claude_spawn/templates.py` — new `write_template_impl(name, options, force)` function: name safety check (SR-7.4) with defensive path-construction check; SR-6.4 options validation; auto-creation of the templates directory; existing-template check with `force` flag; atomic write via temp file + rename within the same directory; serialization through `tomli-w` (already a dependency from Epic 3); returns `{path, options}` on success.
- `src/claude_spawn/mcp_stdio.py` — new `write_template` MCP tool registered alongside the existing seven tools (Epic 4 brought the count to 7; this brings it to 8). Wraps `write_template_impl` through the existing FastMCP try/except wrapper.
- `tests/test_engine_templates_write.py` — NEW impl-level test file (10 cases — see subtasks).
- `tests/test_mcp_stdio_write_template.py` — NEW MCP-level test file (4 cases — see subtasks).
- `tests/test_mcp_stdio_server.py` — tool-count assertion bumped 7 → 8 + `write_template` added to error-wrapping coverage.

## Why

Operators (and LLM orchestrators) can author templates programmatically. The single shared `write_template_impl` ensures the future CLI surface (Epic 6) and the MCP surface produce byte-identical files for identical inputs. The atomic write pattern keeps the canonical path safe across crashes during force-overwrite.

## Success Criteria

- `write_template_impl("orch", {"cwd":"/tmp"}, force=False)` against an empty `fake_templates_dir` writes a `.toml` file that round-trips through the Epic 3 loader to the same options.
- Collision without `force` returns `ErrTemplateExists`; with `force=True` overwrites atomically.
- All five unsafe-name shapes return `ErrTemplateNameUnsafe`.
- All four invalid-options shapes (unknown key, `template` key, invalid `thinking`, type mismatch) return `ErrTemplateOptionsInvalid`.
- Directory auto-creates if absent.
- Unexpected exception in impl is mapped to `ErrUnexpected` via the FastMCP wrapper.
- Tool count in `tests/test_mcp_stdio_server.py` is 8.
- Full `pytest` is green.

## Dependencies

- Epic 3 (`t1.nwq.uu`) — provides `templates.py`, the `_templates_dir()` seam, `fake_templates_dir`, `TEMPLATE_TOML_*` constants, and `tomli-w` runtime dep.
- Epic 4 (`t1.nwq.c4`) — established the 7-tool count this Task increments.
