## Context

SR-11.6 mandates a new test file `tests/test_mcp_stdio_write_template.py` exercising the `write_template` MCP tool via the FastMCP test surface. Confirms the impl is correctly wrapped, that errors propagate through the FastMCP error envelope, and that unexpected exceptions surface as `ErrUnexpected`.

## What Needs to Change

Create `tests/test_mcp_stdio_write_template.py`. Cover:

1. **Happy path:** call the MCP tool with `name="orch", options={"cwd":"/tmp"}, force=False` inside `fake_templates_dir({})`. Assert the returned dict has `path` ending in `orch.toml` and `options={"cwd":"/tmp"}`; no `ok: False`.
2. **Options-invalid call:** call with `options={"bogus": 1}`. Assert `result["ok"] is False`, `err_name="ErrTemplateOptionsInvalid"`, description mentions `bogus`; no file is written.
3. **Collision with force=True:** pre-populate `fake_templates_dir({"orch": ...})` (body programmatically constructed via `tomli_w` from `TEMPLATE_TOML_MINIMAL`'s expected structure, or directly load `TEMPLATE_TOML_MINIMAL` as the body). Call MCP tool with `name="orch", options={"cwd":"/new"}, force=True`. Assert success; assert new content reflects new options.
4. **FastMCP error wrapping:** patch the underlying impl function to raise an unexpected exception. Call the MCP tool. Assert `result["ok"] is False`, `err_name="ErrUnexpected"`, description includes the exception's message.

Every test uses `fake_templates_dir`. No hardcoded TOML literals.
