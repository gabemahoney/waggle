## Context

SR-12.1 requires the README to document the `write_template` MCP tool with all three parameters, success return shape, the `force` flag, and example error responses. Epic 3 already added a "Templates" section to the README; Epic 4 extended it with `list_templates`. This subtask adds the `write_template` subsection.

## What Needs to Change

In `README.md`, add a `write_template` subsection inside the existing "Templates" section. Cover:

- **Parameters:** `name` (string, the template filename stem); `options` (dict matching `spawn_worker`'s named parameters minus `template`); `force` (boolean, default false).
- **Success return shape:** `{path, options}` — absolute on-disk path of the written file and the normalized option map.
- **`force` flag behavior:** without it, a name collision returns `ErrTemplateExists` and the existing file is unchanged. With `force=True`, the file is overwritten atomically (sibling temp file + rename).
- **Example error responses** (one each):
  - `ErrTemplateNameUnsafe` — name contains a path separator, starts with `.`, equals `..`, contains `..` as a substring, or is empty. Show a representative error response dict.
  - `ErrTemplateOptionsInvalid` — `options` contains an unknown key, a `template` key, an invalid `thinking` level, or a type mismatch. Show a representative error response dict that enumerates the rejected keys/values.
  - `ErrTemplateExists` — collision when `force=False`. Show a representative error response dict naming the existing path.
  - `ErrUnexpected` — unexpected server-side exception surfaced through the FastMCP wrapper. One-line mention; no full dict needed.
- **Cross-surface consistency note** (verbatim phrasing): "MCP authoring or hand-edit; the CLI surface added in a later Epic shares the same impl."

Out of scope: do NOT add `claude-spawn write-template` CLI invocation examples (Epic 6).
