# Epic 4: Rename default tmux session prefix

## Outcome
The default tmux session name generated when a caller passes `session_name=None` to `spawn_worker_impl` changes from `waggle-<first8>` to `spawn-<first8>`. Caller-provided session names continue to pass through verbatim.

## Scope

### Source change in `src/claude_spawn/spawn.py`
- `spawn_worker_impl` — the default-session-name generator uses the literal `spawn-` prefix concatenated with `uuid.uuid4().hex[:8]` (or equivalent — the existing first-8-of-UUID semantics is preserved).

### Test changes in `tests/test_engine_spawn_worker.py`
- Update any assertion or regex that checks for the `waggle-` prefix to check for `spawn-` instead.
- A positive test confirms that with `session_name=None`, the resolved session name matches `^spawn-[0-9a-f]{8}$` (or the equivalent UUID-hex shape already used).
- A pass-through test confirms that a caller-provided `session_name` is used verbatim and is not prefixed.

## Acceptance Criteria

- `grep -RIn "waggle-" src/claude_spawn/spawn.py` returns no matches.
- A unit test calling `spawn_worker_impl(session_name=None)` confirms the resulting session name starts with `spawn-` and the suffix matches the UUID-hex shape used previously.
- A unit test calling `spawn_worker_impl(session_name="my-session")` confirms the session name is `my-session` exactly.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures.

## Demo
Run `poetry run pytest tests/test_engine_spawn_worker.py -v -k "session_name"`; observe the prefix and pass-through assertions pass with `spawn-` prefixes. Then `poetry run pytest` overall is green at 212.
