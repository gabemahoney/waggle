## Context
The `spawn_worker_impl` function in `claude_spawn.spawn` generates a default tmux session name when callers pass `session_name=None`. The current generator uses the literal prefix `waggle-`. This subtask flips the prefix to `spawn-`. Caller-provided session names continue to pass through verbatim — that branch of `spawn_worker_impl` is not modified.

## What Needs to Change
- In `src/claude_spawn/spawn.py`, locate the default-session-name f-string in `spawn_worker_impl` (the one of the form `f"waggle-{instance_id[:8]}"`) and change the literal prefix from `"waggle-"` to `"spawn-"`. Leave the `{instance_id[:8]}` suffix and the surrounding control flow untouched.
- DO NOT modify the "waggle-owned workers" docstring prose elsewhere in `spawn.py` (Epic 7 prose sweep).
- DO NOT modify the caller-provided `session_name` pass-through branch.

## Key Files
- `src/claude_spawn/spawn.py` — single string-literal change inside `spawn_worker_impl`.

## Acceptance Criteria
- `grep -n 'f"waggle-' src/claude_spawn/spawn.py` returns no matches.
- `grep -n 'f"spawn-' src/claude_spawn/spawn.py` returns exactly one match (the default-session-name expression in `spawn_worker_impl`).
- A unit test calling `spawn_worker_impl(session_name=None)` confirms the resulting session name starts with `spawn-` and the suffix matches the existing UUID-hex shape (8 hex chars).
- A unit test calling `spawn_worker_impl(session_name="my-session")` confirms the session name is `my-session` exactly.
