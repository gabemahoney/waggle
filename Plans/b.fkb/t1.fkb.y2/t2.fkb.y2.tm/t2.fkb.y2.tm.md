# Task 1: Rename default tmux session prefix from `waggle-` to `spawn-`

## Context
After Epics 1–3, the package and label namespace use `claude_spawn`. The default tmux session-name generator in `claude_spawn.spawn.spawn_worker_impl` still produces names like `waggle-<first8>`. This Task flips the prefix to `spawn-<first8>` (shorter than the long `claude_spawn_*` label prefix because tmux session names are user-visible and brevity wins, per the PRD identifier-table reasoning).

## What Needs to Change
- `src/claude_spawn/spawn.py`: in `spawn_worker_impl`, change the default-session-name f-string from `f"waggle-{instance_id[:8]}"` to `f"spawn-{instance_id[:8]}"`. Caller-provided `session_name` continues to pass through verbatim — that path is unchanged.
- `tests/test_engine_spawn_worker.py`: flip every `waggle-` prefix assertion to `spawn-` in the default-session-name tests, including comment lines that document the prefix.
- Verify the suite still reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure.

## Why
Tmux session names are user-visible; `spawn-` reads as Claude-Spawn-flavored without dragging the longer `claude_spawn` form into a space where brevity matters. The longer `claude_spawn_*` label-key prefix is reserved for programmatic surfaces (labels, env vars) where explicitness wins.

## Out of scope (deferred — DO NOT TOUCH)
- "waggle-owned workers" docstring prose in `spawn.py` (lines 222 and 329) → Epic 7 prose sweep.
- Caller-provided `session_name` pass-through behavior — unchanged.

## Success Criteria
- `grep -n "waggle-" src/claude_spawn/spawn.py` returns only the docstring-prose hits ("waggle-owned workers"); no `f"waggle-{instance_id[:8]}"` remains.
- `spawn_worker_impl(session_name=None)` produces a session name matching `^spawn-[0-9a-f]{8}$`.
- `spawn_worker_impl(session_name="my-session")` produces a session name equal to `"my-session"` verbatim.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures.
