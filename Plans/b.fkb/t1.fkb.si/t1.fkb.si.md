# Epic 5: Rename MCP server's advertised name

## Outcome
The `serverInfo.name` advertised by the stdio MCP server in its `initialize` response flips from `waggle-stdio` to `claude-spawn-stdio`. No other MCP surface changes (tool names, parameter schemas, error envelope, scrollback bounds, send-input semantics, single-question constraint, visibility check — all unchanged).

## Scope

### Source change in `src/claude_spawn/mcp_stdio.py`
- The string literal `"waggle-stdio"` advertised as `serverInfo.name` in the `initialize` response handler becomes `"claude-spawn-stdio"`.

### Test change in `tests/test_mcp_stdio_server.py`
- The assertion that exercises the `initialize` handshake and reads back `result.serverInfo.name` updates from `"waggle-stdio"` to `"claude-spawn-stdio"`.

## Acceptance Criteria

- `grep -RIn "waggle-stdio" src tests` returns no matches.
- `grep -RIn "claude-spawn-stdio" src tests` returns at least the source-side literal and the test-side assertion.
- A unit test sends a synthetic MCP `initialize` request and confirms the response's `serverInfo.name` is `"claude-spawn-stdio"`.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures.

## Demo
Run `poetry run pytest tests/test_mcp_stdio_server.py -v -k "initialize"`; observe the assertion for the advertised server name passes. Then `poetry run pytest` overall is green at 212.
