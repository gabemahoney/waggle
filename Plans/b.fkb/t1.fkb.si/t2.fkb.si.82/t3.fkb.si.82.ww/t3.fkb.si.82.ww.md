## Context
The stdio MCP server constructed in `src/claude_spawn/mcp_stdio.py` advertises its identity to clients via the string passed to `FastMCP(...)`. Currently that literal is `"waggle-stdio"`. Per the PRD identifier table and SRD SR-2.1, it flips to `"claude-spawn-stdio"`. Additionally, the `list_spawned_workers` MCP-tool docstring embeds a literal example of the `claude-status workers` argv used to find owned workers; that example string was `--label waggle_owned=1` and now must read `--label claude_spawn_owned=1` to match the actual `claude_status.workers(label=...)` call after Epic 3.

## What Needs to Change
- In `src/claude_spawn/mcp_stdio.py`, change the `FastMCP(...)` constructor argument from the literal `"waggle-stdio"` to `"claude-spawn-stdio"`.
- In the same file, in the docstring for the `list_spawned_workers` MCP tool, change the embedded example `--label waggle_owned=1` to `--label claude_spawn_owned=1`.
- DO NOT touch module-level docstring lines that say "Waggle stdio MCP server", `(waggle.server)`, `(``waggle mcp``)`, "Waggle-owned workers", "Waggle process restarts" — those prose mentions are handled by Epic 2 Subtask D (`t3.fkb.tf.ev.ik`).
- DO NOT touch other MCP surface behavior (tool names, parameter schemas, error envelopes, scrollback bounds, send-input semantics, single-question constraint, visibility check). The MCP surface contract is unchanged.

## Key Files
- `src/claude_spawn/mcp_stdio.py` — `FastMCP(...)` constructor literal AND `list_spawned_workers` tool docstring example.

## Acceptance Criteria
- `grep -n '"waggle-stdio"' src/claude_spawn/mcp_stdio.py` returns no matches.
- `grep -n '"claude-spawn-stdio"' src/claude_spawn/mcp_stdio.py` returns exactly one match (the `FastMCP(...)` instantiation).
- `grep -n "waggle_owned=1" src/claude_spawn/mcp_stdio.py` returns no matches.
- `grep -n "claude_spawn_owned=1" src/claude_spawn/mcp_stdio.py` returns at least one match (the docstring example).
- After the change, `import claude_spawn.mcp_stdio as ms; ms.mcp.name == "claude-spawn-stdio"` evaluates True.
- The MCP tool registration count remains exactly 6 (`spawn_worker`, `list_spawned_workers`, `send_input`, `get_output`, `terminate_worker`, `answer_question`).
