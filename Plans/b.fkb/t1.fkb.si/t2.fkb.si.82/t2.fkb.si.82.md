# Task 1: Rename MCP server's advertised name from `waggle-stdio` to `claude-spawn-stdio`

## Context
After Epics 1–4, the package, label keys, env-var prefix, and tmux session prefix are renamed. The stdio MCP server still advertises `serverInfo.name = "waggle-stdio"` (because `mcp_stdio.py` constructs `FastMCP("waggle-stdio")`). This Task flips that literal to `claude-spawn-stdio`. Additionally, the `list_spawned_workers` MCP-tool docstring in `mcp_stdio.py` embeds the literal `--label waggle_owned=1` as a documentation example; after Epic 3 the actual `claude-status workers` argv uses `--label claude_spawn_owned=1`, so the docstring example is updated in lockstep to keep it factually accurate.

## What Needs to Change
- `src/claude_spawn/mcp_stdio.py`: change the `FastMCP("waggle-stdio")` instantiation literal to `FastMCP("claude-spawn-stdio")`.
- `src/claude_spawn/mcp_stdio.py`: in the `list_spawned_workers` MCP-tool docstring, change the embedded example `--label waggle_owned=1` to `--label claude_spawn_owned=1`.
- `tests/test_mcp_stdio_server.py`: add a unit test asserting that the constructed FastMCP instance's `name` attribute equals `"claude-spawn-stdio"` after rename.
- Verify the suite reports the expected count (post-Epic: 213 collected, due to the one new test added by this Epic).

## Why
Tool names (`spawn_worker`, `list_spawned_workers`, etc.) stay because orchestrators bind to them, but the server identity itself is project-branded and must reflect the new project name. The embedded docstring example is renamed because Epic 3 flipped the actual filter argv; keeping the docstring stale would be a documentation lie.

## Out of scope (deferred — DO NOT TOUCH)
- Module docstring lines in `mcp_stdio.py` that say "Waggle stdio MCP server", `(waggle.server)`, `(``waggle mcp``)`, "Waggle-owned workers", "Waggle process restarts" — those are prose mentions handled by Epic 2 Subtask D (source-side prose pass).
- `_WAGGLE_PATTERN` symbol references (Epic 6).

## Success Criteria
- `grep -RIn "waggle-stdio" src tests` returns no matches.
- `grep -RIn "claude-spawn-stdio" src tests` returns at least two matches: the `FastMCP(...)` literal in `mcp_stdio.py` and the assertion in `tests/test_mcp_stdio_server.py`.
- `grep -n "waggle_owned=1" src/claude_spawn/mcp_stdio.py` returns no matches.
- A unit test confirms `ms.mcp.name == "claude-spawn-stdio"` after the rename.
- `poetry run pytest` reports 213 collected, 212 passed, 1 skipped, 1 pre-existing failure (one new test added; the pre-existing environmental failure is unchanged). NO new failures from this Task.
