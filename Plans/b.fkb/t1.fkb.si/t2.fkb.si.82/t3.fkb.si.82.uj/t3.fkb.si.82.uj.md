## Context
The Engineer is flipping the `FastMCP(...)` instantiation literal in `mcp_stdio.py` from `"waggle-stdio"` to `"claude-spawn-stdio"`. The test suite currently has no assertion exercising the MCP server's advertised name. Per Epic AC and SRD SR-2.1, a unit test must confirm the renamed name. This subtask adds the test.

## What Needs to Change
- In `tests/test_mcp_stdio_server.py`, add ONE new test method asserting that the constructed FastMCP instance's name attribute equals `"claude-spawn-stdio"`. Place it in the existing `TestToolRegistration` class (which already references `ms.mcp`) or create a dedicated `TestServerName` class — either is acceptable. The method name should be something like `test_server_name_is_claude_spawn_stdio`.
- The assertion can read `ms.mcp.name` directly (the FastMCP attribute) — a full synthetic MCP `initialize` handshake is NOT required; the attribute check is sufficient.
- DO NOT modify existing tool-registration assertions (`test_exactly_six_tools_registered`, `test_all_expected_tools_registered`) — their expected count of 6 is unchanged.
- DO NOT remove any existing tests.

## Key Files
- `tests/test_mcp_stdio_server.py` — add one new test method asserting the server name.
