# Epic 2: Rename Python package, CLI binary, distribution metadata

## Outcome
The Python package directory, distribution metadata, CLI binary name, and every internal/test Python identifier flip from `Waggle` to `Claude Spawn`. The data-string surfaces (label keys, MCP server name, sting pattern, tmux session prefix, env-var names, documentation prose) are intentionally left for later Epics — this Epic touches identifiers, not values.

## Scope

### Package and distribution
- Move `src/waggle/` → `src/claude_spawn/` (preserving all module files: `__init__.py`, `cli.py`, `mcp_stdio.py`, `spawn.py`, `claude_status.py`, `sting.py`, `installer.py`).
- Update `pyproject.toml`:
  - `[tool.poetry] name` → `claude-spawn`
  - `[tool.poetry] description` → remove "Waggle" token (re-word generically; "Claude Spawn" prose is acceptable).
  - `[tool.poetry] packages` → `[{include = "claude_spawn", from = "src"}]`
  - `[tool.poetry.scripts]` → `claude-spawn = "claude_spawn.cli:main"` (drop the legacy `waggle` script entry).
- Update `src/claude_spawn/__init__.py` docstring/identifier references.

### Internal imports and patch targets
- Update every `from waggle.X import ...`, `import waggle.X`, and `import waggle` across the source tree to `claude_spawn.*`.
- Update every `unittest.mock.patch(...)` call in `tests/` whose target string starts with `"waggle."` to use `"claude_spawn."` (e.g. `patch("waggle.installer._run_install_hooks")` → `patch("claude_spawn.installer._run_install_hooks")`).
- Update `tests/helpers.py` `_SEAM` constant from `"waggle.claude_status._run"` to `"claude_spawn.claude_status._run"`.

### Python-only identifiers (rename, not value changes)
- `WaggleArgumentParser` → `ClaudeSpawnArgumentParser` in `src/claude_spawn/cli.py` (and every usage site).
- argparse top-level `prog` → `claude-spawn`.

### Out of scope for this Epic (handled later)
- Label keys (`waggle_owned`, `waggle_session_name`, etc.) — Epic 3.
- Env-var prefix (`CLAUDE_STATUS_LABEL_WAGGLE_*`) — Epic 3.
- Tmux session prefix (`waggle-<first8>`) — Epic 4.
- MCP `serverInfo.name = "waggle-stdio"` — Epic 5.
- `_WAGGLE_PATTERN` and `_key_matches_waggle` / `_has_waggle_in_mcp_servers` helpers — Epic 6.
- README, docs/architecture, docs/spikes, docs/migration prose — Epic 7.

The above unrenamed surfaces are referenced internally by the renamed package; tests for them continue to pass because the data values are untouched.

## Acceptance Criteria

- `python -c "import claude_spawn"` succeeds in the project virtualenv.
- `python -c "import waggle"` raises `ModuleNotFoundError`.
- After `poetry install`, the executable `claude-spawn` is on `PATH`; `which waggle` returns nothing.
- `claude-spawn --help` exits 0.
- `grep -RIn "from waggle\\|import waggle\\|patch(\"waggle\\.\\|\"waggle\\." src tests` returns no matches (modulo data strings such as `"waggle_owned"` that are deliberately deferred).
- `pyproject.toml` shows `name = "claude-spawn"`, `packages = [{include = "claude_spawn", from = "src"}]`, and `[tool.poetry.scripts]` contains exactly one entry: `claude-spawn = "claude_spawn.cli:main"`.
- `WaggleArgumentParser` is not defined anywhere; `ClaudeSpawnArgumentParser` is its replacement and is used by `cli.py`.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures.

## Demo
Run `poetry install && claude-spawn --help && python -c "import claude_spawn; import waggle"`; expect the help output for `claude-spawn` and a `ModuleNotFoundError` on `import waggle`. Then `poetry run pytest` is green at 212.
