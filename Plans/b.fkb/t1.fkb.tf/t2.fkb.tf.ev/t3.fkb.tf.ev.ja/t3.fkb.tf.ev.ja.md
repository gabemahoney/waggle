## Context
The CLI parser class `WaggleArgumentParser` and the argparse `prog="waggle"` define the user-facing CLI identity. Per PRD identifier table and SRD SR-3.1, SR-3.2, SR-3.3, the class is renamed, `prog` flips, and all top-level + per-subcommand help/description strings + option help (including `--auq-order`) become Claude-Spawn-voiced. The argparse subclass's behavior — emit `{"status": "error", "message": ...}` to stdout and exit 2 on usage errors — is unchanged.

The `--auq-order` help text currently uses literal example values `before:waggle` and `after:waggle`; SR-3.3 forbids the help text from referencing `waggle`. Drop those literal examples or replace with neutral placeholders (e.g. `before:<other>` / `after:<other>`); do NOT switch them to `before:claude-spawn` if that misrepresents the actual `claude-status install-hooks` accepted format.

## What Needs to Change
- In `src/claude_spawn/cli.py`: rename `WaggleArgumentParser` → `ClaudeSpawnArgumentParser` at the class definition and the single instantiation site in `main()`.
- Set the top-level argparse `prog` argument from `"waggle"` to `"claude-spawn"`.
- Update the top-level argparse `description` so it names Claude Spawn (not Waggle).
- Update every subparser's `help=` and `description=` for the three subcommands `sting`, `mcp`, `install` so any literal "Waggle"/"waggle" reference becomes "Claude Spawn"/`claude-spawn`.
- Update the `--auq-order` option help text: remove or replace the literal `before:waggle` / `after:waggle` example values per SR-3.3.
- Update inline source comments (e.g. `# waggle sting`, `# waggle mcp`, `# waggle install`) to the new name.
- Do NOT touch any other module. Do NOT touch the deferred data strings (label keys, MCP server name, sting pattern, tmux prefix, env-var fragments).
