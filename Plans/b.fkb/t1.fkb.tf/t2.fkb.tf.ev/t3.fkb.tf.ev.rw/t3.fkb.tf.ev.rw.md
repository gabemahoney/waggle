## Context
The Engineer is moving the package directory and renaming Python identifiers in `src/`. Tests must flip their `import` statements, `unittest.mock.patch` target strings, and the `_SEAM` constant in `tests/helpers.py` from `waggle.*` to `claude_spawn.*` in the same commit, or pytest collection raises `ModuleNotFoundError`. This subtask owns ALL test-side path edits.

## What Needs to Change
- Flip the package prefix `waggle` → `claude_spawn` in every `import waggle.X` / `from waggle.X import ...` / `from waggle import X` statement in the 12 listed test files (including any in-test `import waggle.X as _mod` and `importlib.reload(waggle.X)` calls).
- Flip every `unittest.mock.patch("waggle.X.Y", ...)` target string to `"claude_spawn.X.Y"`, including multi-line patch decorators where the target is on a continuation line.
- Update `tests/helpers.py` `_SEAM` constant from `"waggle.claude_status._run"` to `"claude_spawn.claude_status._run"`.
- In `tests/test_package_entry_point.py`: flip the entry-point *name* assertion from `"waggle"` to `"claude-spawn"` (hyphenated CLI binary name); flip the entry-point *value* assertion from `"waggle.cli:"` startswith to `"claude_spawn.cli:"` (underscore Python module path); flip the `shutil.which("waggle")` / fallback `"waggle"` binary name to `"claude-spawn"`. Test class/method names containing `waggle` in this file are renamed for consistency.
- In `tests/test_sting.py`: flip ONLY the module prefix `waggle.sting` → `claude_spawn.sting` in the `from ... import (...)` block. Preserve the symbol names `_WAGGLE_PATTERN`, `_has_waggle_in_mcp_servers`, `_key_matches_waggle` exactly — those symbol renames belong to Epic 6.
