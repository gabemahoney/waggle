# Task 1: Rename Python package, CLI binary, distribution metadata, and Python-identifier surface

## Context
The codebase is at the end of Epic 1 (`t1.fkb.vx`): dead daemon-era files/dirs and the `_CLI_REFERENCE` constant + `TestCliReference` test class are gone. Test suite reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure (`tests/spikes/test_ask_relay_spike.py::TestAskRelayEndToEnd::test_sentinel_injected_via_hook`). The package directory is still `src/waggle/` and the CLI binary is still `waggle`.

This Task atomically renames the Python package, CLI binary, distribution metadata, and Python-only identifier surface in a single commit. The package cannot be partially renamed — tests fail at collection until both source and test sides reference the new package path.

## What Needs to Change
- Move `src/waggle/` → `src/claude_spawn/` using `git mv` to preserve history; update every internal cross-module import in `src/`.
- Update `pyproject.toml`: `[tool.poetry] name = "claude-spawn"`, `packages = [{include = "claude_spawn", from = "src"}]`, `[tool.poetry.scripts]` becomes exactly one entry `claude-spawn = "claude_spawn.cli:main"` (drop the legacy `waggle` key). Confirm `description` contains no case-insensitive "waggle" substring (current text already does not).
- Rewrite `src/claude_spawn/__init__.py` docstring with the Claude Spawn voice; preserve the `__version__ = "0.1.0"` attribute.
- Rename `WaggleArgumentParser` → `ClaudeSpawnArgumentParser` in `cli.py` and every usage site; set argparse `prog="claude-spawn"`; update top-level + per-subcommand `description`/`help` strings; update option help including `--auq-order` (drop or replace literal `before:waggle` / `after:waggle` example values).
- Rewrite prose mentions of "Waggle"/"waggle" in source-tree docstrings, log strings, error strings, and comments to use "Claude Spawn"/`claude-spawn` — EXCLUDING data-value literals that are deferred to later Epics (see below).
- Update test-side imports, `unittest.mock.patch` target strings, and the `_SEAM` constant in `tests/helpers.py` from `waggle.*` to `claude_spawn.*`.
- Verify the suite still reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure.

## Why
Foundational identifier rename. Without it, no further Epic can land — env-var/label changes, MCP server name changes, sting pattern changes, and doc rewrites all assume the new package name and binary name.

## Out of scope (deferred to later Epics — do NOT touch)
- Label-key strings `waggle_owned`, `waggle_session_name`, `waggle_model`, `waggle_repo` anywhere → Epic 3 (`t1.fkb.zr`)
- Env-var fragments `CLAUDE_STATUS_LABEL_WAGGLE_*` in `_REQUIRED_ENV_VARS` and elsewhere → Epic 3
- `--label waggle_owned=1` argv string → Epic 3
- Default tmux session-name prefix literal `"waggle-"` (the `f"waggle-{instance_id[:8]}"` form) → Epic 4 (`t1.fkb.y2`)
- MCP `FastMCP("waggle-stdio")` / `serverInfo.name` literal → Epic 5 (`t1.fkb.si`)
- Regex pattern `(?i)(?:^|[-_])waggle(?:$|[-_])` and the symbol names `_WAGGLE_PATTERN`, `_key_matches_waggle`, `_has_waggle_in_mcp_servers` → Epic 6 (`t1.fkb.jg`)
- `README.md`, `docs/architecture/`, `docs/spikes/`, `docs/migration.md` prose → Epic 7 (`t1.fkb.xy`)

## Success Criteria
- `python -c "import claude_spawn"` succeeds; `python -c "import waggle"` raises `ModuleNotFoundError`.
- After `poetry install`, `claude-spawn` is on PATH; `which waggle` returns nothing.
- `claude-spawn --help` exits 0 and prints `usage: claude-spawn ...`.
- `grep -RIn "from waggle\b\|import waggle\b" src tests` returns no matches.
- `grep -RIn 'patch("waggle\.\|"waggle\.' src tests` returns no matches on live `import` / `from` / `patch(` / `_SEAM =` lines (data-string occurrences like `"waggle_owned"` that are deliberately deferred are acceptable).
- `pyproject.toml` declares `name = "claude-spawn"`, `packages = [{include = "claude_spawn", from = "src"}]`, and `[tool.poetry.scripts]` contains exactly one entry `claude-spawn = "claude_spawn.cli:main"`.
- `WaggleArgumentParser` is not defined anywhere; `ClaudeSpawnArgumentParser` replaces it.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures.
