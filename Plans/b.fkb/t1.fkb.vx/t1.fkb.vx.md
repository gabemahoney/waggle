# Epic 1: Remove dead daemon-era code

## Outcome
Delete files and code passages that target the removed daemon surface (`waggle serve`, `waggle set-state`, `waggle permission-request`, `waggle ask-relay`, the `waggle.service` systemd unit, the `~/.waggle/hooks/` path) and have no replacement value in the stdio-MCP era. The tree post-Epic reflects only the live stdio-MCP surface.

## Scope

Delete these paths outright:

- `install.sh` — installs the removed `waggle.service` systemd unit.
- `docker/Dockerfile`, `docker/entrypoint.sh`, `docker/phase1.sh`, `docker/phase2.sh`, `docker/auto_approve.sh`, then the now-empty `docker/` directory.
- `plugins/opencode/state-tracker.ts`, then the now-empty `plugins/opencode/` and `plugins/` directories.
- `.claude/skills/ci/` (directory + `SKILL.md`).
- `.claude/skills/release-test/` (directory + `SKILL.md`).

Delete these code passages:

- The `_CLI_REFERENCE` module-level constant in `src/waggle/sting.py`.
- The `TestCliReference` test class in `tests/test_sting.py` (sole consumer of `_CLI_REFERENCE`).
- Any `from waggle.sting import _CLI_REFERENCE` (or equivalent) import that becomes dangling in `tests/test_sting.py` once the class is removed.

The `_remove_legacy_hook_template()` function in `waggle.installer` and its `TestLegacyTemplateRemoval` tests stay — those clean up operator-side state and are not daemon-era surface.

## Acceptance Criteria

- The following paths do not exist in the working tree: `install.sh`, `docker/`, `plugins/`, `.claude/skills/ci/`, `.claude/skills/release-test/`.
- `grep -n '_CLI_REFERENCE' src/waggle/sting.py tests/test_sting.py` returns nothing.
- `grep -n 'class TestCliReference' tests/test_sting.py` returns nothing.
- `poetry run pytest --collect-only` exits 0 with no `ImportError` or `ModuleNotFoundError`.
- `poetry run pytest` reports 212 collected (down from 213 due to the deleted `test_contains_all_subcommands`), 211 passed, 1 skipped, 1 pre-existing failure (`tests/spikes/test_ask_relay_spike.py::TestAskRelayEndToEnd::test_sentinel_injected_via_hook`). No new failures introduced.

Run `poetry run pytest` from the repo root after the Epic lands and observe the green test count drop from 213 to 212; observe `ls install.sh docker plugins .claude/skills/ci .claude/skills/release-test 2>&1 | grep -c "No such"` returns 5.
