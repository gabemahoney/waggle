# Task 1: Remove dead daemon-era files, directories, and code passages

## Context
The repo carries stale files, dirs, and code passages targeting removed daemon surface (`waggle serve`, `waggle.service` systemd unit, `~/.waggle/hooks/`, docker CI flow). The Epic deletes them as a single coherent dead-code-removal commit. Renaming them mechanically would attach the new project name to obviously-broken artifacts; deletion is cheaper and more honest.

## What Needs to Change
- Delete the five filesystem paths: `install.sh`, `docker/`, `plugins/`, `.claude/skills/ci/`, `.claude/skills/release-test/`.
- Delete the `_CLI_REFERENCE` constant from `src/waggle/sting.py`.
- Delete the `TestCliReference` class and the now-dangling `_CLI_REFERENCE` import from `tests/test_sting.py`.
- Verify `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure (the environmental `test_sentinel_injected_via_hook` in `tests/spikes/test_ask_relay_spike.py`).

`_remove_legacy_hook_template()` in `src/waggle/installer.py` and its `TestLegacyTemplateRemoval` tests STAY — they clean up operator-side state and are not daemon-era surface. The `waggle` identifier rename is OUT OF SCOPE for this Epic and Task — it is owned by Epic `t1.fkb.tf` onward.

## Why
Subtractive cleanup of dead surface so the tree post-cutover reflects only the live stdio-MCP surface. Without this Task, Epic 2 (the package rename) would attach the new `claude_spawn` identifier to artifacts that already target removed daemon commands — a much worse state.

## Success Criteria
- The five filesystem paths do not exist in the working tree.
- `grep -n '_CLI_REFERENCE' src/waggle/sting.py tests/test_sting.py` returns nothing.
- `grep -n 'class TestCliReference' tests/test_sting.py` returns nothing.
- `poetry run pytest --collect-only` exits 0 with no `ImportError` / `ModuleNotFoundError`.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures introduced.
