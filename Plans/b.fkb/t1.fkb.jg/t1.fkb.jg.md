# Epic 6: Rename sting MCP-detection pattern and helpers

## Outcome
The Claude Spawn sting subcommand detects its MCP-server registration in `~/.claude.json` and `~/.claude/settings.json` using a `claude[-_]spawn` regex pattern and helper functions named after the new project. The legacy `_WAGGLE_PATTERN` symbol and its helper functions cease to exist.

## Scope

### Source changes in `src/claude_spawn/sting.py`
- Rename module-level constant `_WAGGLE_PATTERN` → `_CLAUDE_SPAWN_PATTERN`.
- Update the compiled regex from `(?i)(?:^|[-_])waggle(?:$|[-_])` to `(?i)(?:^|[-_])claude[-_]spawn(?:$|[-_])`.
- Rename helper `_key_matches_waggle(key)` → `_key_matches_claude_spawn(key)`.
- Rename helper `_has_waggle_in_mcp_servers(mcp_servers)` → `_has_claude_spawn_in_mcp_servers(mcp_servers)`.
- Update every internal call site in `sting.py` to use the renamed symbols.

### Test changes in `tests/test_sting.py`
- Rename the pattern-matching test class `TestWagglePattern` → `TestClaudeSpawnPattern`.
- Rebuild the positive-match cases to cover: `claude-spawn`, `claude_spawn`, `my-claude-spawn`, `CLAUDE-SPAWN`, `claude_spawn_server`, `Claude-Spawn`.
- Rebuild the negative-match cases to cover: `claudespawn` (no separator), `claude-spawnfish` (no trailing separator), `myclaudespawn` (no leading separator), `claude-spawned` (no separator at end of stem).
- Update the `TestDetectMcp` JSON fixtures so positive-detect cases use `claude-spawn`-rooted `mcpServers` keys and negative-detect cases use unrelated keys.
- Update any test that imports or patches `_WAGGLE_PATTERN` / `_key_matches_waggle` / `_has_waggle_in_mcp_servers` to use the new names.

## Acceptance Criteria

- `grep -RIn "_WAGGLE_PATTERN\\|_key_matches_waggle\\|_has_waggle_in_mcp_servers" src tests` returns no matches.
- `grep -RIn "_CLAUDE_SPAWN_PATTERN" src/claude_spawn/sting.py` returns at least the definition and the helper bodies.
- The renamed pattern matches the six positive cases enumerated above; rejects the four negative cases enumerated above.
- `TestClaudeSpawnPattern` exists in `tests/test_sting.py` and exercises the positive and negative cases.
- `poetry run pytest tests/test_sting.py` is fully green.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures.

## Demo
Run `poetry run pytest tests/test_sting.py::TestClaudeSpawnPattern -v` and observe positive/negative match cases pass. Then `poetry run pytest tests/test_sting.py::TestDetectMcp -v` and observe MCP-detection cases pass against `claude-spawn`-rooted fixture keys. Then `poetry run pytest` overall is green at 212.
