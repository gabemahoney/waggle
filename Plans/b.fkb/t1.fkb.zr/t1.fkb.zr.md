# Epic 3: Rename env-var prefix and Claude Status label keys

## Outcome
The env vars that Claude Spawn injects into worker tmux sessions, and the resulting Claude Status `instances.labels` keys, switch from the `waggle_*` namespace to the `claude_spawn_*` namespace. The worker-filter argv that `list_spawned_workers` and `answer_question` send to `claude-status workers` flips to `--label claude_spawn_owned=1`. Test fixtures and assertions are updated in lockstep.

## Scope

### Source changes in `src/claude_spawn/spawn.py`
- `_REQUIRED_ENV_VARS` tuple — replace the four label entries with:
  - `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_OWNED`
  - `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_SESSION_NAME`
  - `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_MODEL`
  - `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_REPO`
  Order within the tuple stays stable (SR-4.2).
- `spawn_worker_impl` — update the `-e KEY=val` argv that flow to `tmux new-session` so the four label env vars use the new names.
- `list_spawned_workers_impl` — call `claude-status workers --label claude_spawn_owned=1` (via the `workers(label=...)` client).
- `answer_question_impl` — same `--label claude_spawn_owned=1` filter; read `labels.claude_spawn_session_name` to resolve the tmux session.
- Stderr log-line prose may reference "Claude Spawn" where it previously said "Waggle" — log shape unchanged.

### Test changes
- `tests/helpers.py` `_DEFAULT_LABELS` — rotate keys to `claude_spawn_owned`, `claude_spawn_session_name`, `claude_spawn_model`, `claude_spawn_repo` (values unchanged).
- `tests/sample_payloads.py` — every worker record dict (`WORKER_WORKING`, `WORKER_ASK_USER_SINGLE`, `WORKER_ASK_USER_MULTI`, `WORKER_ENDED`, and any other with well-formed labels) uses the new keys. `WORKER_MALFORMED_LABELS` retains its intentionally malformed shape.
- `tests/test_engine_spawn_worker.py` — assertions on env-var names (e.g. `test_all_seven_env_vars_present`) iterate `_REQUIRED_ENV_VARS`; any hardcoded `WAGGLE`-prefixed env-var strings update to `CLAUDE_SPAWN`.
- `tests/test_engine_list_spawned_workers.py`, `tests/test_claude_status_client.py` — argv-assertions for `--label` switch from `"waggle_owned=1"` to `"claude_spawn_owned=1"`.
- `tests/test_engine_answer_question.py` — fixture labels and any direct key lookups switch to `claude_spawn_session_name`.
- `tests/test_helpers_smoke.py`, `tests/test_sample_payloads.py` — defaults-label assertions iterate the new key set.

## Acceptance Criteria

- `grep -RIn "WAGGLE\\|waggle_owned\\|waggle_session_name\\|waggle_model\\|waggle_repo" src tests` returns no matches.
- `grep -RIn "CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_OWNED" src/claude_spawn/spawn.py` returns the four expected env-var name hits in `_REQUIRED_ENV_VARS` and the `spawn_worker_impl` body.
- A unit test exercising `spawn_worker_impl` shows the seven env vars passed to `tmux new-session`, with the four label env vars using the `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_*` prefix.
- `list_spawned_workers_impl` argv assertion expects `["workers", "--label", "claude_spawn_owned=1"]`.
- `answer_question_impl` argv assertion expects the same filter; the session-name resolution reads `labels.claude_spawn_session_name`.
- `poetry run pytest` reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure. No new failures.
