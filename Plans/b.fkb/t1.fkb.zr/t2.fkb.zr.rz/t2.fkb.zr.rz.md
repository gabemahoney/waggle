# Task 1: Rename env-var prefix and Claude Status label-key namespace

## Context
After Epics 1 + 2, the package lives at `src/claude_spawn/` and the CLI binary is `claude-spawn`, but the data strings inside `src/claude_spawn/spawn.py` still carry the legacy `waggle_*` namespace for the four worker label keys and their `CLAUDE_STATUS_LABEL_WAGGLE_*` injection env-var names. Test fixtures and assertions in `tests/` reference the same `waggle_*` strings.

This Task atomically flips the env-var prefix and label-key namespace in one commit. Source-side and test-side changes must land together because spawned workers would otherwise carry `claude_spawn_owned=1` labels but `list_spawned_workers` / `answer_question` would still filter on `waggle_owned=1` and return zero rows.

## What Needs to Change
- In `src/claude_spawn/spawn.py`:
  - `_REQUIRED_ENV_VARS` tuple — the four label env vars flip to `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_OWNED|SESSION_NAME|MODEL|REPO`. Tuple order is preserved (SR-4.2). The three non-label env vars (`CLAUDE_STATUS_INSTANCE_ID`, `CLAUDE_STATUS_RELAY_MODE`, `CLAUDE_STATUS_AUQ_MODE`) stay.
  - `spawn_worker_impl` — the four `-e KEY=val` argv pairs flowing to `tmux new-session` flip env-var names to match.
  - `list_spawned_workers_impl` — filter argv `label="waggle_owned=1"` → `label="claude_spawn_owned=1"`; `session_name` projection reads `labels.claude_spawn_session_name`.
  - `answer_question_impl` — filter argv `label="waggle_owned=1"` → `label="claude_spawn_owned=1"`; the session-name read flips to `labels.claude_spawn_session_name`.
- In test files (7 affected): `tests/helpers.py`, `tests/sample_payloads.py`, `tests/test_helpers_smoke.py`, `tests/test_claude_status_client.py`, `tests/test_engine_list_spawned_workers.py`, `tests/test_engine_answer_question.py`, `tests/test_engine_spawn_worker.py`. Updates include `_DEFAULT_LABELS` keys, worker-record label dicts (excluding `WORKER_MALFORMED_LABELS`), argv assertions, env-var-name assertions, and test method names that contain `waggle` (driven by the Epic AC `git grep -i waggle = 0` requirement, not by SR-12.7 which is Epic 6).
- Verify the suite still reports 212 collected, 211 passed, 1 skipped, 1 pre-existing failure.

## Why
Claude Status reads labels from the env vars Claude Spawn injects; the env-var prefix and the label-key namespace must change together. The `list_spawned_workers` and `answer_question` consumers filter by label; their argv string must match the new label-key namespace or workers become invisible to the new code paths.

- `FastMCP("waggle-stdio")` in `mcp_stdio.py` and the embedded `--label waggle_owned=1` docstring example in `list_spawned_workers` MCP-tool docstring → Epic 5 (`t1.fkb.si`).
- `_WAGGLE_PATTERN`, `_key_matches_waggle`, `_has_waggle_in_mcp_servers` symbols and `TestWagglePattern` class → Epic 6 (`t1.fkb.jg`).
- README + `docs/` prose, and the spawn.py docstring lines that mention "waggle-owned workers" or "waggle list_spawned_workers: ..." stderr log prefix → Epic 7 (`t1.fkb.xy`) prose sweep.
- `WORKER_MALFORMED_LABELS` in `tests/sample_payloads.py` — intentionally malformed labels; do not touch.
