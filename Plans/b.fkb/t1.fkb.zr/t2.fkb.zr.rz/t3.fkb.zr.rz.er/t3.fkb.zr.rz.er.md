## Context
The Engineer is flipping the env-var prefix and label-key namespace in `src/claude_spawn/spawn.py` in the same commit as this subtask. Test fixtures and assertions in seven test files must flip the same data strings or assertions on the new source code will fail. The transformation is uniform: `waggle_<suffix>` label-key → `claude_spawn_<suffix>` label-key; `CLAUDE_STATUS_LABEL_WAGGLE_*` env-var name → `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_*`; `"waggle_owned=1"` argv → `"claude_spawn_owned=1"`. Test method names that contain `waggle` rename for consistency, driven by the Epic AC `git grep -i waggle = 0` requirement (PRD "Test files" section).

## What Needs to Change
Across the seven affected test files, flip every `waggle_*` label-key data string to `claude_spawn_*`, every `CLAUDE_STATUS_LABEL_WAGGLE_*` env-var name string to `CLAUDE_STATUS_LABEL_CLAUDE_SPAWN_*`, and every test method name that contains `waggle` to the equivalent `claude_spawn` form. Specific surfaces:

- `tests/sample_payloads.py` — in each of the four well-formed worker records (`WORKER_WORKING`, `WORKER_ASK_USER_SINGLE`, `WORKER_ASK_USER_MULTI`, `WORKER_ENDED`), rename the four label-dict keys. **DO NOT modify `WORKER_MALFORMED_LABELS` — its `labels` field is intentionally malformed and contains no `waggle_*` keys.**
- `tests/helpers.py` — rename the four keys of `_DEFAULT_LABELS`. Update any docstring/example that demonstrates a `fake_claude_status` argv containing `waggle_owned=1` to `claude_spawn_owned=1`.
- `tests/test_helpers_smoke.py` — rename the four-element label-key tuple iterator and the two sites that construct or assert on a `--label waggle_owned=1` argv.
- `tests/test_claude_status_client.py` — flip the five sites that pass `label="waggle_owned=1"` to `cs.workers(...)` or assert argv equality / negative assertions against `waggle_owned=1` / `--label=waggle_owned=1`. Preserve existing test logic (including negative `not in argv` checks).
- `tests/test_engine_list_spawned_workers.py` — rename the test method `test_passes_label_waggle_owned` to `test_passes_label_claude_spawn_owned`. Flip the `waggle_owned=1` argv equality assertion and the `r["labels"]["waggle_session_name"]` label-write.
