# Task 1: Rewrite documentation to use Claude Spawn voice

## Context
After Epics 1–6, all source code, tests, env vars, label keys, tmux prefix, MCP server name, and sting pattern are renamed. The Markdown documentation in the working tree (`README.md`, `docs/architecture/*.md`, `docs/spikes/*.md`, `docs/migration.md`) still references "Waggle" / "waggle" throughout. This Task rewrites every Markdown file so `git grep -i waggle` (no path filter) returns zero matches across the entire working tree.

## What Needs to Change
- `README.md`: replace every "Waggle" / "waggle" occurrence with "Claude Spawn" / `claude-spawn` per the PRD identifier table. Installation snippet uses `pip install . && claude-spawn install`. MCP-registration example uses `claude-spawn` as the operator-chosen `mcpServers.<KEY>`. CLI-usage examples use `claude-spawn sting`, `claude-spawn mcp`, `claude-spawn install`. Architecture/module-layout section references `src/claude_spawn/*.py`.
- `docs/architecture/spawn_worker.md`, `docs/architecture/send_input.md`, `docs/architecture/terminate_worker.md`: replace every "Waggle" / "waggle" occurrence with the new project name and the corresponding identifier. The known pre-existing inaccuracy in `spawn_worker.md` (env-var list uses symbolic names like `WAGGLE_INSTANCE_ID` instead of the actual `CLAUDE_STATUS_*` names) is NOT corrected here — replace the `WAGGLE_` token with `CLAUDE_SPAWN_` so the doc no longer mentions Waggle, but leave the underlying inaccuracy alone (it is a follow-up bee).
- `docs/spikes/ask-relay-spike.md`, `docs/spikes/claude-channels-spike.md`: replace every "Waggle" / "waggle" occurrence with the new project name. Preserve `SUPERSEDED` annotations in spirit (only the name token changes). Prose like "<old-name> v2" rewords to "Claude Spawn v2" (or equivalent).
- `docs/migration.md`: special handling per SR-9.3 — see below.
- Verify the suite still reports 213 collected, 212 passed, 1 skipped, 1 pre-existing failure (this Task touches no code).

## Migration doc handling (SR-9.3)
- Zero occurrences of "Waggle" (case-insensitive).
- Voice uses the new project name; references to the prior name use oblique phrasing such as "this project was previously distributed under a different name."
- Historical filesystem paths that contained the old name as a literal substring (the daemon's `~/.<old-name>/` state directory and the `~/.config/systemd/user/<old-name>.service` systemd unit) are described generically (e.g. "the daemon's state directory under `$HOME`", "the daemon's user-mode systemd unit") — NOT re-spelled with the new name, because those paths never existed under the new name.
- The "What changed" table row that listed removed daemon CLI commands by name (e.g. `waggle serve`, `waggle set-state`, `waggle permission-request`, `waggle ask-relay`) is rephrased to describe the removed surface generically without naming the commands.

## Why
Final cleanup of the rename. The codebase's user-facing and contributor-facing documentation must match the renamed code. The migration doc requires special handling because re-spelling historical daemon paths under the new name would be a documentation lie.

## Out of scope
- Source code, tests, pyproject.toml — all already renamed in Epics 1–6.
- `docs/architecture/spawn_worker.md` env-var-list factual inaccuracy — only flip the `WAGGLE_` token to `CLAUDE_SPAWN_`; the inaccuracy itself is a follow-up bee.
