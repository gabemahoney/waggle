## Context
`docs/migration.md` describes the historical migration from the project's earlier HTTP-daemon distribution to the current stdio-MCP distribution. Unlike the other docs, it has special handling per SR-9.3 because some of its prose names historical filesystem paths (`~/.waggle/`, `~/.config/systemd/user/waggle.service`) that NEVER existed under the new project name. Re-spelling them as `~/.claude-spawn/` would be a documentation lie.

The doc is ~129 lines with ~20 case-insensitive `waggle` references.

## What Needs to Change
- Rewrite `docs/migration.md` to use the Claude Spawn voice throughout. Concrete rules per SR-9.3:
  - Zero occurrences of "Waggle" / "waggle" (case-insensitive) after the rewrite.
  - The doc's primary voice uses "Claude Spawn" / `claude-spawn` to describe the current surface.
  - Aside references to the prior project name use oblique phrasing such as "this project was previously distributed under a different name" or "the earlier daemon distribution." DO NOT name the old project literally.
  - Historical filesystem paths that contained the old name as a literal substring (`~/.waggle/`, `~/.config/systemd/user/waggle.service`) are described generically — for example "the daemon's state directory under `$HOME`" and "the daemon's user-mode systemd unit". DO NOT re-spell them with the new name (those paths never existed under the new name).
  - The "What changed" table row (or equivalent prose) that listed removed daemon CLI commands by name (`waggle serve`, `waggle set-state`, `waggle permission-request`, `waggle ask-relay`) is rephrased to describe the removed surface generically without naming the commands — e.g. "removed daemon-era subcommands for serve / state mutation / permission relay / question relay" or similar.
- DO NOT introduce new sections, new install instructions, or new content beyond what the current doc covers.

## Key Files
- `docs/migration.md` — full rewrite under the SR-9.3 rules.

## Acceptance Criteria
- `grep -i "waggle" docs/migration.md` returns zero matches.
- No string of the form `~/.waggle/` or `~/.claude-spawn/` appears in the doc — historical state-dir paths are described generically.
- No string of the form `~/.config/systemd/user/waggle.service` or `~/.config/systemd/user/claude-spawn.service` appears — the systemd unit is described generically.
- The doc's "What changed" table row (or equivalent prose) describing removed daemon CLI commands does NOT name any of `waggle serve`, `waggle set-state`, `waggle permission-request`, or `waggle ask-relay` literally.
- The doc's primary voice references "Claude Spawn" / `claude-spawn`; aside references to the prior name use oblique phrasing such as "previously distributed under a different name."
- The doc's overall structure (section headings) is preserved.
