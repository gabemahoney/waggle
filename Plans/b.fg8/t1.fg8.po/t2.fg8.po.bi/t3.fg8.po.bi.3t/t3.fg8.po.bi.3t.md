## Context

SR-2.3 requires every `claude-status` invocation to be a fresh subprocess with no retry layer. SR-7.1 requires every failure to be translated into a structured operation-failed shape rather than a raised exception. To make this uniform across the three verbs and testable under the Task-1 `fake_claude_status` helper, the client needs a single low-level invocation seam plus a stderr envelope parser, composed by a shared internal call helper. This Subtask delivers those three primitives.

**Patch-target FQN handshake with Task 1:** the seam's fully-qualified Python name is shared with `tests/helpers.py:fake_claude_status`. Pin the name (e.g. `waggle.claude_status._run`) in this Subtask's implementation; Task 1's helper must reference the same name. Renaming later silently breaks every sibling Epic's tests.

## What Needs to Change

Create `src/waggle/claude_status.py` and add:

- **Subprocess seam** — a module-level function taking a list of CLI arguments (e.g. `["workers", "--label", "waggle_owned=1"]`), invoking `claude-status` via `subprocess.run` with `capture_output=True`, `text=True`, and a bounded `timeout=10` seconds. Returns the `(stdout, stderr, exit_code)` triple. The seam propagates `FileNotFoundError` and `subprocess.TimeoutExpired`; the shared helper below translates them. No retry. No async. No `time.sleep`.
- **Stderr envelope parser** — a function that takes a non-zero-exit stderr string and parses the grammar `ERROR: <ErrName>: <description>
`. Returns `(err_name, description)` on a well-formed line with `err_name` preserved verbatim (no case folding, no prefix stripping). On a malformed line, returns a sentinel `err_name` distinct from the five documented `ErrName` values — pick a clear name (e.g. `ErrMalformedErrorEnvelope`) and document it in the module docstring.
- **Shared internal call helper** — a function that composes seam + parser + JSON parsing into a single result. On exit 0: `json.loads(stdout)` → operation-success containing the parsed payload and the verb name. On exit 0 with non-JSON stdout: operation-failed with `err_name=ErrPayloadMalformed` (the SR-2.3 typed name, since this represents a payload-malformed condition from Claude Status's perspective). On non-zero exit: parse stderr envelope → operation-failed preserving `err_name` verbatim. On `FileNotFoundError` from the seam: operation-failed with `err_name=ErrClaudeStatusNotFound` (Waggle-originated). On `subprocess.TimeoutExpired`: operation-failed with `err_name=ErrClaudeStatusTimeout` (Waggle-originated). The shared helper never raises.

The result shape follows the policy documented on the Task ticket (success/failure discriminator, `operation` field, `err_name` and `err_description` on failure, parsed payload on success). The implementer documents the landed shape in a module-level docstring on `claude_status.py` so Epics 2–5 build against a stable target. Waggle-originated err_names (`ErrClaudeStatusNotFound`, `ErrClaudeStatusTimeout`, `ErrMalformedErrorEnvelope`, `ErrContractVersionMismatch`) are documented alongside the five Claude-Status-originated ones with their origin noted.

**Import-inert:** `python -c 'import waggle.claude_status'` must perform no subprocess fork, no PATH lookup, and no file I/O.

## Key Files

- `src/waggle/claude_status.py` — new file containing the seam, parser, and shared call helper

## Acceptance Criteria

- The seam is a single module-level function whose FQN matches the patch target referenced by `tests/helpers.py:fake_claude_status`.
- The seam invokes `subprocess.run` exactly once per call, with `capture_output=True`, `text=True`, and `timeout=10` (or similar bounded value documented in the module docstring); the seam never retries.
- The envelope parser preserves `ErrName` verbatim and assigns the documented sentinel err_name to malformed input.
- The shared helper translates `FileNotFoundError` → operation-failed with the documented Waggle-originated err_name; translates `TimeoutExpired` → operation-failed with the documented Waggle-originated err_name; translates non-JSON stdout on exit 0 → operation-failed with `err_name=ErrPayloadMalformed`.
- `python -c 'import waggle.claude_status'` succeeds with `subprocess.run` patched to raise on call (proving zero subprocess invocations at import time).
- The module docstring documents the result shape (field names, value types) and enumerates all err_names with their origins.
- No `sqlite3`, `httpx`, or network module is imported. No `time.sleep`. No retry loop. No result caching.
