## Context

This Task adds the consumer-CLI client module `src/waggle/claude_status.py` and its unit tests. Waggle interacts with Claude Status exclusively through this client (SR-2.2). Per SR-2.3 each call is a fresh subprocess with no retry; per SR-7.1 every outcome is a structured result so callers in later Epics can wrap MCP tool bodies uniformly.

## What Needs to Change

- Add `src/waggle/claude_status.py` with: a module-level subprocess invocation seam (FQN pinned and shared with Task 1's `fake_claude_status` patch target); a stderr envelope parser per grammar `ERROR: <ErrName>: <description>
`; verb wrappers for `workers(label=None)`, `worker(instance_id)`, and `capabilities()`; a capability-pin helper that returns operation-failed when the parsed `contract_version` major != 1. Do NOT implement `decide` — Waggle never invokes it (SR-2.2).
- Add `tests/test_claude_status_client.py` covering all three verbs, capability pin acceptance + refusal (including semver edge cases), every documented typed `ErrName` round-tripping verbatim with no retry, malformed-stderr fallback, missing-binary (`FileNotFoundError`), and timeout (`subprocess.TimeoutExpired`).

## Constraints

- No `sqlite3` import. No `httpx` or network module. No direct read of Claude Status's SQLite store.
- No retry layer anywhere — every Waggle call corresponds to exactly one subprocess invocation.
- The module is import-inert: `python -c 'import waggle.claude_status'` performs no subprocess fork, no PATH lookup, and no file I/O.
- The startup-time capability pin enforcement belongs to sibling Epic `t1.fg8.vv` (which builds the MCP server). This Task delivers only the verb function; the caller decides what to do with a refusal.
- No modifications of any kind to pre-existing test files. Pre-existing-test cleanup belongs to sibling Epics `t1.fg8.bp` and `t1.fg8.2g`.

## SR-7.1 result shape (documented policy, applied uniformly)

Every result includes:
- A success/failure discriminator (implementer-chosen field name and encoding).
- `operation` — the verb name (`"workers"`, `"worker"`, `"capabilities"`).

On failure: `err_name` (verbatim Claude Status `ErrName` or a Waggle-originated err_name) and `err_description`. No Python traceback. No timestamp. No request ID.

On success: the parsed JSON payload under a stable key chosen by the implementer.

The landed shape is documented in the module docstring so Epics 2–5 build against a stable target.

## Acceptance Criteria

- `from waggle.claude_status import workers, worker, capabilities` succeeds; no `decide` symbol is exported.
- `pytest -q tests/test_claude_status_client.py` is green.
- All five documented `ErrName` values round-trip verbatim through the client.
- The capability pin accepts `1.x` and refuses any other major (including malformed/missing `contract_version`).
- No regression in pre-existing test count.
