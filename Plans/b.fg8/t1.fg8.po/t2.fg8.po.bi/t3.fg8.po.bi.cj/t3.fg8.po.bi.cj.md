## Context

Per SR-2.3 and SR-7.1, every non-zero exit from a `claude-status` invocation must be parsed against the documented stderr envelope grammar and the `ErrName` must propagate verbatim into the operation-failed result returned to the caller. Waggle must not invent a retry policy — every call corresponds to exactly one subprocess invocation. This Subtask covers all five typed `ErrName` values, the malformed-stderr fallback, the timeout path, and the missing-binary path. It is split from the verb-shape Subtask because the mock setup (non-zero exits, stderr-only fixtures, raised exceptions) and the assertion focus (err_name preservation, invocation count) differ.

## What Needs to Change

Append to `tests/test_claude_status_client.py` (created by the verb-shape Subtask) the error-mapping test cases. Use `fake_claude_status` to stage non-zero-exit triples with the canonical stderr envelope strings from `tests/sample_payloads`. For exception paths, use `fake_claude_status`'s mechanism for raising exceptions from the seam (or wrap `subprocess.run` directly via `unittest.mock.patch` if the helper does not support exception injection).

Coverage required:

- Parameterized typed-error tests: for each of `ErrInstanceNotFound`, `ErrNoPendingRequest`, `ErrSchemaMismatch`, `ErrStoreUnavailable`, `ErrPayloadMalformed`: stage the canonical stderr envelope with a non-zero exit code; invoke a verb; assert (a) the result is operation-failed, (b) `err_name` equals the documented `ErrName` string verbatim, (c) the recorded seam-invocation count is exactly 1 (no retry).
- Malformed stderr: stage a non-zero exit with stderr that does not match the documented envelope grammar (e.g. `"some unexpected text
"`); assert operation-failed without raising; assert `err_name` is the documented sentinel from `claude_status.py`'s module docstring (e.g. `ErrMalformedErrorEnvelope`).
- Non-JSON stdout on exit 0: stage stdout `"not json
"` with exit 0; assert operation-failed with `err_name=ErrPayloadMalformed` (the SR-2.3 typed name); no exception escapes.
- `FileNotFoundError` from the seam (simulating `claude-status` missing from PATH): assert operation-failed with the documented Waggle-originated err_name (e.g. `ErrClaudeStatusNotFound`); no traceback propagates; result includes context to diagnose the missing binary.
- `subprocess.TimeoutExpired` from the seam (simulating a hung `claude-status` subprocess): assert operation-failed with the documented Waggle-originated err_name (e.g. `ErrClaudeStatusTimeout`); no traceback propagates; seam invocation count is exactly 1.

No `conftest.py`. No inline `ERROR:` literals — all envelope strings from `tests/sample_payloads`.
