## Context

The Engineer is delivering `src/waggle/claude_status.py` exposing the `workers`, `worker`, and `capabilities` verbs (Task-2 Subtasks 1–2). This Subtask covers the success-path and contract-pin behaviors of those verbs: argv construction, JSON parsing on exit 0, and the major-version refusal of the capability pin. Error-mapping behaviors live in a sibling Subtask so each Subtask's mock setup stays homogeneous. All test data is sourced from `tests/sample_payloads.py` — no inline JSON literals.

## What Needs to Change

Create `tests/test_claude_status_client.py` containing the verb-shape and capability-pin tests. Use `fake_claude_status` from `tests/helpers.py` to stage success triples (exit 0, JSON stdout, empty stderr) and assert on (a) the recorded argv passed to the seam and (b) the parsed return value visible to the caller.

Coverage required:

- `workers()` (no label): argv is exactly the equivalent of `claude-status workers` (no `--label` entry); returned value matches the canonical envelope.
- `workers(label="waggle_owned=1")`: argv includes `"--label"` and `"waggle_owned=1"` as separate adjacent entries; returned value matches the canonical envelope.
- An empty `{"workers": [], "skipped": []}` response is operation-success — NOT operation-failed.
- `worker("inst-xyz")` happy path: argv is the equivalent of `claude-status worker inst-xyz`; returned dict matches the canonical Worker JSON.
- `capabilities()` happy path: argv is the equivalent of `claude-status capabilities`; returned payload is the parsed dict.
- Parameterized contract-version acceptance: `"1.0.0"` and `"1.7.3"` produce operation-success.
- Parameterized contract-version refusal: `"2.0.0"`, `"0.9.0"`, `"1"`, `"1.0"`, missing key, and non-string value each produce operation-failed with `err_name=ErrContractVersionMismatch`. The implementer's documented choice on whether `"1"` and `"1.0"` are accepted may flip those two cases to operation-success; tests assert against whichever choice the implementer documented in the module docstring — the test author reads the docstring and parameterizes accordingly.

No `conftest.py`. No inline JSON literals — all payloads from `tests/sample_payloads`. No imports from `waggle.*` other than `waggle.claude_status` (the surface under test).
