## Context

With the seam, parser, and shared call helper in place, the three public verb functions can be written as thin orchestrations over the shared helper. SR-2.2 pins Waggle to consumer contract `1.x` and requires `capabilities()` to refuse anything else; SR-2.3 forbids retries; SR-7.1 requires structured results. Only the three read verbs are implemented — Waggle never invokes `decide` per SR-2.2.

**Capability pin is operation-only, not module-load-time.** SR-2.2 says the pin runs at MCP server startup, not at module import. Task 2 delivers the `capabilities()` verb function only; startup-time enforcement belongs to sibling Epic `t1.fg8.vv` (which builds the MCP server). This Subtask MUST NOT add any module-load-time call that would break Subtask 5's "import performs no subprocess fork" assertion.

## What Needs to Change

In `src/waggle/claude_status.py`, add three public functions:

- `workers(label=None)`: builds argv `["workers"]` plus, when `label` is non-None, `["--label", <label>]` (as two separate argv entries). Calls the shared helper. The `label` argument is passed through verbatim — the client does not parse or validate label syntax; Claude Status is the authority. An empty `workers` list in the response (`{"workers": [], "skipped": []}`) is operation-success — must not be treated as a failure.
- `worker(instance_id)`: builds argv `["worker", instance_id]`. Calls the shared helper.
- `capabilities()`: builds argv `["capabilities"]`. Calls the shared helper. On operation-success: parse `contract_version` by splitting on `.`, taking the first component, and `int`-parsing it. If the major is exactly `1`, return operation-success with the parsed dict. Any other major (including `0`, `2`, etc.), or any parse failure (missing key, non-string, no dots, non-numeric major), returns operation-failed with `err_name=ErrContractVersionMismatch` (Waggle-originated). Operation-failed results from the shared helper propagate unchanged.

All three:
- Are importable from `waggle.claude_status`.
- Never raise; every code path returns a structured result.
- Maintain no module-level state, cache, or retry counter between calls.
- Are sync (no `async def`). `asyncio_mode = "auto"` in the pytest config does not require async test bodies; sync tests are fine.

The module exports no symbol named `decide`.

## Key Files

- `src/waggle/claude_status.py` — extended with the three public verb functions and the capability-pin parser

## Acceptance Criteria

- `from waggle.claude_status import workers, worker, capabilities` succeeds; `import waggle.claude_status; assert not hasattr(waggle.claude_status, "decide")` passes.
- `workers(label="waggle_owned=1")` (with seam stubbed to a successful envelope) invokes the seam with argv whose 0th element is `"workers"` and which contains `"--label"` and `"waggle_owned=1"` as separate adjacent entries. Returns operation-success with the parsed envelope.
- `workers()` without label invokes the seam with argv containing `"workers"` and NOT containing `"--label"`.
- `worker("abc-123")` invokes the seam with argv `["worker", "abc-123"]` (in that order).
- An empty `{"workers": [], "skipped": []}` response from `workers()` is operation-success, NOT operation-failed.
- `capabilities()` with stubbed stdout `{"contract_version": "1.0.0", ...}` → operation-success.
- `capabilities()` with stubbed stdout `{"contract_version": "1.7.3", ...}` → operation-success.
- `capabilities()` with stubbed stdout `{"contract_version": "2.0.0", ...}` → operation-failed with `err_name=ErrContractVersionMismatch`.
- `capabilities()` with stubbed stdout `{"contract_version": "0.9.0", ...}` → operation-failed with the same err_name.
- `capabilities()` with stubbed stdout `{"contract_version": "1", ...}`, `{"contract_version": "1.0", ...}`, a missing key, or a non-string value: each returns operation-failed with the same err_name (these are parser edge cases — implementer chooses whether to treat the integer-major `"1"` or two-component `"1.0"` as accepted or rejected, but must apply that choice consistently and document it).
- Module import remains side-effect-free; no subprocess fork at import time.
