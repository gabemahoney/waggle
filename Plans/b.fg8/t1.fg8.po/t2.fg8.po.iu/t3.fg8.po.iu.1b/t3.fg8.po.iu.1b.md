## Context

The constants in `tests/sample_payloads.py` are the contract surface every downstream test depends on. Without structural invariants asserted somewhere, drift in `sample_payloads.py` could pass silent collection and surface as confusing failures across sibling Epic test files. This Subtask is the structural-invariants test for the constants module.

## What Needs to Change

Create `tests/test_sample_payloads.py` asserting:

- Every defined Worker JSON dict contains the required keys (`instance_id`, `status`, `labels`, `pending`); `status` is one of the six documented values; `labels` is a `dict`; `pending` is either `None` or a `dict` with a `kind` key.
- Every defined pending-payload dict has a `kind` field whose value is one of `"permission_request"` or `"ask_user_question"`; an `ask_user_question` payload has `tool_input.questions` of type `list`; the multi-question variant has `len(tool_input.questions) == 2`.
- The two capabilities response dicts each contain a `contract_version` key whose value is a string; the accepted one has major `1`, the refused one has major `2`.
- Every stderr envelope string matches the regex `^ERROR: (ErrInstanceNotFound|ErrNoPendingRequest|ErrSchemaMismatch|ErrStoreUnavailable|ErrPayloadMalformed): .+
$` (i.e. correct grammar, documented `ErrName` token, non-empty description, trailing newline).

The file imports only from `tests.sample_payloads`; no imports from `waggle` source.
