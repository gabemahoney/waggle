## Context

The four helpers in `tests/helpers.py` are the public testing contract sibling Epics 2–5 depend on. If the helpers have import errors, wrong return shapes, or missing attributes, sibling-Epic test files fail at collection time with unhelpful errors. A named smoke-test file localizes those failures and makes regressions attributable.

## What Needs to Change

Create `tests/test_helpers_smoke.py` asserting:

- All four helpers (`fake_claude_status`, `fake_worker_record`, `fake_workers_response`, `fake_tmux_pane`) and the canonical constants from `tests/sample_payloads` are importable without error.
- `fake_worker_record("inst-abc", "waiting")` returns a dict with keys `instance_id`, `status`, `labels`, `pending`; `pending` is `None` when omitted; `labels` is a dict containing at least `waggle_owned`, `waggle_session_name`, `waggle_model`, `waggle_repo`.
- `fake_worker_record("inst-xyz", "ask_user", pending={"kind": "ask_user_question", "request_id": 7, "tool_input": {"questions": []}})` round-trips `pending["kind"]` as `"ask_user_question"`.
- `fake_workers_response([r1, r2])` returns `{"workers": [r1, r2], "skipped": []}`; `fake_workers_response([], skipped=("err-row",))` returns a dict whose `skipped` length is 1.
- `fake_tmux_pane("line one
line two").capture_pane()` returns a list whose joined form equals `"line one
line two"`.
- `fake_claude_status` is importable and (when used as a fixture or context manager) the recorded-invocations list starts empty.

The file imports only from `tests.helpers` and `tests.sample_payloads`; no imports from `waggle.claude_status` or any other production source.
