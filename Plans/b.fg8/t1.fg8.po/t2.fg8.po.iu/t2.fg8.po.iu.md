## Context

This Task introduces the shared test infrastructure required by SR-11.2 and consumed by every sibling Epic's test suite. SR-11.1 forbids `conftest.py`, so the helpers live as plain importable modules.

## What Needs to Change

- Add `tests/sample_payloads.py`: canonical Worker JSON dicts (one per distinct shape tests will assert on — not six near-identical variants), pending-payload dicts for both `permission_request` and `ask_user_question` (single-question and multi-question forms), capabilities response dicts for `contract_version="1.0.0"` and `"2.0.0"`, and stderr envelope strings for each documented `ErrName` (`ErrInstanceNotFound`, `ErrNoPendingRequest`, `ErrSchemaMismatch`, `ErrStoreUnavailable`, `ErrPayloadMalformed`).
- Add `tests/helpers.py`: `fake_claude_status` (patches a documented subprocess seam in `waggle.claude_status` — the patch-target FQN is shared with Task 2 and pinned in this Task's Subtask), `fake_worker_record(instance_id, status, labels=None, pending=None)`, `fake_workers_response(records, skipped=())`, `fake_tmux_pane(text)` (capture-pane returns `list[str]` matching libtmux's contract).
- Add test files validating the infrastructure: `tests/test_sample_payloads.py` and `tests/test_helpers_smoke.py`.

## Constraints

- No `conftest.py`.
- No `jsonschema` runtime dependency.
- No retry/backoff machinery in `fake_claude_status` — pure record-and-return.
- No modifications of any kind to pre-existing test files in this Task. Pre-existing-test cleanup belongs to sibling Epics `t1.fg8.bp` and `t1.fg8.2g`.

## Acceptance Criteria

- `from tests.sample_payloads import ...` and `from tests.helpers import fake_claude_status, fake_worker_record, fake_workers_response, fake_tmux_pane` both succeed without error.
- All new test files pass under `pytest -q`.
- No regression in pre-existing test count; nothing pre-existing is deleted, skipped, or xfailed.
