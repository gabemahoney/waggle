## Context

SR-11.2 requires shared test helpers that patch Waggle's `claude-status` subprocess seam, construct typed worker-record dicts, build the `workers` verb envelope, and produce fake libtmux pane objects. Sibling Epics 2–5 consume these directly. SR-11.1 forbids `conftest.py`, so the helpers live in a plain importable module.

## What Needs to Change

Create `tests/helpers.py` providing the four helpers from SR-11.2:

- `fake_claude_status`: a pytest-compatible fixture or context manager that patches the `waggle.claude_status` module-level subprocess invocation seam. **Patch-target FQN is shared with Task 2** and must be agreed before this Subtask lands (e.g. `waggle.claude_status._run` or similar — pinned in coordination with Task 2's seam). The helper accepts an ordered sequence of canned `(stdout_str, stderr_str, exit_code)` triples, consumed one per seam invocation, and records each invocation's full argv list so tests can assert on `--label` flags. It raises `AssertionError` if the seam is called more times than canned triples exist, and the symmetric assertion: an un-consumed triple remaining at fixture teardown is also a test failure (this catches tests that silently skip an expected preflight call).
- `fake_worker_record(instance_id, status, labels=None, pending=None)`: returns a dict matching a single `workers`-array entry under contract `1.0.0`. `status` must be one of the six documented values. `labels` defaults to a minimal set including `waggle_owned`, `waggle_session_name`, `waggle_model`, `waggle_repo`.
- `fake_workers_response(records, skipped=())`: returns `{"workers": list(records), "skipped": list(skipped)}`.
- `fake_tmux_pane(text)`: returns a fake libtmux pane object whose `capture_pane` method returns `text` as a `list[str]` of lines (matching what libtmux's `Pane.capture_pane()` returns and how Waggle's existing `_capture_pane_sync` consumes it).

No retry/backoff machinery anywhere. No production-source imports beyond the patch target.

## Acceptance Criteria

- `from tests.helpers import fake_claude_status, fake_worker_record, fake_workers_response, fake_tmux_pane` succeeds.
- `fake_claude_status` (used as fixture or context manager) patches the agreed seam name; a test that uses it does not fork a real `claude-status` binary even if one is on PATH (verifiable by setting `PATH=/nonexistent` in the test).
- Calling the seam more times than canned triples raises `AssertionError`; ending the fixture/context with un-consumed triples raises `AssertionError`.
- `fake_worker_record("abc", "working")` returns a dict whose `instance_id` is `"abc"`, `status` is `"working"`, `labels` is a dict including `waggle_owned`, and `pending` is `None`.
- `fake_workers_response([r1, r2])` returns `{"workers": [r1, r2], "skipped": []}`.
- `fake_tmux_pane("hello
world").capture_pane()` returns `["hello", "world"]` (or the exact list-of-strings shape libtmux returns).
- No pytest fixtures wrapped in `conftest.py`; helpers live in this module only.
