## Description

With the new stdio MCP surface fully built (Epics 2–4) and the new installer in place (Epic 5), delete the daemon-era code paths.

Source deletions (SR-1.2):
- `src/waggle/daemon.py` (daemon entry point)
- `src/waggle/server.py` (HTTP/SSE-bound FastMCP server — the new stdio server lives elsewhere)
- `src/waggle/rest.py`, `src/waggle/inbound_processor.py`, `src/waggle/outbound_processor.py`, `src/waggle/queue.py`, `src/waggle/recovery.py`, `src/waggle/state_monitor.py`, `src/waggle/cma_client.py`, `src/waggle/mailer.py`, `src/waggle/auth.py`, `src/waggle/middleware.py`
- `waggle.service` (the systemd unit at repo root)

CLI surface changes (SR-4.1):
- Remove the `waggle serve`, `waggle set-state`, `waggle permission-request`, `waggle ask-relay` subcommands from `cli.py`.
- The four removed MCP tools (`register_caller`, `list_workers`, `check_status`, `approve_permission`) disappear with `server.py`; the new stdio server from Epics 2–4 never registered them and is unaffected (SR-3.1).

Dependency pruning (SR-9.2):
- Drop `starlette`, `uvicorn`, `persist-queue` from `pyproject.toml`'s runtime dependencies.
- Drop `httpx` only after verifying via grep that no surviving source file imports it.
- Keep `libtmux`. Keep `fastmcp` only if its stdio transport is the runtime used by Epic 2's server.

Test deletions (SR-11.4):
- `tests/test_rest.py`, `test_auth.py`, `test_middleware.py`, `test_queue.py`, `test_restart_recovery.py`, `test_state_monitor.py`, `test_cma_client.py`, `test_mailer.py`, `test_tls.py`, `test_set_state.py`
- `tests/test_mcp_integration.py` (rewritten as part of Epic 2 or deleted entirely if subsumed by `test_mcp_stdio_server.py`)
- `tests/test_integration_cma.py`, `tests/test_integration_ssh_spawn.py`
- `tests/test_relay_cli.py`, `test_relay_engine.py`, `test_relay_integration.py`
- Update `tests/test_package_entry_point.py` (or replace it) so it asserts the new entry points (`waggle mcp`, `waggle install`, `waggle sting`) and not the deleted ones.
- Pre-existing `tests/test_engine.py` and `tests/test_cli.py` — keep only the cases that exercise surviving behavior; delete or migrate the rest.

Documentation (SR-12, partial):
- Rewrite `README.md` to document the stdio MCP launch model (`waggle mcp`), the dependency on Claude Status, the install path (`waggle install`), and the six surviving MCP tools. The migration story for orchestrators (use `claude-status workers` / `worker` / `decide` directly for general state reads and permission approval) is called out explicitly.
- Delete `docs/architecture/relay.md` and `docs/architecture/waggle_set_state.md`. The remaining `docs/architecture/spawn_worker.md`, `send_input.md`, `terminate_worker.md` files are reviewed and updated to reflect the stdio surface; outdated daemon-era language is removed.
