## Context

With the daemon and HTTP surface deleted, rewrite `README.md` for the stdio MCP launch model and clean up `docs/architecture/` per SR-12.

## What Needs to Change

**README.md rewrite:**
- Remove all daemon-era language: REST/HTTP, systemd, `waggle.service`, `waggle serve`, persist-queue, CMA, mailer, SSH auth middleware.
- Document the stdio MCP launch model: `waggle mcp` is launched per orchestrator session as a stdio MCP server; no daemon, no listener.
- Document the dependency on Claude Status (relay mode for permissions, record mode for AUQs, consumer-CLI for state reads).
- Document the install path: `waggle install` invokes `claude-status install-hooks`.
- Document the six surviving MCP tools: `spawn_worker`, `list_spawned_workers`, `terminate_worker`, `send_input`, `get_output`, `answer_question`.
- Document the migration story for orchestrators: state reads via `claude-status workers` / `worker`; permission approvals via `claude-status decide` (NOT via Waggle).
- Link to `docs/migration.md` (written in Epic 7).

**docs/architecture/ cleanup:**
- Delete `docs/architecture/relay.md` (the relay path is owned by Claude Status now).
- Delete `docs/architecture/waggle_set_state.md` (the `set-state` subcommand is gone).
- Review remaining files (`spawn_worker.md`, `send_input.md`, `terminate_worker.md`): update to reflect the stdio surface; remove daemon-era language; preserve content describing the current MCP tool semantics.

## Constraints

- README must have zero mentions of HTTP, REST, systemd, `waggle.service`, `waggle serve`.
- The six-tool list must match exactly the six tools registered on the stdio server (no preview tools, no removed tools).
- Migration link target may be a stub at this Task's landing (the doc body lands in Epic 7); just ensure the link resolves to the future path.

## Acceptance Criteria

- `grep -REn "systemd|waggle\.service|REST|HTTP daemon|SSE|register_caller|approve_permission" README.md` returns nothing.
- `find docs/architecture/ -name "relay.md" -o -name "waggle_set_state.md"` returns nothing.
- README documents all six MCP tools by name.
- A reader who knows nothing about Waggle can follow the README to: install Claude Status, run `waggle install`, run `waggle mcp`, and understand each of the six tools' purposes.
