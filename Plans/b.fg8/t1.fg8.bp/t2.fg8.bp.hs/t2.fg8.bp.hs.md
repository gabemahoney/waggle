## Context

`cli.py` currently registers `serve`, `set-state`, `permission-request`, `ask-relay` subcommands that dispatch to daemon-era handlers. Removing them must happen BEFORE deleting the daemon source modules (otherwise `cli.py` is broken). The new subcommands `mcp`, `install`, `sting` (Epics 2, 5) remain untouched.

## What Needs to Change

- Remove `serve`, `set-state`, `permission-request`, `ask-relay` subparser registrations and their dispatch handlers from `src/waggle/cli.py`.
- Remove any helper imports those handlers used (but only if no other code path imports them — that's verified in the next Task's deletions).
- Update or delete `tests/test_set_state.py` and `tests/test_relay_cli.py` if they still exist (they are listed for deletion in this Epic's larger sweep, but the CLI surface change may cause earlier collection failures — delete the relevant test cases here if needed).

## Constraints

- Existing `mcp`, `install`, `sting` subcommands remain unchanged.
- `waggle <removed-subcommand>` must exit non-zero with an "unknown command" error (argparse default behavior).
- Codebase remains green after this Task.

## Acceptance Criteria

- `waggle serve`, `waggle set-state`, `waggle permission-request`, `waggle ask-relay` each exit non-zero with an argparse "unknown command" error.
- `waggle mcp --help`, `waggle install --help`, `waggle sting --help` each exit 0 and print help.
- `pytest -q` is green (any test referencing the removed subcommands has been deleted or migrated).
