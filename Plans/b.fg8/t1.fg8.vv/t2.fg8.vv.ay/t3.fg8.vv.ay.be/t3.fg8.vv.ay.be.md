## Context

SR-3.3 specifies the exclusion contract for two row classes: `skipped` rows (Claude Status couldn't parse the row) and valid rows missing `labels.waggle_session_name` (Waggle can't project them). Both classes emit one JSON-per-line stderr log entry containing `instance_id`, `error`, `error_description` and are excluded from the result. This Subtask covers both.

## What Needs to Change

Append to `tests/test_engine_list_spawned_workers.py` exclusion tests:

- One skipped row: stage a response with `skipped=[<one row from sample_payloads>]`; capture stderr via `capsys` (no `conftest.py` needed); assert exactly one stderr line is emitted; the line is parseable as JSON; the JSON contains `instance_id`, `error` (the `ErrName`), `error_description`; the returned list does NOT contain that instance_id.
- Two skipped rows: assert exactly two JSON-per-line stderr entries, one per row.
- Missing-label row: stage a valid worker row whose `labels` dict is present but lacks the `waggle_session_name` key. Assert the row is excluded from the result and one JSON-per-line stderr entry is emitted with the three mandatory fields.
- Mixed response: stage `workers=[<two valid>, <one missing-label>]`, `skipped=[<one row>]`. Assert the result contains exactly the two valid workers; assert exactly two JSON-per-line stderr entries (one for the missing-label, one for the skipped).

All test data from `tests/sample_payloads.py`.

## Key Files

- `tests/test_engine_list_spawned_workers.py` — same file as the argv+projection Subtask; this Subtask appends exclusion tests

## Acceptance Criteria

- Each stderr line is parseable as JSON (`json.loads(line)` does not raise).
- Each JSON object contains `instance_id`, `error`, `error_description`.
- The result list count + the stderr-entry count equals the input row count for every test.
- The missing-label test catches a regression where the tool returns rows with empty or default `session_name`.
- Tests pass under `pytest -q`.
