## Context

SR-1.1 (stdio-only, no TCP/UDS/HTTP listener) is the non-negotiable transport contract for Waggle's MCP server. This Subtask validates the contract with negative-space testing — patching every socket-binding API and asserting zero calls during server construction and startup.

## What Needs to Change

Create `tests/test_mcp_stdio_server.py` with tests that:
- Patch `socket.socket`, `socket.bind`, `asyncio.start_server`, `asyncio.start_unix_server`, `uvicorn.Server`, and any Starlette ASGI constructor before instantiating/starting the stdio server; assert zero calls to any of them during construction and `run()`. The stdio transport is stubbed so no real stdin/stdout takeover happens.
- Assert the set of registered tools at the Task-1 landing point is EMPTY (zero tools). `spawn_worker` and `list_spawned_workers` register in Tasks 2 and 3; `terminate_worker`, `send_input`, `get_output`, `answer_question` register in sibling Epics.

## Key Files

- `tests/test_mcp_stdio_server.py` — new file (this Subtask adds the transport + registration test class)

## Acceptance Criteria

- A test constructs the stdio server and asserts `socket.socket`, `socket.bind`, `asyncio.start_server`, `asyncio.start_unix_server`, `uvicorn.Server`, and Starlette ASGI ctors were called zero times.
- A test inspects the server's tool registry (via FastMCP's introspection surface) and asserts the registered-tool count is 0.
- No `conftest.py` is added or modified.
- No pre-existing test file is modified.
- Tests pass under `pytest -q`.
