## Context

Lands the new stdio-only MCP surface. The `waggle mcp` subcommand launches a FastMCP server bound to stdio (no TCP/UDS/HTTP listener per SR-1.1), enforces the SR-2.2 capability pin at startup, and exposes a shared SR-7.1 error-translation wrapper that subsequent Tasks' tool bodies (and sibling Epics' tools) all plug into. The existing daemon, HTTP, and old MCP server remain untouched in this Epic — their deletion belongs to `t1.fg8.bp`.

## What Needs to Change

- New module (e.g. `src/waggle/mcp_stdio.py`) hosting the stdio FastMCP server and the SR-7.1 wrapper.
- New `waggle mcp` CLI subcommand in `cli.py` that invokes the module's `run()` entry point after running the SR-2.2 capability pin.
- Append a "Stdio MCP launch model" section to `README.md` (no daemon-section edits).
- New tests in `tests/test_mcp_stdio_server.py` covering transport bring-up, zero tools registered at Task-1 landing, and SR-7.1 wrapper behavior on success/failure paths.

## Constraints

- No TCP/UDS/HTTP listener. No imports of `starlette`, `uvicorn`, `socket.socket`-bind paths, `httpx`, or `fastapi` from the new module.
- Module is import-inert: `python -c 'import waggle.mcp_stdio'` performs no subprocess fork, no socket bind, no daemon-era module import.
- SR-7.1 envelope shape is shape-identical to the one landed in `src/waggle/claude_status.py` (Epic `t1.fg8.po`). Reuse the constructor or implement against the same documented shape.
- No `_smoke` or test-only tool registered on the server. Wrapper is tested as a callable.
- No `--port`, `--host`, `--bind` flag on `waggle mcp`.
- Daemon, `server.py`, and `waggle serve`/`set-state`/`permission-request`/`ask-relay` subcommands remain unchanged.

## Acceptance Criteria

- `waggle mcp --help` exits 0 with text describing the stdio MCP server.
- `waggle mcp` attaches FastMCP to stdio; `lsof -p <pid>` shows no TCP/UDS listener.
- On startup, `waggle mcp` invokes `claude_status.capabilities()`; on operation-failed (missing binary, contract mismatch, unparseable, store unavailable) the subcommand exits non-zero with the structured error written to stderr.
- `waggle mcp` exits cleanly (exit 0) on stdin EOF.
- The SR-7.1 wrapper, applied to a function that raises `Exception("boom")`, returns operation-failed conforming to the Epic-1 envelope shape; no traceback escapes.
- The SR-7.1 wrapper passes a `claude_status`-shaped operation-failed dict through unchanged (no double-wrapping).
- `pytest -q tests/test_mcp_stdio_server.py` is green.
