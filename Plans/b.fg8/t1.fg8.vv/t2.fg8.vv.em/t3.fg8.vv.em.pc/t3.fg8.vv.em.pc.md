## Context

The SR-7.1 wrapper is the contract every Waggle MCP tool body relies on for failure surfacing. Test it as a direct callable (no in-server stub tool — that was explicitly rejected at planning time). Validate both the success path (value passes through into operation-success) and the failure path (exception translates to operation-failed shape-identical to the Epic-1 `waggle.claude_status` envelope).

## What Needs to Change

Append to `tests/test_mcp_stdio_server.py` the wrapper-behavior tests:

- A test invokes the wrapper around a function returning a plain value (e.g. `{"hello": "world"}`); asserts the wrapped result is operation-success with the value preserved verbatim.
- A test invokes the wrapper around a function raising `Exception("boom")`; asserts the wrapped result is operation-failed with shape identical to the envelope from `waggle.claude_status`. Specifically, compare an `ErrInstanceNotFound` operation-failed dict from `claude_status.worker(...)` (using `fake_claude_status` + canonical stderr from `sample_payloads`) and an exception-translated operation-failed from the SR-7.1 wrapper — assert both conform to the same shape (same discriminator field, same key for `err_name`, same key for `operation`, etc.).
- A test asserts the exception does NOT propagate out of the wrapper (no `pytest.raises` needed; the wrapped call returns normally).
- A test asserts the wrapper does NOT retry: a function with an invocation counter that raises is called exactly once per wrapped invocation.
- A test invokes the wrapper around a function returning a `waggle.claude_status`-shaped operation-failed dict; asserts the wrapper returns that dict unchanged (no double-wrapping).

## Key Files

- `tests/test_mcp_stdio_server.py` — same file as the transport-tests Subtask; this Subtask appends wrapper tests
- `tests/helpers.py`, `tests/sample_payloads.py` — read-only dependencies

## Acceptance Criteria

- All wrapper tests pass under `pytest -q`.
- The shape-identity assertion explicitly compares dicts produced by `claude_status.worker` operation-failed vs. wrapper exception-translation operation-failed; both have the same set of top-level keys.
- The no-retry assertion shows the invocation counter at 1 after a wrapped call.
- The pass-through test demonstrates operation-failed dicts from `claude_status` are not re-wrapped.
