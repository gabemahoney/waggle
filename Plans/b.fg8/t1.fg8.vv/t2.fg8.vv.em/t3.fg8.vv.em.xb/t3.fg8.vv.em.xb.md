## Context

SR-1.1 requires Waggle's MCP server to be stdio-only — no TCP, UDS, or HTTP listener. SR-7.1 requires every tool body to be wrapped in error-translation that converts uncaught exceptions, OS errors, subprocess failures, and unparseable JSON into a structured operation-failed result conforming to the envelope landed by `src/waggle/claude_status.py` (Epic `t1.fg8.po`). This Subtask establishes the new stdio-server module and the shared wrapper so Tasks 2 and 3 of this Epic, and the four tools in sibling Epics `t1.fg8.4n` and `t1.fg8.fd`, plug in uniformly. The existing daemon code remains in place; deletion belongs to Epic `t1.fg8.bp`.

## What Needs to Change

Add a new module (suggested path `src/waggle/mcp_stdio.py`) that:

- Constructs a `FastMCP` server instance bound to stdio via the explicit stdio transport entry point. Exposes a module-level `run()` (or equivalent) that the CLI subcommand invokes; returns when stdin closes.
- Exports the SR-7.1 error-translation wrapper — a decorator or callable — applicable to any tool body. The wrapper catches all `Exception` subclasses, builds an operation-failed result whose shape is identical to the envelope produced by `waggle.claude_status` (reuse a constructor or implement against the same documented shape), and never re-raises. Operation-failed results coming from `waggle.claude_status` are passed through unchanged (no double-wrapping).
- Registers ZERO tools at this Subtask's landing point. No `_smoke` or test-only tool. The wrapper is exercised by direct unit tests in the test Subtask.

## Constraints

- No imports of `starlette`, `uvicorn`, `fastapi`, `httpx`, or socket-binding stdlib symbols. `grep -REn 'starlette|uvicorn|fastapi|httpx|socket\.bind' <new module>` returns nothing.
- Module is import-inert: importing it performs no subprocess fork, no socket bind, no PATH lookup, no `claude-status` capability check.
- Module's import graph does not transitively pull in `daemon.py`, `rest.py`, `inbound_processor.py`, `outbound_processor.py`, `queue.py`, `recovery.py`, `state_monitor.py`, `cma_client.py`, `mailer.py`, `auth.py`, or `middleware.py`.
- Does NOT replace or modify the existing `server.py` (HTTP-bound). That file's deletion belongs to Epic `t1.fg8.bp`.

## Key Files

- `src/waggle/mcp_stdio.py` (or chosen path) — new module hosting the stdio FastMCP server and the SR-7.1 wrapper

## Acceptance Criteria

- The module imports without performing any subprocess fork, socket bind, or daemon-module import.
- The SR-7.1 wrapper is importable and callable; applied to a function that raises `ValueError("boom")` it returns an operation-failed dict shape-identical to the one produced by `waggle.claude_status` for typed errors (same discriminator, same `err_name` key, same `operation` key).
- Applied to a function that returns a `waggle.claude_status`-shaped operation-failed dict, the wrapper returns the dict unchanged.
- The server instance has exactly zero tools registered at this Subtask's completion.
- The module file contains no imports of `starlette`, `uvicorn`, `fastapi`, `httpx`, or socket-binding stdlib symbols.
