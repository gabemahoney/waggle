## Context

SR-3.2 requires `spawn_worker` to create the tmux session with seven env vars injected at session-creation time via `tmux new-session -e KEY=VAL`. The existing `_create_session_sync` in `tmux.py` only injects two hardcoded env vars and uses `session.set_environment` for one more — which does not propagate to processes spawned within the session. The fix is to pass all seven env vars through `libtmux.Server.new_session(environment=...)` (or the equivalent libtmux path that maps to `tmux new-session -e`) in ONE invocation. To avoid disturbing existing tests of `_create_session_sync` (per the no-edits-to-pre-existing-tests constraint), add a new helper rather than mutating the existing one.

## What Needs to Change

- In `src/waggle/tmux.py`, add `_create_session_with_env_sync(session_name, repo_path, env)` (sync) and `create_session_with_env(...)` (async wrapper using `asyncio.to_thread` per SR-6.1). The `env` parameter is a `dict[str, str]` containing the seven keys SR-3.2 mandates; the helper passes it through to `libtmux.Server.new_session` (or equivalent libtmux call) in ONE invocation. Multiple `set-environment` calls are NOT acceptable — they have different propagation semantics.
- The new helper returns the same success/error dict shape used by other `tmux.py` helpers.
- The existing `_create_session_sync` / `create_session` are NOT modified.
- Add a code comment near the new helper: `# TODO(t1.fg8.bp): collapse with _create_session_sync after daemon removal`.

## Constraints

- Confirm the libtmux call used produces the same effect as `tmux new-session -e KEY=VAL` — i.e. env vars propagate to all processes spawned within the session (per PRD Assumptions). If libtmux's API doesn't support this cleanly for the installed version, fall back to a direct `subprocess.run` of the `tmux` binary with the `-e` flags built explicitly.
- No mutation of the existing `_create_session_sync` or its tests.

## Key Files

- `src/waggle/tmux.py` — add the new sync helper and async wrapper

## Acceptance Criteria

- `_create_session_with_env_sync` accepts a full env dict and passes it to one `libtmux.Server.new_session` call (or one `tmux new-session` subprocess invocation) — verifiable via a unit test that mocks the libtmux entry point and asserts the env mapping.
- The existing `_create_session_sync` / `create_session` are unchanged (`git diff` shows no edits to those functions).
- The existing `tests/test_tmux.py` continues to pass without modification.
- The new helper is exported from `waggle.tmux` and importable by the `spawn_worker` body.
- The `# TODO(t1.fg8.bp): collapse with _create_session_sync after daemon removal` comment exists near the new helper.
