## Context

`spawn_worker(model, repo, session_name?)` is the first production MCP tool registered on the stdio server. Per SR-3.2 it creates a tmux session with seven env vars injected at session creation time, sends the Claude launch command to window 0/pane 0, and returns `{instance_id, session_name}` immediately without polling Claude Status. The env vars must persist for the worker's entire lifetime — `tmux new-session -e KEY=VAL` propagates env vars to all subprocesses spawned within the session.

## What Needs to Change

- New `_create_session_with_env_sync` (+ async wrapper) in `tmux.py` that accepts the full env dict and passes it through to `libtmux` via the path that maps to `tmux new-session -e KEY=VAL` (one `new-session` invocation only — not `new-session` + multiple `set-environment` calls). Existing `_create_session_sync` and `create_session` are untouched.
- Register `spawn_worker` on the stdio server from Task 1. Body wrapped in the Task-1 SR-7.1 wrapper.
- New tests in `tests/test_engine_spawn_worker.py` (split into return-shape, env-injection, and launch/no-poll/wrapping suites).

## Constraints

- No call to any `waggle.claude_status` verb from the spawn body. No polling. No placeholder row in Claude Status.
- UUID for `instance_id` via `uuid.uuid4()`. Default `session_name` = `waggle-<first 8 hex chars of UUID>` (i.e. `uuid.uuid4().hex[:8]` or `str(uuid).split('-')[0]`).
- Lower-case the model at BOTH sites: `CLAUDE_STATUS_LABEL_WAGGLE_MODEL` env var value AND the `claude --model <model>` send-keys command. Do not lower-case the orchestrator's input parameter as returned/stored elsewhere.
- Send-keys targets `<session>:0.0` explicitly (not the active pane). Robustness; consistency with Epic `t1.fg8.4n`'s targeting work.
- Tool body returns the bare dict `{instance_id: str, session_name: str}`; the SR-7.1 wrapper handles envelope framing.
- New env-injecting helper has a `# TODO(t1.fg8.bp): collapse with _create_session_sync after daemon removal` comment.
- No modifications of any kind to pre-existing test files.

## Acceptance Criteria

- `spawn_worker(model="sonnet", repo="/tmp/x")` returns operation-success wrapping `{instance_id: <uuid-string>, session_name: "waggle-<first8hex>"}`.
- `libtmux` session creation receives all seven env vars in one `new-session` call: `CLAUDE_STATUS_INSTANCE_ID`, `CLAUDE_STATUS_RELAY_MODE=on`, `CLAUDE_STATUS_AUQ_MODE=record`, `CLAUDE_STATUS_LABEL_WAGGLE_OWNED=1`, `CLAUDE_STATUS_LABEL_WAGGLE_SESSION_NAME=<session_name>`, `CLAUDE_STATUS_LABEL_WAGGLE_MODEL=<lower-cased model>`, `CLAUDE_STATUS_LABEL_WAGGLE_REPO=<repo>`.
- The Claude launch command sent to `<session>:0.0` is `claude --model <lower-cased model>` followed by Enter.
- `spawn_worker` body imports nothing from `waggle.claude_status`; patching `claude_status.worker` and `claude_status.workers` to raise does NOT affect the spawn path.
- Injected `LibTmuxException` or `OSError` at the session-creation seam yields operation-failed via the SR-7.1 wrapper; no exception propagates.
- `pytest -q tests/test_engine_spawn_worker.py` is green.
