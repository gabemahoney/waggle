## Context

SR-3.2 mandates seven env vars passed in one `new-session` invocation. The model string is lower-cased at BOTH sites: the env-var value AND the launch command. This Subtask covers env-var injection and model-lowercasing in isolation.

## What Needs to Change

Append to `tests/test_engine_spawn_worker.py` env-injection tests:

- Parameterized over model casing (`"Sonnet"`, `"OPUS"`, `"haiku"`, `"opus"`): assert each of the seven env vars is present in the recorded `new_session` env mapping with the correct value:
  - `CLAUDE_STATUS_INSTANCE_ID` matches the returned instance_id
  - `CLAUDE_STATUS_RELAY_MODE` is `"on"`
  - `CLAUDE_STATUS_AUQ_MODE` is `"record"`
  - `CLAUDE_STATUS_LABEL_WAGGLE_OWNED` is `"1"`
  - `CLAUDE_STATUS_LABEL_WAGGLE_SESSION_NAME` equals the returned session_name
  - `CLAUDE_STATUS_LABEL_WAGGLE_MODEL` equals the lower-cased model
  - `CLAUDE_STATUS_LABEL_WAGGLE_REPO` equals the repo argument verbatim
- For `spawn_worker(model="Sonnet")`: assert env value is `"sonnet"`.
- For `spawn_worker(model="OPUS")`: assert env value is `"opus"`.
- Assert the env mapping is passed to ONE `new_session` call — the test patches both `new_session` and `set_environment`, asserts `new_session` called once with the full env, and `set_environment` called zero times.

## Key Files

- `tests/test_engine_spawn_worker.py` — same file as the return-shape Subtask; this Subtask appends env-injection tests

## Acceptance Criteria

- All seven env-var keys are asserted explicitly in at least one parameterized test (no coarse subset checks).
- Model lowercasing is verified for at least three input casings.

- Tests pass under `pytest -q`.
