## Context

SR-3.2: Claude Code launches via `tmux send-keys` of `claude --model <model>` plus Enter to window 0, pane 0; `spawn_worker` returns immediately without polling Claude Status. SR-7.1: failures wrap to operation-failed via the Task-1 wrapper. This Subtask covers all three properties in one test class — they share the full-spawn execution setup.

## What Needs to Change

Append to `tests/test_engine_spawn_worker.py` launch-command + no-poll + wrapper tests:

- Send-keys target assertion: the recorded send-keys call targets `<session>:0.0` explicitly (not the active pane). The text sent is `claude --model <lower-cased model>` and Enter is sent.
- Parameterized over model casing: assert the launch-command text uses the lower-cased model (matching the env-var lowercasing from the sibling Subtask).
- No-poll assertion: patch `waggle.claude_status.worker` to raise `AssertionError("spawn must not poll claude-status")` and assert `spawn_worker` returns successfully — i.e. the patched function was never called.
- Same for `waggle.claude_status.workers`: patch to raise; assert never called.
- Failure-injection: force the underlying tmux create-session call to raise `OSError("tmux explode")` and assert `spawn_worker` returns operation-failed (the SR-7.1 wrapper shape) with no exception escaping.
- Second failure-injection: force a `LibTmuxException` and assert the same wrapper behavior.

## Key Files

- `tests/test_engine_spawn_worker.py` — same file as sibling Subtasks; this Subtask appends launch-command + no-poll + wrapping tests
- `tests/helpers.py` — used for `fake_claude_status` if needed; also direct patches via `unittest.mock.patch`

## Acceptance Criteria

- A test asserts send-keys was called with target `<session>:0.0` (assertable via the mock's call args) and text starting with `"claude --model "`.
- A test asserts `claude_status.worker` and `claude_status.workers` were never called during `spawn_worker` execution.
- A test asserts an injected `OSError` produces operation-failed via the SR-7.1 wrapper; no `pytest.raises` needed.
- A test asserts an injected `LibTmuxException` produces the same wrapper behavior.
- All tests pass under `pytest -q`.
