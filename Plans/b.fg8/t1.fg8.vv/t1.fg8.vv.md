## Description

Introduce the new stdio-only MCP surface that will, by the end of the refactor, be the only Waggle surface. Add a `waggle mcp` CLI subcommand that launches a FastMCP server bound to stdio — no TCP/UDS/HTTP listener (SR-1.1). Register two tools, each wrapped in the SR-7.1 error-translation envelope:

`spawn_worker(model, repo, session_name?)` → `{instance_id, session_name}` per SR-3.2. Construct a UUID for `CLAUDE_STATUS_INSTANCE_ID`. Default `session_name = waggle-<first 8 chars of instance_id>`. Create the tmux session via `tmux new-session -d -s <session_name>` with the following env vars injected via `-e`:

- `CLAUDE_STATUS_INSTANCE_ID=<uuid>`
- `CLAUDE_STATUS_RELAY_MODE=on`
- `CLAUDE_STATUS_AUQ_MODE=record`
- `CLAUDE_STATUS_LABEL_WAGGLE_OWNED=1`
- `CLAUDE_STATUS_LABEL_WAGGLE_SESSION_NAME=<session_name>`
- `CLAUDE_STATUS_LABEL_WAGGLE_MODEL=<lower-cased model>`
- `CLAUDE_STATUS_LABEL_WAGGLE_REPO=<repo>`

Launch Claude by sending `claude --model <model>
` to window 0, pane 0 via tmux send-keys. Return as soon as the launch command has been sent; do not wait for the first hook event and do not insert any placeholder into Claude Status.

`list_spawned_workers()` → `[{instance_id, session_name}]` per SR-3.3. Invoke `claude_status.workers(label="waggle_owned=1")` (Epic 1). Project each entry in `workers` to `{instance_id, session_name}` (the `session_name` is `labels.waggle_session_name`). For `skipped` rows, emit one structured stderr log line per row containing `instance_id`, `error` (`ErrName`), and `error_description`; do not include skipped rows in the returned list. No Waggle in-memory state; the result is recomputed on every call.

The existing daemon, HTTP surface, and old eight-tool MCP server are left untouched in this Epic — they continue to function in parallel. The new tools share `engine.py` helpers only insofar as they need tmux session creation; new code lives behind a clear seam so Epic 6 can delete the daemon without touching the new code path.

Tests: new files `tests/test_mcp_stdio_server.py` (transport bring-up; the server registers exactly the two tools defined so far; SR-7.1 error wrapping converts a raised exception to operation-failed), `tests/test_engine_spawn_worker.py` (returns `{instance_id, session_name}`; default session-name format; every required env var is passed to `tmux new-session -e`; `claude --model <model>` is sent to window 0/pane 0), `tests/test_engine_list_spawned_workers.py` (argv includes `--label waggle_owned=1`; skipped rows are stderr-logged but absent from the return; happy-path projection).

Initial README pass: a new "Stdio MCP launch model" section documents `waggle mcp` and the two new tools. The daemon section is left in place; the big rewrite happens in Epic 6.

## Acceptance Criteria

- After `pip install -e .` (or equivalent) the user can run `waggle mcp` and the process attaches to its stdio without binding any TCP/UDS port (verifiable with `lsof -p <pid>`).
- The user can drive the server with a minimal stdio MCP client and call `spawn_worker(model="sonnet", repo="/tmp/scratch")`; within 60s of the new worker's first hook event, `claude-status workers --label waggle_owned=1` shows the row with labels `waggle_owned=1`, `waggle_session_name`, `waggle_model=sonnet`, `waggle_repo=/tmp/scratch`.
- The user calls `list_spawned_workers()` and receives the `{instance_id, session_name}` pair for the worker spawned above.
- Restart survival: the user exits the stdio MCP server (Ctrl-D), relaunches `waggle mcp`, calls `list_spawned_workers()` again, and still sees the prior worker. Existing tmux session is unaffected by the server restart.
- All new tests pass; pre-existing Waggle tests remain green.
