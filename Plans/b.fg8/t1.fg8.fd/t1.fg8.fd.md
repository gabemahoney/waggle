## Description

Register `answer_question(question_id, answer)` on the stdio MCP server per SR-3.5, wrapped in the SR-7.1 error-translation envelope. `question_id` is the Claude Status `agent_request.request_id` (positive integer) the orchestrator extracted from `claude-status worker <id>` or `workers --label waggle_owned=1`. `answer` is a string; option-pick answers are stringified by the orchestrator (e.g. `"2"`).

Implementation order, exact:

1. Invoke `claude_status.workers(label="waggle_owned=1")` (Epic 1). From the `workers` array, find the row whose `pending.kind == "ask_user_question"` and `pending.request_id == question_id`. No match → operation-failed with `err_name` indicating no pending request.
2. If the matched row's `pending.questions` has length > 1 → operation-failed with `ErrMultiQuestionUnsupported`. v1 supports only single-question AUQs.
3. Extract `pending.questions[0].question` (the question text) and `labels.waggle_session_name`.
4. Capture window 0, pane 0 of the named tmux session via `tmux.py`'s capture-pane path with a 50-line scrollback.
5. Whitespace-normalize both captured pane text and the stored question text — collapse every run of whitespace (spaces, tabs, newlines) to a single space and strip leading/trailing — then substring-search. If the normalized question is not present in the normalized pane → operation-failed with `err_name` `ErrQuestionNoLongerVisible`. Normalization absorbs line-wrap that the TUI may introduce on narrow terminals.
6. Send `answer` followed by Enter (a single send-keys call that ends with the literal Enter key) into window 0, pane 0 via `tmux.py`'s send-keys path. Return success.

The TOCTOU window between step 5 and step 6 is acknowledged as unavoidable per SR-3.5; the orchestrator is responsible for re-reading state if a previous call returned `ErrQuestionNoLongerVisible`.

Tests: new `tests/test_engine_answer_question.py` covering: happy path (matched row, question visible, keystrokes sent); refusal when `pending.questions` has length 2 (`ErrMultiQuestionUnsupported`); refusal when no row's `pending.request_id` matches; refusal when the question text is absent from the captured pane (`ErrQuestionNoLongerVisible`); whitespace-normalization succeeds when the question wraps across two pane lines on a narrow terminal; the Enter is sent in the same send-keys call as the answer (not separately).

## Acceptance Criteria

- Integration test: the user spawns a worker via Epic 2; the worker hits a single-question `AskUserQuestion` (Claude Status records it under some `request_id N`); the user calls `answer_question(question_id=N, answer="2")`; Waggle validates the question text against the pane and types `2
`; the worker unblocks and `claude-status worker <id>` shows the `pending` for that worker cleared.
- Race test: the user spawns a worker, drives it to a single-question AUQ, then sends `clear
` to pane 0 (so the question scrolls out of the captured window); calling `answer_question(question_id=N, answer="2")` returns operation-failed with `err_name` `ErrQuestionNoLongerVisible`, and `tmux capture-pane` shows no new `2` was typed.
- Refusal test: the user drives a worker to a *multi*-question AUQ (two-question payload) and calls `answer_question`; the result is operation-failed with `err_name` `ErrMultiQuestionUnsupported`.
- All new tests pass; pre-existing Waggle tests remain green.

## Dependencies

Blocked by Epic 2 (the stdio MCP server, the Epic-1 client, and the `spawn_worker` env-var injection that causes workers to record their AUQs in Claude Status).
