## Context

`answer_question(question_id, answer)` per SR-3.5: a six-step algorithm composing Epic 1's `workers` client, Epic 3's `capture_pane_explicit` helper, and a new Enter-appending send-keys helper. Two Waggle-originated errors: `ErrMultiQuestionUnsupported`, `ErrQuestionNoLongerVisible`. The TOCTOU window between step 5 (substring check) and step 6 (send-keys) is acknowledged as unavoidable; orchestrator retries.

## What Needs to Change

- New name-keyed, window-0/pane-0, Enter-appending send-keys helper in `src/waggle/tmux.py` (`send_keys_with_enter` or similar; existing `_send_keys_sync` and Epic 3's `send_keys_no_enter` left untouched).
- Register `answer_question(question_id, answer)` on the stdio server from Epic `t1.fg8.vv`. Body executes the six steps; wrapped by that Epic's SR-7.1 wrapper.
- Update `tests/test_mcp_stdio_server.py` (introduced by Epic `t1.fg8.vv`) tool-count/tool-set assertion to include `answer_question`. This file is NOT pre-existing daemon-era; Epic 4 owns updating it.
- New test file `tests/test_engine_answer_question.py`.

## Constraints

- New tmux helper carries `# TODO(t1.fg8.bp): collapse with existing _send_keys_sync after daemon removal`. Existing helpers untouched.
- Tool body calls ONLY `waggle.claude_status.workers`. Does NOT call `worker`, `capabilities`, or `decide`.
- Whitespace normalization (collapse `\s+` to single space, strip leading/trailing) on BOTH the captured pane text and the stored question text before the substring match.
- Step 6 sends `answer + Enter` in ONE `pane.send_keys(..., enter=True)` call, not two calls.
- No retry. No lock. No re-check between steps 5 and 6.
- The two Waggle-originated err_names (`ErrMultiQuestionUnsupported`, `ErrQuestionNoLongerVisible`) documented in the stdio server module's docstring alongside the others from Epic 2.
- No modifications to pre-existing daemon-era test files.

## Acceptance Criteria

- `answer_question(question_id=7, answer="yes")` happy path: calls `workers(label="waggle_owned=1")`, finds matching row, captures pane 0:0 with scrollback 50, substring-matches with normalization, sends `"yes"` + Enter to `<session>:0.0`, returns operation-success.
- No matching row → operation-failed with no-pending-request err_name; no tmux interaction.
- Multi-question pending payload → operation-failed with `ErrMultiQuestionUnsupported`; no tmux interaction.
- Question text absent from pane → operation-failed with `ErrQuestionNoLongerVisible`; send-keys NOT called.
- Line-wrapped pane text (e.g. `"Should we
deploy now?"` matches question `"Should we deploy now?"`) still matches.
- TOCTOU non-mitigation: each `answer_question` call invokes `workers` once and `capture-pane` once.
- `pytest -q tests/test_engine_answer_question.py` is green.
- `tests/test_mcp_stdio_server.py` tool-set assertion includes `answer_question` (and remains green).
