## Context

SR-3.5 step 6 requires `answer_question` to send `answer + Enter` to window 0, pane 0 in ONE `send-keys` call. Existing `_send_keys_sync` is keyed by `session_id` and uses active pane; Epic 3's `send_keys_no_enter` is keyed by `session_name` but `enter=False`. Add the third variant: name-keyed, explicit pane 0:0, `enter=True`.

## What Needs to Change

In `src/waggle/tmux.py`, add a sync helper (e.g. `_send_keys_with_enter_sync`) and async wrapper (e.g. `send_keys_with_enter`) that:
- Take `(session_name, text)`.
- Resolve to a libtmux session, target window 0, pane 0 explicitly.
- Call `pane.send_keys(text, enter=True)` — Enter rides on the same `send_keys` call (not a separate invocation).
- Return the standard success/error dict.
- Carry a `# TODO(t1.fg8.bp): collapse with existing _send_keys_sync after daemon removal` comment.

Do not modify any existing helper.

## Key Files

- `src/waggle/tmux.py` — add the new helper

## Acceptance Criteria

- New helper passes `enter=True` in a single `send_keys` call (not two).
- Targets window 0, pane 0 explicitly (no active-pane fallback).
- Keyed by `session_name`.
- Existing `_send_keys_sync`, `send_keys`, `send_keys_no_enter` unchanged.
- TODO comment present.
- Missing session returns the error dict shape (no raise).
- Exported from `waggle.tmux`.
