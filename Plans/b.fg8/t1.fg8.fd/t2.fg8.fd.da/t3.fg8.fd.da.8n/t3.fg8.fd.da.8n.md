## Context

The six-step algorithm from SR-3.5, composing the Epic-1 client, Epic-3's capture-pane helper, and Subtask 1.1's Enter-appending send-keys helper.

## What Needs to Change

In the stdio-server module (Epic `t1.fg8.vv`), register `answer_question(question_id, answer)`. Body wrapped by Epic `t1.fg8.vv`'s SR-7.1 wrapper. Six steps in order:

1. **Workers query.** Call `claude_status.workers(label="waggle_owned=1")`. Operation-failed → propagate unchanged. Scan `workers` array for the row whose `pending` is non-None, `pending["kind"] == "ask_user_question"`, and `pending["request_id"] == question_id`. No match → operation-failed with no-pending-request err_name.
2. **Multi-question guard.** If matched row's `pending["questions"]` length > 1 → operation-failed with `err_name=ErrMultiQuestionUnsupported` (Waggle-originated).
3. **Extract.** `question_text = matched["pending"]["questions"][0]["question"]`; `session_name = matched["labels"]["waggle_session_name"]`.
4. **Capture pane.** Call `capture_pane_explicit(session_name, scrollback=50)` from Epic `t1.fg8.4n`. Error → propagate via wrapper.
5. **Whitespace-normalized substring check.** Normalizer: collapse every run of `\s+` to a single space, strip leading/trailing. Apply to BOTH the captured pane content and `question_text`. If normalized `question_text` not in normalized pane → operation-failed with `err_name=ErrQuestionNoLongerVisible` (Waggle-originated).
6. **Send.** Call `send_keys_with_enter(session_name, answer)` from Subtask 1.1. Return bare success value.

Also update `tests/test_mcp_stdio_server.py` tool-set assertion to include `answer_question` (this file is from Epic 2, not pre-existing daemon-era — in scope for additive modification).

Document `ErrMultiQuestionUnsupported` and `ErrQuestionNoLongerVisible` in the stdio server module's docstring.

## Constraints

- Tool body imports ONLY `waggle.claude_status.workers` (not `worker`, `capabilities`, `decide`).
- No retry. No lock. No re-check between steps 5 and 6.
- No `time.sleep` / `asyncio.sleep`.
- Bare return value; wrapper frames.

## Key Files

- `src/waggle/mcp_stdio.py` (or chosen path) — register `answer_question`
- `tests/test_mcp_stdio_server.py` — update tool-set assertion

## Acceptance Criteria

- Six steps execute in SR-3.5 order; verifiable from source.
- Step 1's argv includes `--label waggle_owned=1`.
- Step 5's normalization: line-wrap (`"X
Y"` matches `"X Y"`), tab→space (`"X	Y"` matches `"X Y"`), multiple spaces collapsed.
- Step 6 invokes the helper with the bare `(session_name, answer)` — Enter rides on the same call.
- Permission-request rows and `pending is None` rows do NOT match.
- Both Waggle-originated err_names documented in the module docstring.
- `tests/test_mcp_stdio_server.py` tool-set assertion now lists all six tools and remains green.
