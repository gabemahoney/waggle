## Context

Per SR-8.1, `config.py` is reduced to env-var helpers or deleted. The MCP server starts and operates with no `~/.waggle/config.json` file. All daemon-era keys are gone.

## What Needs to Change

- Audit `src/waggle/config.py`. What survives:
  - Env-var helpers (e.g. a tmux executable path override) — KEEP.
  - Reading `~/.waggle/config.json` or any other config file — REMOVE.
  - Daemon-era keys (`db_path`, `queue_path`, `http_port`, `relay_timeout_seconds`, `max_workers`, repo paths, CMA endpoints, admin emails) — REMOVE.
  - The `get_output` scrollback default is hardcoded to 50 in Epic 3's tool body, not an env-var knob — confirm `config.py` does not expose a scrollback setting.
- If nothing meaningful remains after the cleanup, delete `config.py` entirely and remove its imports from any remaining caller.
- Update `tests/test_config.py`:
  - If `config.py` survives, rewrite tests to cover the surviving surface only.
  - If `config.py` is deleted, delete `tests/test_config.py`.

## Constraints

- The stdio MCP server (`waggle mcp`) must start with no `~/.waggle/` directory present (`rm -rf ~/.waggle` first).
- No `~/.waggle/config.json` is read anywhere in Waggle source.

## Acceptance Criteria

- `grep -REn "db_path|queue_path|http_port|relay_timeout_seconds|max_workers|cma_endpoint|admin_email" src/waggle/` returns nothing.
- `grep -REn "config\.json|\.waggle/config" src/waggle/` returns nothing.
- On a host with no `~/.waggle/` directory at all, `waggle mcp --help` exits 0 and `waggle mcp` would start serving stdio (verifiable in a test).
- `pytest -q` is green.
