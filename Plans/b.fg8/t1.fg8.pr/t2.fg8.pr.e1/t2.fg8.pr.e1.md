## Context

`waggle sting` is preserved per SR-4.1 but its body is intentionally rewritten: it no longer scans Waggle's daemon/DB state. It introspects Claude Status's readiness via Epic 1's `capabilities()` verb. This is the Epic's explicit carve-out from the strict no-modifications-to-pre-existing-files rule: `sting.py`'s body and `tests/test_sting.py`'s body are both rewritten because the function's behavior is intentionally changing.

## What Needs to Change

- Rewrite `src/waggle/sting.py`:
  - Remove daemon-era helpers: `_detect_mcp`, `_has_waggle_in_mcp_servers`, `_key_matches_waggle`, `_WAGGLE_PATTERN`, `_CLI_REFERENCE`.
  - Implement new `handle_sting(args)`:
    1. `shutil.which("claude-status")` → if absent, print actionable red-status message naming the missing dependency, exit non-zero.
    2. Call `waggle.claude_status.capabilities()`. If operation-failed → print red-status message including the `err_name`, exit non-zero.
    3. On operation-success → parse `contract_version` major (leading integer before first `.`). Major == 1 → print non-empty green status line and exit 0. Otherwise → print red-status message indicating contract-version mismatch (include observed version), exit non-zero.
  - Do NOT import any daemon-era module.
- Rewrite `tests/test_sting.py` (Test Writer's Subtask owns this).

## Constraints

- `sting.py` calls `shutil.which` and `capabilities()` exactly once each per invocation; no other `claude_status` verbs invoked.
- Green status line is non-empty.
- No file/path scans of `~/.claude.json`, `~/.claude/settings.json`, or any other config.

## Acceptance Criteria

- `grep -RE '_detect_mcp|_has_waggle_in_mcp_servers|_key_matches_waggle|_WAGGLE_PATTERN|_CLI_REFERENCE|\.claude\.json|\.claude/settings\.json' src/waggle/sting.py` returns nothing.
- Missing `claude-status` → non-zero exit with actionable message.
- `capabilities()` operation-failed → non-zero exit with `err_name` surfaced.
- `contract_version` major 1 (e.g. `"1.0.0"`, `"1.7.3"`) → exit 0 with non-empty green status line.
- `contract_version` major 2 or 0 → non-zero exit with mismatch message.
- `pytest -q tests/test_sting.py` is green after the test rewrite.
