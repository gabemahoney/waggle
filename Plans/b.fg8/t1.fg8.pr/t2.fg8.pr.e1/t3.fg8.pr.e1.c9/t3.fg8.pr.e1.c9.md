The body rewrite removes symbols `test_sting.py` imports. Sanctioned by the Epic body: this is the carve-out from the "no modifications to pre-existing files" rule. The Engineer rewrites `sting.py`; the Test Writer rewrites `test_sting.py` to match.

## What Needs to Change

Rewrite `tests/test_sting.py`:
- Remove all imports of deleted symbols (`_CLI_REFERENCE`, `_WAGGLE_PATTERN`, `_detect_mcp`, `_has_waggle_in_mcp_servers`, `_key_matches_waggle`) and the test classes that exercised them.
- Import `handle_sting` from `waggle.sting` and `fake_claude_status` from `tests.helpers`.
- Tests for each of the four branches of the new `handle_sting`:
  1. Missing binary: patch `shutil.which` against the `sting` module to return `None`; assert non-zero `SystemExit` and actionable message.
  2. Green 1.x: stage `fake_claude_status` with the canonical 1.0.0 `capabilities` response from `sample_payloads`; assert `SystemExit` code 0 and non-empty green status line in captured output (`capsys`).
  3. Parameterized 1.x acceptance: cover `"1.0.0"`, `"1.7.3"` — all exit 0.
  4. Parameterized refusal: `"0.9.0"`, `"2.0.0"`, `"3.1.0"` — all non-zero exit with contract-mismatch message.
  5. Store unavailable: stage `capabilities` op-failed with `err_name=ErrStoreUnavailable` (envelope from `sample_payloads`); assert non-zero exit with `err_name` surfaced.

Use `pytest.raises(SystemExit)` to capture exit codes; `capsys` for output assertions. All test data from `tests.sample_payloads`. No inline JSON literals.
