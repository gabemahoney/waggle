## Context

`waggle sting`'s body is intentionally rewritten: no more daemon/DB scan. Use Epic 1's `capabilities()` client. Sanctioned modification of a pre-existing file per the Epic body.

## What Needs to Change

Rewrite `src/waggle/sting.py`:
- Remove daemon-era helpers: `_detect_mcp`, `_has_waggle_in_mcp_servers`, `_key_matches_waggle`, `_WAGGLE_PATTERN`, `_CLI_REFERENCE`.
- Implement `handle_sting(args)`:
  1. `shutil.which("claude-status")` PATH check → if absent, print actionable red-status message naming missing dependency, `sys.exit` non-zero.
  2. Call `waggle.claude_status.capabilities()`. Operation-failed → print red-status message including `err_name`, `sys.exit` non-zero.
  3. On operation-success → parse `contract_version` major (leading integer before first `.`; handle missing/malformed by treating as mismatch). Major == 1 → print non-empty green status line, exit 0. Otherwise → print red-status message including observed version, `sys.exit` non-zero.

No daemon-era module imports. No file/path scans of `~/.claude.json` / `~/.claude/settings.json`.

## Constraints

- `shutil.which` and `capabilities()` each called exactly once per invocation.
- No other `claude_status` verbs invoked.
- Green status line non-empty.

## Key Files

- `src/waggle/sting.py` — body rewrite

## Acceptance Criteria

- `grep -RE '_detect_mcp|_has_waggle_in_mcp_servers|_key_matches_waggle|_WAGGLE_PATTERN|_CLI_REFERENCE|\.claude\.json|\.claude/settings\.json' src/waggle/sting.py` returns nothing.
- Missing `claude-status` → non-zero exit, actionable message.
- `capabilities()` op-failed → non-zero exit, `err_name` surfaced.
- `contract_version` major 1 → exit 0, non-empty green line.
- Major 2 or 0 or missing/malformed → non-zero exit, mismatch message.
