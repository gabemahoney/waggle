## Description

Add the new `waggle install` CLI subcommand and refresh `waggle sting` to introspect Claude Status rather than Waggle's own state.

`waggle install`:
- Verifies that `claude-status` is on `PATH`. If absent, exits non-zero with an actionable message naming the missing dependency and pointing the operator at the Claude Status installation step (SR-5.1).
- Invokes `claude-status install-hooks` as a subprocess with `CLAUDE_STATUS_RELAY_MODE=on` and `CLAUDE_STATUS_AUQ_MODE=record` set in the subprocess environment. The hook installer itself reads those env vars when wiring `settings.json`, so they must be present in the subprocess `env` dict (SR-5.1).
- Forwards an optional `--auq-order=before:waggle|after:waggle|last` argument to `claude-status install-hooks`, supporting coexistence with any prior AUQ hook during transition (SR-5.1).
- Deletes `hooks/settings.json.template` and any Waggle-specific Claude Code hook bodies from the repo (SR-5.2). If `install.sh` referenced those files, update it accordingly (or, where appropriate, delete it — the new install path is purely `waggle install`).

`waggle sting` refresh:
- `sting.py` no longer checks Waggle's own DB or daemon. Instead it uses the Epic 1 client to call `claude-status capabilities` and report green/red based on (a) `claude-status` being on `PATH`, (b) the capabilities call succeeding, and (c) `contract_version` major matching `1`. Anything else is red with an actionable message.
- Update `tests/test_sting.py` to drive the new code path through the `fake_claude_status` fixture (Epic 1).

Tests: new `tests/test_install_hooks.py` covering: missing-`claude-status` exits non-zero with the right message; happy path invokes the subprocess with the expected env and argv (asserted via the subprocess seam); `--auq-order` is forwarded; the obsolete hook template files are deleted from the repo working copy under `tmp_path` checkout.

## Acceptance Criteria

- On a host where `claude-status` is not on `PATH` (e.g. `PATH=/usr/bin:/bin waggle install`), the command exits with a non-zero status and stderr names the missing `claude-status` dependency.
- On a host where `claude-status` is on `PATH`, `waggle install` exits 0; Claude Code's user-scope `~/.claude/settings.json` afterwards has hooks pointing at `claude-status hook` (verifiable with `jq '.hooks' ~/.claude/settings.json`); spawning a worker through Epic 2 and triggering a permission prompt confirms the hook fires Claude Status's relay (worker blocks until the operator decides).
- `waggle install --auq-order=last` results in the AUQ hook entry appearing last in the hooks list (verifiable in `settings.json`).
- `waggle sting` exits 0 on a green host and prints a non-empty status line. Replacing `claude-status` with a stub that returns capability `contract_version="2.0.0"` makes `waggle sting` exit non-zero with a message indicating contract-version mismatch.
- `find hooks/ -name 'settings.json.template'` returns nothing after the Epic merges.
- All new and updated tests pass; pre-existing Waggle tests remain green.
