## Context

Validate `waggle install`: PATH check, subprocess invocation argv + env, `--auq-order` forwarding, timeout handling, non-zero child propagation, and the SR-5.2 file-deletion side effect.

## What Needs to Change

Create `tests/test_install_hooks.py` covering:
- Missing binary: patch `shutil.which("claude-status")` to return `None`; invoke `waggle install`; assert non-zero exit and stderr names missing dependency.
- Happy path: patch `shutil.which` to return a path; patch `subprocess.run` to return success; assert `subprocess.run` called exactly once with argv `["claude-status", "install-hooks"]` (assert literal list, not substring); assert `env` parameter contains `CLAUDE_STATUS_RELAY_MODE=on` and `CLAUDE_STATUS_AUQ_MODE=record`; assert `timeout=10`.
- Parameterized `--auq-order` forwarding: `before:waggle`, `after:waggle`, `last` each appear verbatim in the recorded subprocess argv.
- `--auq-order=bogus` rejected at CLI layer (verifiable: `subprocess.run` never called, exit non-zero, usage error listing accepted values in stderr).
- Timeout: patch `subprocess.run` to raise `subprocess.TimeoutExpired`; assert non-zero exit, stderr contains clear timeout message, no traceback escapes (capture stderr, assert no `"Traceback"` substring).
- Non-zero child exit: patch `subprocess.run` to return non-zero `returncode` with stderr content; assert non-zero parent exit with the child stderr surfaced.
- File-deletion assertion: `(repo_root / "hooks" / "settings.json.template").exists()` is `False` (discover repo root via `__file__` parent chain).

No `conftest.py`. Use `unittest.mock.patch` for the subprocess seam and `shutil.which`.
