## Context

SR-5.1: `waggle install` verifies `claude-status` on PATH, invokes `claude-status install-hooks` as a subprocess with required env vars EXPLICITLY set on the child, forwards optional `--auq-order` flag. 10s subprocess timeout (consistent with `waggle.claude_status`).

## What Needs to Change

In `src/waggle/cli.py` (or new `src/waggle/install_hooks.py` if Engineer prefers a thin CLI), register the `install` subparser:
- Flag: `--auq-order` (optional). Validated at CLI layer; accepted values: `before:waggle`, `after:waggle`, `last`. Bogus value → usage error before any subprocess fires.
- Handler:
  - `shutil.which("claude-status")` PATH check; if absent → exit non-zero with actionable stderr.
  - Build subprocess argv: `["claude-status", "install-hooks"]` + `["--auq-order", <value>]` when supplied.
  - Build subprocess env: copy of `os.environ` with `CLAUDE_STATUS_RELAY_MODE=on` and `CLAUDE_STATUS_AUQ_MODE=record` EXPLICITLY set (not inherited).
  - `subprocess.run(argv, env=env, capture_output=True, text=True, timeout=10)`.
  - `TimeoutExpired` → non-zero exit + actionable stderr; no traceback.
  - Non-zero child exit → propagate; surface child stderr to operator.
  - Exit 0 on success.

Subprocess seam exposed as a module-level function so tests can patch cleanly.

## Constraints

- Existing `serve`, `set-state`, `permission-request`, `ask-relay`, `mcp`, `sting` subcommands unchanged.
- Handler does NOT import `waggle.claude_status`.
- Subprocess env is EXPLICITLY constructed (not relying on inherited environment).

## Key Files

- `src/waggle/cli.py` — register `install` subparser
- `src/waggle/install_hooks.py` (optional) — handler module

## Acceptance Criteria

- `waggle install --help` exits 0, documents `--auq-order` with three accepted values.
- Missing `claude-status` → non-zero exit with stderr naming missing dependency.
- Happy path: subprocess argv starts `claude-status install-hooks`; env contains both required keys; timeout = 10s.
- `--auq-order=last` forwards `--auq-order last` to child argv.
- `--auq-order=bogus` → usage error at CLI layer; no subprocess fires.
- `TimeoutExpired` → non-zero exit + actionable stderr; no traceback.
- Non-zero child exit → non-zero parent exit; child stderr surfaced.
- Existing subcommands unchanged.
