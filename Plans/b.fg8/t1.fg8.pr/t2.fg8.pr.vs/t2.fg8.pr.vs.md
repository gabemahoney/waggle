## Context

New `waggle install` subcommand per SR-5.1; delete Waggle's hook templates per SR-5.2. The subcommand verifies `claude-status` is on PATH, then invokes `claude-status install-hooks` as a subprocess with the required env vars explicitly set on the child.

## What Needs to Change

- Delete `hooks/settings.json.template` and any Waggle-specific hook body files. If `hooks/` becomes empty, delete the directory.
- Audit `install.sh` — leave any daemon-era branches (deleted in Epic `t1.fg8.bp`) but remove references to the deleted template; annotate surviving daemon-era logic with `# TODO(t1.fg8.bp): remove daemon-era logic when daemon is deleted`.
- New `waggle install` subcommand in `cli.py` (handler may live in `cli.py` or a new module like `src/waggle/install_hooks.py`):
  - `shutil.which("claude-status")` PATH check → exit non-zero with actionable stderr if absent.
  - Subprocess invocation: `subprocess.run(["claude-status", "install-hooks", ...], env=env, capture_output=True, text=True, timeout=10)` where `env` is a copy of `os.environ` with `CLAUDE_STATUS_RELAY_MODE=on` and `CLAUDE_STATUS_AUQ_MODE=record` explicitly set.
  - Optional `--auq-order` flag validated at CLI layer (accepted values: `before:waggle`, `after:waggle`, `last`). Forwarded verbatim when supplied; omitted from child argv if not.
  - `TimeoutExpired` → non-zero exit with stderr message; non-zero child exit → propagate with child stderr surfaced.
- New test file `tests/test_install_hooks.py`.

## Constraints

- Subprocess seam exposed as a module-level function so tests can patch it cleanly.
- Per-invocation timeout = 10s (consistent with `waggle.claude_status`).
- `env` is BUILT explicitly (copy + overwrite); not relying on inheritance.
- Existing `serve`, `set-state`, `permission-request`, `ask-relay`, `mcp`, `sting` subcommands unchanged.
- Handler does NOT import `waggle.claude_status` (installer is the layer below the consumer-CLI client).

## Acceptance Criteria

- `waggle install --help` exits 0 and documents `--auq-order` with the three accepted values.
- Missing `claude-status` on PATH → non-zero exit with stderr naming the missing dependency.
- Happy path: subprocess argv is `["claude-status", "install-hooks"]` (or with `--auq-order <value>`); env contains both required variables; timeout is 10s.
- Bogus `--auq-order` value rejected at the CLI layer with usage error before any subprocess fires.
- `TimeoutExpired` → non-zero exit with actionable stderr; no traceback.
- Non-zero child exit → non-zero parent exit; child stderr surfaced.
- `find hooks/ -name 'settings.json.template'` returns nothing.
- `grep -REn 'settings\.json\.template' . --exclude-dir=.git` returns nothing.
- `pytest -q tests/test_install_hooks.py` is green.
