## Context

The existing `_send_keys_sync(session_id, text)` hardcodes `enter=True` and is keyed by `session_id`. SR-3.4 requires `send_input` to type verbatim (no implicit Enter) and is keyed by `session_name`. The downstream `answer_question` tool in Epic `t1.fg8.fd` depends on send-keys being a transparent passthrough. To preserve existing daemon-era test coverage, add a new helper rather than mutating the existing one.

## What Needs to Change

In `src/waggle/tmux.py`, add a sync helper (e.g. `_send_keys_no_enter_sync`) and async wrapper (e.g. `send_keys_no_enter`) that:
- Take `(session_name, text)`.
- Resolve `session_name` to a libtmux session, target window 0, pane 0 explicitly (e.g. `session.windows[0].panes[0]` — not via the active-pane lookup).
- Call `pane.send_keys(text, enter=False)`.
- Return the standard success/error dict shape used elsewhere in `tmux.py`.
- Carry a `# TODO(t1.fg8.bp): collapse with existing _send_keys_sync after daemon removal` comment.

Do not modify `_send_keys_sync` / `send_keys`.

## Key Files

- `src/waggle/tmux.py` — add the new helper
