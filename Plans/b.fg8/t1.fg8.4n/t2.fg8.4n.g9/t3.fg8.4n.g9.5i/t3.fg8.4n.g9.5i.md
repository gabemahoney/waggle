## Context

With the new no-Enter send-keys helper in place (Subtask 1.1), register `send_input(session_name, text)` on the Epic `t1.fg8.vv` stdio server.

## What Needs to Change

In the stdio-server module, register `send_input(session_name, text)` wrapped by Epic `t1.fg8.vv`'s SR-7.1 wrapper. The body:
- Calls `send_keys_no_enter(session_name, text)` from `tmux.py` (Subtask 1.1).
- Returns a bare success value (the wrapper handles envelope framing).
- Does NOT call any `waggle.claude_status` verb.

## Key Files

- `src/waggle/mcp_stdio.py` (or chosen path from Epic `t1.fg8.vv`) — register `send_input`

## Acceptance Criteria

- Tool registered on the stdio server.
- Body invokes the new send-keys helper with the bare `(session_name, text)` pair.
- Body returns a bare success value; wrapper frames into operation-success.
- A libtmux-level error from the helper yields operation-failed via the wrapper.
- No import from `waggle.claude_status` in the `send_input` body.
