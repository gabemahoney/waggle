## Context

Validate every observable property of `send_input` at the unit-test level: verbatim text typing (no implicit Enter), explicit window-0/pane-0 targeting, multi-line input preservation, SR-7.1 wrapper integration on failure, and absence of Claude Status polling.

## What Needs to Change

Create `tests/test_engine_send_input.py` covering:
- `send_input(session_name="s", text="hello")` records send-keys text exactly `"hello"` — no trailing Enter, no newline, no whitespace mutation.
- Recorded send-keys target is `<session>:0.0` (window 0, pane 0) explicitly — not the session's active pane.
- Multi-line input (`"line1
line2"`) is sent verbatim in ONE send-keys call; the embedded newline is part of the typed text, not a structural separator.
- A libtmux exception (e.g. `LibTmuxException` for missing session) yields operation-failed per SR-7.1; no exception escapes.
- Patching `waggle.claude_status.worker` and `waggle.claude_status.workers` to raise on call demonstrates `send_input` does not invoke either.

Use the existing libtmux mock pattern from `tests/test_tmux.py` (`unittest.mock.patch` against `libtmux.Server`). No `conftest.py`.
