## Context

`send_input(session_name, text)` types `text` verbatim into window 0, pane 0 of the named tmux session. SR-3.4 mandates NO implicit Enter — the orchestrator controls whether Enter is part of the input. Downstream tools (notably `answer_question` in Epic `t1.fg8.fd`) rely on `send_input` being a transparent passthrough.

## What Needs to Change

- New name-keyed, no-Enter, window-0/pane-0-targeting send-keys helper in `src/waggle/tmux.py` (existing `_send_keys_sync` hardcodes `enter=True` and is keyed by `session_id` — leave it untouched until Epic `t1.fg8.bp`).
- Register `send_input(session_name, text)` on the stdio server from Epic `t1.fg8.vv`. Body wrapped by that Epic's SR-7.1 wrapper. Returns a bare success value.
- New test file `tests/test_engine_send_input.py`.

## Constraints

- New tmux helper passes `enter=False` to `pane.send_keys` and targets window 0, pane 0 explicitly (no active-pane fallback).
- New helper carries `# TODO(t1.fg8.bp): collapse with existing _send_keys_sync after daemon removal`.
- Tool body does NOT call any `waggle.claude_status` verb.
- No modifications of any kind to pre-existing test files.

## Acceptance Criteria

- `send_input(session_name="s", text="hello")` records send-keys text exactly `"hello"` (no trailing Enter, no newline).
- Recorded send-keys target is `<session>:0.0` explicitly.
- Multi-line text (`"line1
line2"`) is sent verbatim in one call — embedded newline preserved.
- Non-existent session yields operation-failed via SR-7.1 wrapper; no traceback escapes.
- Patching `claude_status.worker` and `claude_status.workers` to raise does NOT affect the send-input path.
- `pytest -q tests/test_engine_send_input.py` is green.
