## Description

Add three tmux-keyed MCP tools to the stdio server from Epic 2, each wrapped in the SR-7.1 error-translation envelope.

`send_input(session_name, text)` — type `text` (verbatim, no implicit Enter appended) into window 0, pane 0 of the named tmux session via `tmux.py`'s send-keys path (SR-3.4).

`get_output(session_name, scrollback=50)` — capture window 0, pane 0 via `tmux.py`'s capture-pane path. Clamp behavior: omit → 50 lines; `scrollback ∈ [1, 1000]` → use as-is; out-of-range → operation-failed (do not silently clamp). Return the unmodified text emitted by `tmux capture-pane` — no formatting, no munging. Claude Status's transcript file is not a source.

`terminate_worker(session_name)` — kill the named tmux session via `tmux.py`'s kill-session path (SR-3.4). Claude Status records the worker as `ended` independently via its normal lifecycle hooks.

Update `tmux.py`'s capture-pane helper to target window 0, pane 0 explicitly. Today it relies on the session's active pane; this Epic makes the targeting explicit (SR-3.4) so `get_output` and `answer_question` (Epic 4) have a stable contract.

Tests: `tests/test_engine_send_input.py` (text is typed verbatim; no implicit Enter; targets window 0/pane 0), `tests/test_engine_get_output.py` (window 0/pane 0 targeting; scrollback default; scrollback=0 and scrollback=1001 both refuse with operation-failed; raw-text passthrough), `tests/test_engine_terminate_worker.py` (kill-session invoked with the right name; absent session returns operation-failed). Existing `test_send_input.py` is reviewed; if it still covers daemon-only behavior, the still-applicable cases migrate into the new file and the rest stays put until Epic 6.

## Acceptance Criteria

- Using a worker spawned via Epic 2 in another stdio session: the user calls `send_input(session_name, text="hello")` and observing `tmux capture-pane -p -t <session>:0.0` shows `hello` appended to pane 0 with no trailing Enter.
- The user calls `get_output(session_name)` and the returned string matches `tmux capture-pane -p -t <session>:0.0` (last 50 lines).
- The user calls `get_output(session_name, scrollback=2000)` and receives an operation-failed result naming the out-of-range scrollback.
- The user calls `terminate_worker(session_name)`; `tmux has-session -t <session>` exits non-zero immediately; within seconds of the worker's final hook event, `claude-status worker <id>` reports status `ended`.
- All new tests pass; pre-existing Waggle tests remain green.
