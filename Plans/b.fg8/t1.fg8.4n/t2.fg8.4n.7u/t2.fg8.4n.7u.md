## Context

`terminate_worker(session_name)` kills the named tmux session. SR-3.4: Claude Status records the worker as `ended` independently via its lifecycle hooks — Waggle does NOT poll, insert placeholders, or wait.

## What Needs to Change

- New name-keyed kill-session helper in `src/waggle/tmux.py` (existing `_kill_session_sync` is keyed by `session_id` — left untouched).
- Register `terminate_worker(session_name)` on the stdio server from Epic `t1.fg8.vv`. Body wrapped by that Epic's SR-7.1 wrapper. Returns bare success value.
- New test file `tests/test_engine_terminate_worker.py`.

## Constraints

- New helper carries `# TODO(t1.fg8.bp): collapse with existing _kill_session_sync after daemon removal`.
- Tool body does NOT call any `waggle.claude_status` verb. No polling, no waiting, no `time.sleep` / `asyncio.sleep`.
- No modifications of any kind to pre-existing test files.

## Acceptance Criteria

- `terminate_worker(session_name="s")` invokes the name-keyed kill helper with `"s"` exactly once.
- Non-existent session yields operation-failed via SR-7.1 wrapper.
- Patching `claude_status.worker` and `claude_status.workers` to raise does NOT affect terminate path.
- Patching `time.sleep` and `asyncio.sleep` and asserting zero calls passes (no polling/waiting).
- `pytest -q tests/test_engine_terminate_worker.py` is green.
