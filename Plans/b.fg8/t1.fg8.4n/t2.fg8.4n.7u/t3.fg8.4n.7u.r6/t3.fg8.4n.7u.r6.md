## Context

SR-3.4 requires `terminate_worker(session_name)` to kill the named tmux session. The existing `_kill_session_sync(session_id)` is keyed by `session_id` (`$N`). Add a name-keyed helper rather than mutating the existing one.

## What Needs to Change

In `src/waggle/tmux.py`, add a sync helper (e.g. `_kill_session_by_name_sync`) and async wrapper (e.g. `kill_session_by_name`) that:
- Take `(session_name,)`.
- Resolve `session_name` to a libtmux session and call `.kill()` on it.
- Return the standard success/error dict shape (no raise on missing session).
- Carry a `# TODO(t1.fg8.bp): collapse with existing _kill_session_sync after daemon removal` comment.

Do not modify `_kill_session_sync` / `kill_session`.

## Key Files

- `src/waggle/tmux.py` — add the new helper

## Acceptance Criteria

- New helper keyed by `session_name`.
- Missing session returns the helper's error dict shape (no raise).
- Existing `_kill_session_sync` / `kill_session` unchanged.
- TODO comment present.
- Exported from `waggle.tmux`.
