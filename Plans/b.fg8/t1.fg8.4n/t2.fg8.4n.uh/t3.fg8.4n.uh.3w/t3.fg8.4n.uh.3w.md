## Context

With the new explicit-target capture-pane helper in place (Subtask 2.1), register `get_output(session_name, scrollback=50)` on the Epic `t1.fg8.vv` stdio server. SR-3.4 mandates scrollback validation BEFORE invoking the helper — outside `[1, 1000]` returns operation-failed (no silent clamping). Claude Status's transcript file is NOT a source.

## What Needs to Change

In the stdio-server module, register `get_output(session_name, scrollback=50)` wrapped by Epic `t1.fg8.vv`'s SR-7.1 wrapper. The body:
- Validates `scrollback`: must be an integer in `[1, 1000]` inclusive. Otherwise return operation-failed via the wrapper with a clear `err_name` (e.g. `ErrScrollbackOutOfRange`). The check happens BEFORE any tmux call.
- Calls `capture_pane_explicit(session_name, scrollback)` from `tmux.py` (Subtask 2.1).
- On success, returns the bare captured string. The wrapper handles envelope framing.
- Does NOT call any `waggle.claude_status` verb. Does NOT read Claude Status's transcript file.

## Key Files

- `src/waggle/mcp_stdio.py` (or chosen path from Epic `t1.fg8.vv`) — register `get_output`

## Acceptance Criteria

- Tool registered on the stdio server.
- `get_output(session_name="x")` defaults to scrollback 50.
- `scrollback=0`, `scrollback=1001`, `scrollback=-1`, non-integer scrollback → operation-failed with the chosen `err_name`. The capture-pane helper is NOT called in these cases.
- Body returns a bare string on success; wrapper frames into operation-success.
- A libtmux-level error from the helper yields operation-failed via the wrapper.
- No import from `waggle.claude_status` in the `get_output` body.
