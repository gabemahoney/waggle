## Context

SR-3.4 mandates `get_output` (and Epic 4's `answer_question`) capture window 0, pane 0 explicitly. The existing `_capture_pane_sync(session_id, pane_id, scrollback)` resolves to the session's active pane when `pane_id is None` — wrong contract for the stateless model.

## What Needs to Change

In `src/waggle/tmux.py`, add a sync helper (e.g. `_capture_pane_explicit_sync`) and async wrapper (e.g. `capture_pane_explicit`) that:
- Take `(session_name, scrollback)` — keyed by name, not id.
- Resolve `session_name` to a libtmux session and target window 0, pane 0 explicitly (no active-pane lookup).
- Call `pane.capture_pane(start=-scrollback)` matching the existing scrollback convention.
- Return the standard success/error dict shape; success payload's `content` field is the unmodified joined string of captured lines (`"
".join(...)`).
- Carry a `# TODO(t1.fg8.bp): collapse with existing _capture_pane_sync after daemon removal` comment.

Do not modify `_capture_pane_sync` / `capture_pane`.

## Key Files

- `src/waggle/tmux.py` — add the new helper

## Acceptance Criteria

- New helper targets window 0, pane 0 explicitly — no `active_window.active_pane` in code path.
- Preserves `start=-scrollback` convention.
- Existing `_capture_pane_sync` / `capture_pane` unchanged (`git diff` shows only additions).
- Returns unmodified pane text — no formatting, no stripping.
- TODO comment present.
- Exported from `waggle.tmux` so `get_output` and `answer_question` (Epic 4) can import.
