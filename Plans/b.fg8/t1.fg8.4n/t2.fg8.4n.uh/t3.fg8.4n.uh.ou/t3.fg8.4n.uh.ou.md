## Context

Validate `get_output`: explicit window-0/pane-0 targeting, scrollback default + clamping (with pre-invocation refusal), raw-text passthrough, SR-7.1 wrapper integration, and absence of Claude Status polling.

## What Needs to Change

Create `tests/test_engine_get_output.py` covering:

- Recorded capture-pane call targets window 0, pane 0 explicitly (verifiable through the libtmux mock's target argument).
- `get_output(session_name="s")` (omitted scrollback) → recorded capture-pane invocation uses scrollback 50.
- `scrollback=1` and `scrollback=1000` accepted (boundary). The capture-pane mock is called with the value.
- `scrollback=0`, `scrollback=1001`, `scrollback=-1` each return operation-failed AND the capture-pane mock was never called (pre-invocation refusal).
- Non-integer scrollback (e.g. `scrollback="50"`) returns operation-failed without calling the mock.
- Raw-text passthrough: stage capture-pane to return a recognizable string with embedded whitespace, trailing whitespace, and an ANSI escape sequence; assert `get_output` returns byte-for-byte equal.
- A libtmux exception (e.g. missing session) yields operation-failed per SR-7.1; no exception escapes.
- Patching `waggle.claude_status.worker` and `waggle.claude_status.workers` to raise on call demonstrates `get_output` does not invoke either; transcript file is not read.

Use the libtmux mock pattern from `tests/test_tmux.py`. No `conftest.py`.
