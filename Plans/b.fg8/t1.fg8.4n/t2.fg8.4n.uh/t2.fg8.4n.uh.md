## Context

`get_output(session_name, scrollback=50)` captures window 0, pane 0 explicitly of the named tmux session and returns the raw text. SR-3.4 mandates scrollback validation — `[1, 1000]` accepted, outside range returns operation-failed (NO silent clamping). The existing `_capture_pane_sync` relies on the session's active pane, which is the wrong contract for the stateless model.

## What Needs to Change

- New name-keyed, explicit-window-0/pane-0 capture-pane helper in `src/waggle/tmux.py` (existing helper untouched — daemon-era tests unaffected).
- Register `get_output(session_name, scrollback=50)` on the stdio server from Epic `t1.fg8.vv`. Body validates `scrollback` BEFORE invoking the helper; returns bare string on success.
- New test file `tests/test_engine_get_output.py`.

## Constraints

- New tmux helper targets window 0, pane 0 explicitly — no `active_window.active_pane` fallback.
- New helper carries `# TODO(t1.fg8.bp): collapse with existing _capture_pane_sync after daemon removal`.
- Scrollback validation refuses out-of-range BEFORE any tmux call — the libtmux mock must NOT be invoked for invalid input.
- Out-of-range scrollback returns operation-failed via SR-7.1 wrapper with a clear `err_name` (e.g. `ErrScrollbackOutOfRange`).
- Return value is unmodified text from `tmux capture-pane` — no formatting, no munging, no stripping.
- Tool body does NOT call any `waggle.claude_status` verb. Transcript file is NOT a source.
- No modifications of any kind to pre-existing test files.

## Acceptance Criteria

- `get_output(session_name="s")` defaults to scrollback 50.
- `scrollback=1` and `scrollback=1000` accepted (boundary).
- `scrollback=0`, `scrollback=1001`, and `scrollback=-1` each return operation-failed; libtmux capture-pane is never called.
- Capture-pane target is window 0, pane 0 explicitly.
- Raw text with embedded whitespace, trailing whitespace, and ANSI escapes round-trips byte-for-byte.
- Non-existent session yields operation-failed.
- Patching `claude_status.worker` and `claude_status.workers` to raise does NOT affect `get_output`.
- `pytest -q tests/test_engine_get_output.py` is green.
