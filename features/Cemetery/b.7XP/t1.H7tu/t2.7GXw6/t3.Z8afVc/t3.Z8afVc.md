## Context
spawn_agent needs three new functions in tmux.py: session creation, agent launch, and session resolution. These build on the existing tmux.py module from t1.3dyn and the LLM detection utility from SR-8.

## What Needs to Change
- `create_session(session_name, repo_path)`: uses `server.new_session(session_name=..., start_directory=..., attach=False)`, returns Session object or error dict. Catches `TmuxSessionExists`.
- `launch_agent_in_pane(pane, agent, model=None, settings=None)`: constructs CLI command (e.g., `claude --model sonnet`), sends via `pane.send_keys(cmd, enter=True)`.
- `resolve_session(session_name, repo_path)`: implements 4-case logic from SR-6.2. Uses `server.sessions.get()` for lookup, `pane_current_command` for LLM detection, path normalization for repo comparison. Returns `{action: "create"|"reuse"|"error", ...}`.
- All functions wrapped in `asyncio.to_thread()` for async safety (SR-1.2).
- All libtmux exceptions caught and converted to structured error dicts (SR-10).

## Key Files
- `src/waggle/tmux.py` — add three new functions

## Acceptance Criteria
- create_session() creates a tmux session at specified directory via libtmux
- launch_agent_in_pane() sends correct CLI command for claude/opencode with optional model and settings
- resolve_session() correctly handles all 4 SR-6.2 cases with normalized path comparison
- All functions async-safe via asyncio.to_thread()
- Errors returned as structured dicts, never raw exceptions
