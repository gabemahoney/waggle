## Context
spawn_agent needs low-level tmux functions to create sessions, launch LLM agents, and resolve session state. These build on the tmux.py module created in t1.3dyn and the LLM detection utility from SR-8.

## What Needs to Change
- Add `create_session()` to tmux.py: creates tmux session at repo path via `server.new_session(session_name=..., start_directory=..., attach=False)`
- Add `launch_agent_in_pane()` to tmux.py: sends claude/opencode CLI command to a pane with optional model and settings flags
- Add `resolve_session()` to tmux.py: implements 4-case session resolution logic (SR-6.2):
  1. Session doesn't exist → return {action: "create"}
  2. Session exists + LLM running → return {action: "error", message: "LLM already running in session"}
  3. Session exists + no LLM + same repo → return {action: "reuse", session: Session}
  4. Session exists + no LLM + different repo → return {action: "error", message: "session exists but is in wrong repo"}
- Reuse shared LLM detection utility (pane_current_command check from SR-8)
- All functions async-safe via asyncio.to_thread() (SR-1.2)
- Error handling: catch libtmux exceptions, return structured error dicts (SR-10)

## Success Criteria
- create_session() creates tmux session at specified directory
- launch_agent_in_pane() sends correct CLI command for claude/opencode with model/settings
- resolve_session() correctly handles all 4 SR-6.2 cases
- All libtmux calls wrapped in asyncio.to_thread()
- Errors returned as structured dicts, never raw exceptions
