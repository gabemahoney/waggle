## Context
In Claude Agent Teams multi-pane sessions, all teammates share the same tmux session. Without gating, each teammate's hook fires would overwrite the team lead's DB entry. This task adds an environment variable check to `set_state.sh` so only the team lead (or solo agents) update waggle's DB. The `CLAUDE_CODE_AGENT_TYPE` env var is set by Claude Code for each agent in a team.

## What Needs to Change
- Modify `hooks/set_state.sh` to check `CLAUDE_CODE_AGENT_TYPE` before writing to DB
  - Unset → write (solo agent, backward compatible)
  - `"team-lead"` → write
  - Anything else (e.g., `"teammate"`) → skip silently (exit 0, no error output)
- Add gating tests to `tests/test_hooks.py`

## Why
Without this gating, Agent Teams sessions corrupt waggle's DB because teammates overwrite the team lead's state entry.

## Success Criteria
- teammate skips DB write silently
- team-lead writes to DB normally
- Unset writes to DB normally (backward compatible)
- All tests pass
