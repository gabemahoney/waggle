## Outcome
In Claude Agent Teams multi-pane sessions, only the team lead's state is tracked in waggle's DB — teammates don't overwrite each other's entries.

## Work Items

- Modify `hooks/set_state.sh` to check `CLAUDE_CODE_AGENT_TYPE` environment variable before DB write
- Logic:
  - Variable unset → write to DB (solo agent, backward compatible)
  - Variable equals `"team-lead"` → write to DB
  - Variable equals anything else (e.g., `"teammate"`) → skip DB write silently
- No schema changes needed — hook gating is sufficient per research findings
- Update `tests/test_hooks.py` with new test cases for the gating logic
- Update `test_schema_conformance.py` if schema changes are needed (unlikely per design)

## References
- SRD: SR-9.3 (Team Lead DB Gating)
- `features/b.7XP/research-findings.md` section 1 (Multi-Pane / Agent Teams)

## Acceptance Criteria
1. `set_state.sh` called with `CLAUDE_CODE_AGENT_TYPE=teammate` does NOT write to DB
2. `set_state.sh` called with `CLAUDE_CODE_AGENT_TYPE=team-lead` DOES write to DB
3. `set_state.sh` called with `CLAUDE_CODE_AGENT_TYPE` unset DOES write to DB (backward compatible)
4. Skip is silent — no error output, exit 0
5. Tests pass: `poetry run pytest tests/test_hooks.py`
