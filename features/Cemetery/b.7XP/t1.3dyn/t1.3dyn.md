## Outcome
All existing tmux interaction uses libtmux. Zero `subprocess.run(["tmux"...])` calls remain in waggle.

## Work Items

- Add `libtmux` dependency to `pyproject.toml` (pinned minor version range — pre-1.0 library)
- Create `src/waggle/tmux.py` module with async-safe libtmux wrappers (all calls wrapped in `asyncio.to_thread()` — bundle sequential operations into single sync function)
- Implement shared LLM detection utility: `pane_current_command` equals `claude` or `opencode` (case-insensitive). Single tmux query, no process tree walking (SR-8)
- Migrate `list_agents` from `subprocess.run(["tmux", "list-sessions", "-F", ...])` to libtmux `server.sessions` enumeration
- Migrate `cleanup_dead_sessions` to libtmux session enumeration
- Remove constants: `TMUX_FMT_WITH_PATH`, `TMUX_FMT_KEYS_ONLY`, `TMUX_FIELD_SEP`
- Error handling: catch `LibTmuxException` subclasses AND `QueryList.get()` exceptions (known issue — may not inherit from base). Graceful tmux-unavailable handling
- Update existing tests in `test_server.py` to mock libtmux instead of `subprocess.run`
