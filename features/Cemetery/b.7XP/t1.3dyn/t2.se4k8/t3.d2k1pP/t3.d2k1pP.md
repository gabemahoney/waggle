## Context
The tmux enrichment block in `list_agents` (~lines 218-253 of `server.py`) makes a `subprocess.run(["tmux", "list-sessions", "-F", TMUX_FMT_WITH_PATH])` call, parses the tab-separated output into a lookup map keyed by composite key, then enriches DB agents with `directory`. This needs to be replaced with the session enumeration function from `waggle.tmux`.

## What Needs to Change
- In `src/waggle/server.py`:
  - Import the session enumeration function from `waggle.tmux`
  - Replace the `subprocess.run(["tmux", "list-sessions", "-F", TMUX_FMT_WITH_PATH])` block with a call to the tmux.py function
  - Build the `tmux_sessions` lookup map from the tmux.py return value instead of parsing format strings
  - Remove the `subprocess.TimeoutExpired`, `subprocess.CalledProcessError`, `FileNotFoundError` exception handlers — replace with handling for whatever tmux.py returns on failure (empty results)
  - Preserve: agents with no tmux match get `directory: None`

## Key Files
- `src/waggle/server.py`

## Acceptance Criteria
- `list_agents` returns same response structure as before
- DB agents enriched with `directory` when tmux session exists
- `directory` is `None` when tmux unavailable or session not found
- No `subprocess.run` calls in the list_agents function
