## Context
`list_agents` in `server.py` uses `subprocess.run(["tmux", "list-sessions", "-F", TMUX_FMT_WITH_PATH])` to enrich DB-sourced agents with tmux session paths. This task replaces that subprocess block with the session enumeration function from `tmux.py`. The behavioral contract is preserved: DB is source of truth, tmux provides enrichment, tmux failures are non-fatal.

## What Needs to Change
- Replace the subprocess tmux call block in `list_agents` (the enrichment section ~lines 218-253) with a call to `tmux.py`'s session enumeration function
- Update the enrichment loop to use the data structure from `tmux.py` instead of parsing format strings
- Remove subprocess exception handling (TimeoutExpired, CalledProcessError, FileNotFoundError) — tmux.py handles errors internally
- Preserve: DB as source of truth, directory enrichment, non-fatal tmux failures, filtering by name/repo
- Update all list_agents tests to mock tmux.py instead of subprocess.run

## Why
Migrates the first of two existing tools to libtmux, moving toward zero subprocess tmux calls.

## Success Criteria
- `list_agents` returns same response structure as before
- DB agents enriched with `directory` when tmux session exists, `None` when unavailable
- No subprocess calls in the list_agents function
- All list_agents tests pass with updated mocks
- `poetry run pytest` passes
