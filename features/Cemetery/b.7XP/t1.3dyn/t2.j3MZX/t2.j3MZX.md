## Context
`cleanup_dead_sessions` uses `subprocess.run(["tmux", "list-sessions", "-F", TMUX_FMT_KEYS_ONLY])` to enumerate active sessions. After this migration, zero subprocess tmux calls will remain in waggle's source, and all legacy tmux constants can be removed. This completes the Epic.

## What Needs to Change
- Replace subprocess call in `cleanup_dead_sessions` with tmux.py's active session key function
- Remove all legacy tmux infrastructure: `TMUX_FIELD_SEP`, `TMUX_FMT_WITH_PATH`, `TMUX_FMT_KEYS_ONLY` constants
- Remove `import subprocess` from server.py
- Preserve: orphan detection logic, batch deletion, silent failure pattern (never raises)
- Update all cleanup_dead_sessions tests to mock tmux.py instead of subprocess.run
- Remove legacy test infrastructure (SEP constant, mock_tmux_subprocess fixture)
- Final verification: all 5 Epic acceptance criteria met

## Why
Completes the migration. After this task, waggle has a single, consistent libtmux interface with zero subprocess tmux calls.
