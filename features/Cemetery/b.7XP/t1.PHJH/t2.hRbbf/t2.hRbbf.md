## Context
close_session is the MCP tool for cleanly shutting down agent sessions with safety guards. It combines DB validation, name/ID cross-validation, LLM protection via force flag, and DB-first cleanup ordering.

## What Needs to Change
- Add `close_session` tool to server.py using @mcp.tool() decorator
- Parameters (SR-7.1): session_id (required), session_name (optional), force (optional, default false)
- DB validation (SR-7.2): session_id must exist in waggle DB
- Name/ID cross-validation (SR-7.2): if both provided, verify they refer to same session via validate_session_name_id()
- LLM protection (SR-7.3): check pane_current_command via SR-8 shared utility (is_llm_running or equivalent). LLM running + force=false → error: "Active LLM agent, call again with force=true to confirm". LLM running + force=true → proceed.
- Cleanup ordering (SR-7.4): remove DB entry FIRST, then kill tmux session. If tmux kill fails after DB removal → error indicating DB cleaned but session may still be alive.
- Return contract (SR-7.5): {status, message}
- All error paths return structured dicts per SR-10.1

## Success Criteria
- close_session kills session and removes DB entry
- Refuses without force when LLM active with exact error message
- Proceeds with force=true when LLM active
- Name/ID mismatch rejected with descriptive error
- DB-first cleanup ordering: DB cleaned even if tmux kill fails
- Tests pass: poetry run pytest tests/test_server.py -k close_session
