## Outcome
An orchestrator can cleanly shut down agent sessions with safety guards for active LLM agents.

## Work Items

- Implement `close_session` MCP tool in `server.py`, delegating to `tmux.py`
- Parameters:
  - `session_id` (required): tmux session ID
  - `session_name` (optional): tmux session name for disambiguation
  - `force` (optional, default false): required when session has active LLM
- Validation:
  - `session_id` must exist in waggle DB
  - If both `session_name` and `session_id` provided: verify they refer to same session, error if mismatch
- LLM protection (SR-7.3):
  - Check `pane_current_command` via LLM detection utility (from Epic 1)
  - LLM running + `force=false` → error: "Active LLM agent, call again with force=true to confirm"
  - LLM running + `force=true` → proceed
- Cleanup ordering (SR-7.4):
  - Remove DB entry FIRST
  - Then kill tmux session
  - If tmux kill fails after DB removal: return error indicating DB cleaned but session may still be alive
- Return contract: `{status, message}`
- Create tool tests

## Acceptance Criteria
1. `close_session` kills session and removes DB entry in mocked environment
2. Refuses without `force` when LLM is active — clear error message
3. Proceeds with `force=true` when LLM is active
4. Name/ID mismatch rejected with descriptive error
5. DB-first cleanup ordering: DB cleaned even if tmux kill fails
6. Tests pass: `poetry run pytest tests/test_server.py -k close_session`
