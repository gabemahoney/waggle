## Context
close_session needs two tmux.py utilities: one to kill sessions and one to cross-validate session name/ID pairs. These build on the existing tmux.py module from t1.3dyn.

## What Needs to Change
- `kill_session(session_id)`: resolves session via `server.sessions.get(session_id=...)`, calls `session.kill_session()`. Returns `{status: "success"}` or `{status: "error", message: ...}`. Catches `LibTmuxException` subclasses AND `QueryList.get()` exceptions (known libtmux issue #541).
- `validate_session_name_id(session_id, session_name)`: looks up session by ID, compares `session.session_name` against provided name. Returns success if match, error with descriptive message if mismatch or session not found.
- Both wrapped in `asyncio.to_thread()` (SR-1.2)

## Key Files
- `src/waggle/tmux.py` — add two functions

## Acceptance Criteria
- kill_session() terminates session via libtmux, returns structured dict
- validate_session_name_id() correctly detects match/mismatch
- Both handle missing sessions gracefully
- Async-safe, all exceptions caught
