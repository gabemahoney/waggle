## Context
close_session needs low-level functions for killing tmux sessions and cross-validating session name/ID pairs. These extend the tmux.py module from t1.3dyn.

## What Needs to Change
- Add `kill_session(session_id)` to tmux.py: resolves session via `server.sessions.get(session_id=...)`, calls `session.kill_session()`. Returns structured success/error dict. Catches libtmux exceptions including QueryList.get() exceptions (SR-10.1).
- Add `validate_session_name_id(session_id, session_name)` to tmux.py: looks up session by ID, compares resolved name against provided name. Returns success if match, error if mismatch or session not found.
- Both functions async-safe via asyncio.to_thread() (SR-1.2)
- All libtmux exceptions caught and converted to structured error dicts

## Success Criteria
- kill_session() terminates a tmux session via libtmux
- validate_session_name_id() correctly validates matching and mismatching pairs
- All functions async-safe, errors returned as structured dicts
