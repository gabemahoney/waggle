## Context
Each fixture represents what `pane.capture_pane()` returns for a real Claude Code session in a specific state. The PRD (features/b.7XP/b.7XP.md) provides example text for AskUserQuestion and CheckPermission. For Working and Done states, construct realistic snapshots based on the research findings detection patterns (features/b.7XP/research-findings.md section 3): Working has `Esc to interrupt`, Done has the `>` input prompt at the bottom.

## What Needs to Change
- Create `tests/fixtures/pane_snapshots/` directory
- Create fixture files:
  - `working.txt` — pane content showing an agent actively generating output, with `Esc to interrupt` indicator visible
  - `done.txt` — pane content showing an agent idle with the `>` input prompt at the bottom
  - `ask_user.txt` — pane content showing an AskUserQuestion prompt with `❯` selector, numbered options, and `───` separator (use PRD example as reference)
  - `check_permission.txt` — pane content showing a CheckPermission prompt with `Do you want to proceed?` and `Permission rule` text (use PRD example as reference)
  - `unknown.txt` — pane content that doesn't match any known state (e.g., a plain shell prompt, or a crashed agent)
- Each fixture should include realistic surrounding context (agent output above the prompt, whitespace, etc.) — not just the minimal matching pattern
