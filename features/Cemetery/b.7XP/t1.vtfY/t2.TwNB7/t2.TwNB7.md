## Context
send_command is the tool that lets an orchestrator talk to downstream agents. It combines pre-send state validation, state-aware formatting, and delivery verification to ensure reliable command delivery.

## What Needs to Change
- Implement `send_command` MCP tool in `server.py`
- Parameters: session_id (required), command (required), pane_id (optional)
- Pre-send: read pane state, refuse working ("agent is busy") and unknown ("agent state unknown, cannot safely send") — exact error messages per SR-5.2
- State-aware formatting (SR-5.5): done → freeform, ask_user → validated option number, check_permission → validated "1"/"2"
- Input sequence: Ctrl+C clear + command + Enter (SR-5.3)
- Delivery verification: poll state transition, 5s timeout, NO automatic retry (SR-5.4)
- Return: {status, message} (SR-5.6)
- Add comprehensive tests to test_server.py

## Success Criteria
- Delivers text to done-state agent
- Refuses working/unknown with exact SR-5.2 error messages
- Validates ask_user option numbers (valid accepted, invalid rejected)
- Validates check_permission ("1"/"2" accepted, others rejected)
- Delivery verification detects transition within 5s, returns error on timeout without retry
- All tests pass including full suite
