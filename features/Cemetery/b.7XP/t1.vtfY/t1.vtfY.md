## Outcome
An orchestrator can send text to a Done agent, answer AskUserQuestion prompts with option numbers, and respond to CheckPermission prompts.

## Work Items

- Implement `send_command` MCP tool in `server.py`, delegating to `tmux.py`
- Parameters:
  - `session_id` (required): tmux session ID
  - `command` (required): text to send
  - `pane_id` (optional): tmux pane ID for multi-pane targeting
- Pre-send validation:
  - `session_id` must exist in waggle DB
  - Read pane state via `read_pane` logic. Refuse if `working` (error: "agent is busy") or `unknown` (error: "agent state unknown, cannot safely send")
- Input sequence:
  - Send Ctrl+C to clear any partial input
  - Send command text + Enter keystroke
- State-aware formatting (SR-5.5):
  - `done`: send freeform text as-is
  - `ask_user`: caller sends option number string (e.g., `"1"`). Validate against `prompt_data` options, then send keystroke
  - `check_permission`: caller sends `"1"` (Yes) or `"2"` (No). Validate and send keystroke
- Delivery verification:
  - Poll pane state after sending to confirm transition (e.g., `done` → `working`)
  - Timeout after 5 seconds — return error on failure
  - No automatic retry
- Return contract: `{status, message}`
- Create tool tests — mock tmux layer

## Acceptance Criteria
1. `send_command` delivers text to a mocked agent in `done` state
2. Refuses with clear error when agent is `working` or `unknown`
3. Validates option numbers for `ask_user` state against prompt_data — rejects invalid options
4. Validates `check_permission` responses (only `"1"` or `"2"` accepted)
5. Delivery verification detects state transition within 5s timeout
6. Tests pass: `poetry run pytest tests/test_server.py -k send_command`
