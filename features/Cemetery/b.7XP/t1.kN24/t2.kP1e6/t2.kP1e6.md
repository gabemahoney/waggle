## Context
The `read_pane` MCP tool needs to capture pane content and validate pane targeting. The tmux.py module (created in Epic t1.3dyn) must be extended with pane-level operations. These functions will also be used by `send_command` (Epic t1.vtfY).

## What Needs to Change
- Add pane content capture function to `src/waggle/tmux.py`:
  - Accept session_id and optional pane_id
  - Use `pane.capture_pane()` with scrollback range support (SR-4.1: default 50, no waggle-imposed max)
  - When pane_id omitted: target active pane of session's active window (SR-9.1)
  - When pane_id provided: target that specific pane via `server.panes.get(pane_id=...)` (SR-9.1)
  - Return raw pane text as string (joined from list of lines)
- Add pane validation function:
  - Validate pane_id exists within specified session (SR-9.2)
  - Return descriptive error if pane not found or doesn't belong to session
- Async-safe: wrap with `asyncio.to_thread()` following existing pattern
- Error handling: catch libtmux exceptions, return structured errors
- Add tests to test_tmux.py

## Success Criteria
- Pane capture returns content with configurable scrollback depth
- Active pane targeted when pane_id omitted
- Specific pane targeted when pane_id provided
- Descriptive error for invalid pane_id
- Tests pass
