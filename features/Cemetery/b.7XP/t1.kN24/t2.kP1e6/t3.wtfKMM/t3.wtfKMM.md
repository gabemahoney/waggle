## Context
The tmux.py module (created in Epic t1.3dyn) currently has session enumeration and LLM detection. It needs pane-level operations for the read_pane and send_command tools. The pane capture function wraps libtmux's `pane.capture_pane()` with scrollback control and pane targeting logic.

## What Needs to Change
- In `src/waggle/tmux.py`, add functions for:
  - **Pane content capture**: given a session_id and optional pane_id, capture pane content with configurable scrollback. Use `pane.capture_pane(start=-scrollback)` which returns a list of strings. Join into single string for return.
  - **Pane resolution**: when pane_id provided, find pane via `server.panes.get(pane_id=pane_id)` and validate it belongs to the session. When omitted, use `session.active_window.active_pane`.
  - **Pane validation**: verify pane_id exists within the session. Return descriptive error if pane not found or belongs to a different session.
  - **Session lookup**: find session by session_id (e.g., `$0`) using `server.sessions.get(session_id=session_id)`.
- All functions async-safe via `asyncio.to_thread()` following existing module pattern
- Error handling: catch libtmux exceptions, return structured error dicts

## Key Files
- `src/waggle/tmux.py`

## Acceptance Criteria
- Pane capture returns string content with correct scrollback depth
- Active pane used when pane_id omitted
- Specific pane targeted when pane_id provided
- Descriptive error returned for invalid/non-existent pane_id
- Descriptive error returned for pane belonging to different session
- Session lookup by session_id works correctly
