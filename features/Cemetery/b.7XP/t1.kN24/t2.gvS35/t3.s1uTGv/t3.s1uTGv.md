## Context
read_pane is the first of the 4 new MCP tools. It combines pane capture (tmux.py) with state classification (state_parser.py) to give orchestrators structured insight into agent state.

## What Needs to Change
- In `src/waggle/server.py`:
  - Add `@mcp.tool()` decorated async function `read_pane`
  - Parameters: `session_id` (required str), `pane_id` (optional str), `scrollback` (optional int, default 50)
  - Validate session_id against waggle DB — return error if not registered
  - Call tmux.py pane capture function (handles pane_id targeting internally)
  - Pass captured content to state_parser.py's parse function
  - Return dict: `{status: "success", agent_state: str, content: str, prompt_data: dict|None}`
  - On validation/capture failure: return `{status: "error", message: str}`

## Key Files
- `src/waggle/server.py`

## Acceptance Criteria
- read_pane is registered as an MCP tool
- Returns correct state classification for each of the 5 states
- content field always populated (even for unknown state)
- prompt_data populated for ask_user and check_permission, None otherwise
- Unregistered session_id returns clear error
- Invalid pane_id returns descriptive error
