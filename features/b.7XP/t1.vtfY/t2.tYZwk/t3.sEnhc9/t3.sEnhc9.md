## Context
The tmux.py module (created in t1.3dyn) already has pane capture and validation functions. This subtask adds the write side: sending keystrokes and clearing input.

## What Needs to Change
- Add `send_keys_to_pane()` async function to `src/waggle/tmux.py`
  - Accepts session_id, text, optional pane_id, optional enter flag (default True)
  - Resolves pane using existing pane resolution logic from t1.kN24
  - Uses `pane.send_keys(text, enter=enter)` via libtmux
  - Wrapped in `asyncio.to_thread()` following existing module pattern
- Add `clear_pane_input()` async function
  - Sends Ctrl+C to the target pane to clear any partial input (SR-5.3)
  - Uses `pane.send_keys('C-c', enter=False)` or equivalent
- Both functions must catch libtmux exceptions and return structured error dicts
- Both functions must validate session_id against waggle DB before acting

## Key Files
- `src/waggle/tmux.py` — add new functions

## Acceptance Criteria
- `send_keys_to_pane()` delivers text + Enter to a pane via libtmux
- `clear_pane_input()` sends Ctrl+C to clear partial input
- Pane targeting works with specific pane_id or falls back to active pane
- libtmux exceptions are caught and returned as structured `{status: "error", message: ...}` dicts
- Async-safe: all libtmux calls wrapped in `asyncio.to_thread()`
