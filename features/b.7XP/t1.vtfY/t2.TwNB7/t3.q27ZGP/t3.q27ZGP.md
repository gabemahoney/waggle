## Context
send_command is the MCP tool that enables an orchestrator to send text to downstream agents. It combines pre-send state validation, state-aware formatting, input clearing, and delivery verification.

## What Needs to Change
- Add `send_command` tool to `server.py` using `@mcp.tool()` decorator
- Parameters: session_id (required), command (required), pane_id (optional)
- Pre-send validation:
  - Validate session_id against waggle DB — return error if not registered
  - Read pane state before sending
  - Refuse if `working` state — exact error: "agent is busy" (SR-5.2)
  - Refuse if `unknown` state — exact error: "agent state unknown, cannot safely send" (SR-5.2)
- State-aware formatting (SR-5.5):
  - `done`: send freeform text as-is
  - `ask_user`: validate command is a valid option number against prompt_data options, then send keystroke
  - `check_permission`: validate command is "1" (Yes) or "2" (No), then send keystroke
- Input sequence: call clear_pane_input() then send_keys_to_pane() from tmux.py (SR-5.3)
- Delivery verification: poll pane state after sending to confirm transition (e.g., done → working). Timeout after 5 seconds. Return error on timeout. No automatic retry (SR-5.4)
- Return contract: {status: "success"|"error", message: str} (SR-5.6)
- Delegate tmux operations to tmux.py module functions

## Key Files
- `src/waggle/server.py` — add send_command tool

## Acceptance Criteria
- send_command delivers text to a done-state agent
- Refuses working state with exact "agent is busy" error message
- Refuses unknown state with exact "agent state unknown, cannot safely send" error message
- Validates ask_user option numbers — accepts valid options, rejects invalid with descriptive error
- Validates check_permission responses — accepts only "1" or "2", rejects others
- Delivery verification polls for state transition within 5s
- Returns timeout error without retry when verification fails
- Return format matches {status, message} contract
