## Context
Waggle currently uses raw `subprocess.run(["tmux", ...])` with manual format string parsing for all tmux interaction. This task creates the foundational `src/waggle/tmux.py` module using libtmux, establishing the async-safe wrapper pattern that all tmux interactions (migrated and new) will build on.

## What Needs to Change
- Add `libtmux` as a runtime dependency in `pyproject.toml` (pinned minor version range — pre-1.0 library)
- Create `src/waggle/tmux.py` with:
  - Async-safe wrapper pattern: bundle synchronous libtmux calls into single functions, wrap with `asyncio.to_thread()` (SR-1.2)
  - Session enumeration: function to list all sessions with their paths (replaces `TMUX_FMT_WITH_PATH` subprocess pattern)
  - Active session key building: function to get composite keys of live sessions (replaces `TMUX_FMT_KEYS_ONLY` subprocess pattern)
  - LLM detection utility: check `pane_current_command` for `claude` or `opencode` (case-insensitive). Shared utility per SR-8 — no process tree walking
  - Error handling: catch `LibTmuxException` subclasses AND `QueryList.get()` exceptions (known issue #541). Return structured responses (SR-10.1)
  - Graceful tmux-unavailable: if tmux not running or not installed, return empty results or clear error (SR-10.2)

## Why
This module is the foundation for all 4 new MCP tools (read_pane, send_command, spawn_agent, close_session) and the migration of 2 existing tools. Every subsequent Epic depends on it.

## Success Criteria
- Module imports cleanly with no side effects
- `poetry install` succeeds with libtmux
- Session enumeration returns correct data when tmux available, empty results when unavailable
- LLM detection correctly identifies claude/opencode pane commands
- `poetry run pytest` passes (all tests including new test_tmux.py)
