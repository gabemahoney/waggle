## Outcome
An orchestrator can programmatically launch a Claude or OpenCode agent in a tmux session and optionally deliver an initial command after the agent is ready.

## Work Items

- Implement `spawn_agent` MCP tool in `server.py`, delegating to `tmux.py`
- Parameters:
  - `repo` (required): directory path of the repo
  - `session_name` (required): tmux session name
  - `agent` (required): `claude` or `opencode`
  - `model` (optional): Haiku, Sonnet, or Opus
  - `command` (optional): initial command to deliver after agent readiness
  - `settings` (optional): CLI parameters (e.g., `--dangerously-skip-permissions`)
- Session resolution logic (SR-6.2):
  - `session_name` doesn't exist → create new session at `repo` path, launch LLM
  - `session_name` exists + LLM running → error: "LLM already running in session"
  - `session_name` exists + no LLM + same repo → launch LLM in existing session
  - `session_name` exists + no LLM + different repo → error: "session exists but is in wrong repo"
- DB registration: register agent immediately after session creation (session_id, session_name, repo, agent type) (SR-6.5)
- Command delivery behavior (SR-6.3):
  - Without `command`: return immediately after launch
  - With `command`: poll `read_pane` until `done` state (60s timeout), then deliver via `send_command`
  - Timeout returns error with last known agent state
- Return contract: `{status, session_id, session_name, message}`
- Create tool tests
