## Context
The spawn_agent MCP tool orchestrates session resolution, agent launch, and DB registration. This is the core tool implementation without the command delivery polling.

## What Needs to Change
- Add `spawn_agent` to server.py with `@mcp.tool()` decorator
- Parameters: repo (str, required), session_name (str, required), agent (str, required: "claude"/"opencode"), model (Optional[str]), command (Optional[str]), settings (Optional[str]), ctx (Context)
- Validation: normalize repo path, validate agent is claude/opencode (case-insensitive)
- Call resolve_session() from tmux.py, handle create/reuse/error actions
- On create: call create_session() then launch_agent_in_pane()
- On reuse: call launch_agent_in_pane() on existing session's active pane
- On error: return error dict immediately
- DB registration (SR-6.5): immediately after session creation, insert composite key `{session_name}+{session_id}+{session_created}` with initial status "working"
- Return contract (SR-6.4): all paths return {status, session_id, session_name, message}
- Top-level exception handling per SR-10.1
