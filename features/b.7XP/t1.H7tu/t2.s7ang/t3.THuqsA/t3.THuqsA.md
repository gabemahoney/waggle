## Context
When spawn_agent is called with a `command` parameter, it must wait for the agent to reach `done` state before delivering the command (SR-6.3). This requires a polling loop with timeout.

## What Needs to Change
- In spawn_agent tool, after agent launch and DB registration:
  - If no command: return immediately with success
  - If command provided: enter polling loop (max 60 seconds, 1s interval)
    - Call read_pane logic to get agent state
    - If state == "done": deliver command via send_command, return success
    - Otherwise: continue polling
  - On timeout: return error with last known agent state: "Agent readiness timeout after 60s. Last state: {state}"
- Reuse read_pane and send_command tool logic (call the tmux.py and state_parser functions directly, not the MCP tools)

## Key Files
- `src/waggle/server.py` — add polling logic to spawn_agent

## Acceptance Criteria
- Without command: returns immediately after launch
- With command: polls at 1s intervals until done state
- Delivers command via send_command logic when agent ready
- Returns error on 60s timeout with last known pane state included in message
- No automatic retry on timeout
