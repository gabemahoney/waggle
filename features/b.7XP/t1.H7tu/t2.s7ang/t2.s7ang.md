## Context
spawn_agent is the MCP tool that lets an orchestrator programmatically launch agents. It orchestrates session resolution, agent launch, DB registration, and optional command delivery with readiness polling.

## What Needs to Change
- Add `spawn_agent` tool to server.py using @mcp.tool() decorator
- Parameters (SR-6.1): repo (required), session_name (required), agent (required: claude/opencode), model (optional), command (optional), settings (optional)
- Delegates to tmux.py functions: resolve_session(), create_session(), launch_agent_in_pane()
- DB registration immediately after session creation (SR-6.5): composite key, initial status "working"
- Command delivery (SR-6.3): without command returns immediately; with command polls read_pane until done state (60s timeout), then delivers via send_command. On timeout returns error with last known agent state.
- Return contract (SR-6.4): {status, session_id, session_name, message}
- All error paths return structured dicts per SR-10.1

## Success Criteria
- spawn_agent creates session and registers DB entry
- All 4 session resolution cases handled correctly
- Without command: returns immediately with session info
- With command: polls until done, delivers command, returns success
- 60s timeout returns error with last known state (e.g., "Agent readiness timeout after 60s. Last state: working")
- Tests pass: poetry run pytest tests/test_server.py -k spawn_agent
