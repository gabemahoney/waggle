## Context
The `CLAUDE_CODE_AGENT_TYPE` environment variable is set by Claude Code for each agent in a team session. Solo agents don't have this variable set. The hook must maintain backward compatibility — when the variable is unset, it behaves exactly as before. The gating check should happen early in the script, before any tmux or database operations, for efficiency.

## What Needs to Change
- In `hooks/set_state.sh`:
  - After the existing state parameter check (line 8: `if [[ -z "$1" ]]`) and delete mode check (line 14), add a new check for `CLAUDE_CODE_AGENT_TYPE`
  - If `CLAUDE_CODE_AGENT_TYPE` is set AND does not equal `"team-lead"`: exit 0 immediately (skip all DB operations)
  - If `CLAUDE_CODE_AGENT_TYPE` is unset OR equals `"team-lead"`: continue with normal execution
  - The skip must be silent — no error output to stderr, exit code 0

## Key Files
- `hooks/set_state.sh`

## Acceptance Criteria
- `CLAUDE_CODE_AGENT_TYPE=teammate` → script exits 0 without writing to DB
- `CLAUDE_CODE_AGENT_TYPE=team-lead` → script writes to DB normally
- `CLAUDE_CODE_AGENT_TYPE` unset → script writes to DB normally (backward compatible)
- `CLAUDE_CODE_AGENT_TYPE=""` (empty string) → script writes to DB normally (treat empty as unset)
- No error output in any case
- Exit code is always 0
