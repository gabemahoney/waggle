## Context
This is the core parser module. It must have NO imports from waggle.tmux, no libtmux, no I/O, no subprocess — pure string operations only (SR-3.1). The module will be imported by `read_pane` in the server module. The PRD provides example text for AskUserQuestion and CheckPermission states that define the patterns to detect.

## What Needs to Change
- Create `src/waggle/state_parser.py` with:
  - Main parse function accepting a string (raw pane text) and returning a tuple of (state_string, prompt_data_dict_or_none)
  - `agent_state` is one of: `"working"`, `"done"`, `"ask_user"`, `"check_permission"`, `"unknown"`
  - Working detection: look for `Esc to interrupt` text in pane content
  - Done detection: look for the `>` input prompt at the bottom of the pane. The prompt appears as a standalone `>` character on the last non-empty line. Must NOT match `>` characters that appear in agent output text (e.g., blockquotes, code output)
  - AskUserQuestion detection: look for `❯` (Unicode heavy right arrow) selector character AND `───` horizontal rule separator. When detected, extract prompt_data containing: the question text (lines above the options), and a list of options where each option has a number, label, and optional description
  - CheckPermission detection: look for both `Do you want to proceed?` AND `Permission rule` text. When detected, extract prompt_data containing: tool type (e.g., "Bash command"), command text, and description
  - Unknown: default when no other pattern matches
  - Priority ordering: CheckPermission > AskUserQuestion > Working > Done > Unknown. This ensures the most specific match wins when patterns overlap (e.g., permission prompts contain numbered options)
  - `prompt_data` is `None` for working, done, and unknown states

## Key Files
- `src/waggle/state_parser.py` (new)

## Acceptance Criteria
- Module imports with no side effects and no external dependencies beyond Python stdlib
- Parse function returns correct state for each of the 5 states
- prompt_data populated with structured question/options for ask_user state
- prompt_data populated with tool/command/description for check_permission state
- prompt_data is None for working, done, unknown states
- Priority ordering prevents misclassification when patterns overlap
