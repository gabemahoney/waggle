## Context
Parser tests must use pre-recorded pane content snapshots stored as text files, not inline string literals (SR-11.3). Snapshots must represent realistic pane output including whitespace, prompt characters, and formatting (SR-11.2). The PRD contains example AskUserQuestion and CheckPermission text that defines the expected patterns.

## What Needs to Change
- Create `tests/fixtures/pane_snapshots/` directory with text files for each state (working, done, ask_user, check_permission, unknown)
- Create `tests/test_state_parser.py` with comprehensive test coverage:
  - Classification tests for all 5 states
  - Prompt data extraction tests for ask_user and check_permission
  - False positive prevention (numbered lists != ask_user, `>` in output != done)
  - Edge cases (empty input, very long content)
- All tests load fixture files — no inline pane content strings

## Why
Fixture-based testing ensures the parser works against realistic pane output and serves as regression canaries if Claude Code's terminal UI changes.

## Success Criteria
- `poetry run pytest tests/test_state_parser.py` passes
- Each of the 4 states correctly classified from fixture snapshots
- prompt_data correctly extracted for ask_user (question + options) and check_permission (tool + command)
- No false positives from numbered lists or `>` in output text
- unknown returned for unrecognized content
- `poetry run pytest` full suite passes
