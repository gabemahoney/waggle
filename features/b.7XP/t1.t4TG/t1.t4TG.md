## Outcome
A pure-function parser correctly classifies raw pane text into Working/Done/AskUser/CheckPermission/Unknown and extracts structured prompt data.

## Work Items

- Create `src/waggle/state_parser.py` — **no tmux dependency, no I/O**, plain string operations only
- Implement state detection with defined priority ordering (SR-3.6) — most specific state wins when multiple patterns match
- **Working** (SR-3.2): detect `Esc to interrupt` indicator
- **Done** (SR-3.3): detect empty `>` prompt at bottom of pane. Must not false-positive on `>` in output text
- **AskUserQuestion** (SR-3.4): detect `❯` selector + `───` horizontal rule separator. Extract structured `prompt_data`: question text, list of options (number, label, description)
- **CheckPermission** (SR-3.5): detect `Do you want to proceed?` + `Permission rule` text. Extract structured `prompt_data`: tool type, command text, description
- **Unknown**: fallback for unrecognized content
- Capture real Claude Code pane output snapshots for all 4 states → `tests/fixtures/pane_snapshots/` (one text file per state)
- Create `tests/test_state_parser.py` — all tests use fixture files, no inline string literals for pane content
- Cover: correct classification per state, prompt_data extraction, unknown fallback, false positive prevention

## Acceptance Criteria
1. `poetry run pytest tests/test_state_parser.py` passes
2. Each of the 4 states correctly classified from fixture snapshots
3. `prompt_data` correctly extracted for `ask_user` (question + options) and `check_permission` (tool + command)
4. No false positives: numbered lists in output not confused with `ask_user`, `>` in output not confused with `done` prompt
5. `unknown` returned for unrecognized content with no crash
