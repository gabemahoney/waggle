## Context
close_session is the MCP tool for shutting down agent sessions with safety guards. Combines DB validation, name/ID cross-validation, LLM protection, and DB-first cleanup.

## What Needs to Change
- Add `close_session` to server.py with `@mcp.tool()` decorator
- Parameters: session_id (required), session_name (optional), force (optional, default false)
- DB validation: session_id must exist in waggle DB, error if not registered
- Name/ID cross-validation: if both provided, call validate_session_name_id(), error on mismatch
- LLM protection (SR-7.3): check via SR-8 shared utility (is_llm_running). LLM + force=false → error: "Active LLM agent, call again with force=true to confirm". LLM + force=true → proceed.
- Cleanup ordering (SR-7.4): DELETE DB entry first (`DELETE FROM state WHERE key = ?`), THEN call kill_session(). If tmux kill fails after DB removal → error: "DB entry removed but tmux session kill failed: {details}"
- Return contract: all paths return {status, message} (SR-7.5)
- Top-level exception handling per SR-10.1

## Key Files
- `src/waggle/server.py` — add close_session tool

## Acceptance Criteria
- DB validation rejects unregistered session_ids
- Name/ID mismatch rejected with descriptive error
- LLM protection enforced: refuses without force, proceeds with force=true
- DB-first cleanup ordering: DB cleaned even if tmux kill fails
- All return paths use {status, message} contract
- No raw exceptions propagate to MCP client
