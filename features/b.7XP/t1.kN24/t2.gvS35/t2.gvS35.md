## Context
The read_pane tool is the primary way an orchestrator observes agent state. It combines pane capture (from tmux.py) with state classification (from state_parser.py) to return structured agent state information.

## What Needs to Change
- Implement `read_pane` MCP tool in `server.py` using `@mcp.tool()` decorator
- Parameters: session_id (required), pane_id (optional), scrollback (optional, default 50)
- Validation: session_id must exist in waggle DB, pane_id must exist within session
- Wire up: tmux.py for pane capture → state_parser.py for classification
- Return contract: {status, agent_state, content, prompt_data} (SR-4.3)
- Add comprehensive tests to test_server.py following existing patterns

## Success Criteria
- read_pane returns correct state and content for all 5 agent states
- Validation rejects unregistered session IDs and invalid pane IDs
- Multi-pane targeting works correctly
- Scrollback parameter controls capture depth
- All tests pass including full suite
