## Outcome
An orchestrator calls `read_pane` and receives the agent's state, raw pane content, and structured prompt data.

## Work Items

- Implement `read_pane` MCP tool in `server.py`, delegating to `tmux.py` (pane capture) and `state_parser.py` (classification)
- Parameters:
  - `session_id` (required): tmux session ID
  - `pane_id` (optional): tmux pane ID (e.g., `%5`) for multi-pane targeting
  - `scrollback` (optional, default 50, no waggle-imposed max): lines of scrollback
- Validation:
  - `session_id` must exist in waggle DB — error if not registered
  - `pane_id` must exist within specified session — descriptive error if not
- When `pane_id` omitted: target active pane of session's active window
- Return contract: `{status, agent_state, content, prompt_data}`
  - `agent_state`: one of `working`, `done`, `ask_user`, `check_permission`, `unknown`
  - `content`: raw pane text (always populated)
  - `prompt_data`: structured question/options for `ask_user`/`check_permission`, null otherwise
- Create tool tests in `tests/test_server.py` — mock tmux layer, test validation and return contracts

## Acceptance Criteria
1. `read_pane` returns correct state and content for mocked agent sessions (all 5 states)
2. Validation rejects unregistered session IDs with clear error
3. Validation rejects invalid/non-existent pane IDs with descriptive error
4. Multi-pane targeting works: `pane_id` provided targets that pane, omitted targets active pane
5. Scrollback parameter controls capture depth
6. Tests pass: `poetry run pytest tests/test_server.py -k read_pane`
